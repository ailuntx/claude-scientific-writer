# 信息图表设计原则

本参考涵盖创建有效、专业的**信息图表**的基本设计原则。

---

## 视觉层级

视觉层级引导观众的眼睛按照预设顺序浏览信息图表，确保关键信息首先被看到。

### 层级金字塔

1. **主要元素**（首先看到）
   - 标题和大字标题
   - 大数字或关键统计数据
   - 主图或主要插图
   - 行动号召元素

2. **次要元素**（其次看到）
   - 副标题和章节标题
   - 图表和图形
   - 图标和视觉标记
   - 关键辅助文本

3. **辅助元素**（最后看到）
   - 正文和描述
   - 图例和标签
   - 来源引用
   - 小字和脚注

### 创建层级

**尺寸**：较大的元素首先吸引注意力
- 标题：比正文大200-300%
- 关键统计数据：数字比标签大2-4倍
- 重要图标：比辅助图标大1.5-2倍

**颜色**：明亮的对比色吸引目光
- 谨慎使用强调色来突出重点
- 将最明亮的颜色留给最重要的元素
- 使用柔和的颜色呈现辅助信息

**位置**：左上角和中间最先被看到
- 最重要的内容放在顶部或中间
- 辅助细节放在底部或边缘
- 阅读流程：从上到下，从左到右（西方文化）

**对比度**：高对比度元素更突出
- 关键文字使用深色配浅色或浅色配深色
- 彩色元素置于中性背景上
- 使用边框和阴影突出关键元素

**留白**：孤立吸引注意力
- 重要元素周围留出空间
- 不要挤占关键信息
- 使用间距将相关项目分组

## 布局模式

### F型布局

最适合：文本密集的信息图表、列表、文章

```
┌─────────────────────────────────────┐
│ ████████████████████████████████████│  ← 顶部水平扫描
├─────────────────────────────────────┤
│ █████████████████                   │  ← 第二次水平扫描
├─────────────────────────────────────┤
│ █████                               │
│ █████                               │  ← 沿左侧向下扫描
│ █████                               │
│ █████                               │
└─────────────────────────────────────┘
```

**应用：**
- 标题横跨顶部整个宽度
- 重要的副标题放在第二行
- 关键内容左对齐
- 次要内容放在右侧

### Z型布局

最适合：内容精简、落地页、单一信息的信息图表

```
┌─────────────────────────────────────┐
│ ●────────────────────────────────→ ●│  ← 从左上角开始，向右扫描
├─────────────────────────────────────┤
│      ╲                              │
│        ╲                            │  ← 对角线扫描
│          ╲                          │
├─────────────────────────────────────┤
│ ●────────────────────────────────→ ●│  ← 从底部左侧到右侧
└─────────────────────────────────────┘
```

**应用：**
- 标志/标题放在左上角
- 关键视觉元素放在右上角
- 目光沿对角线穿过中心
- 行动号召放在右下角

### 单列布局

最适合：移动端友好、滚动内容、流程信息图表

```
┌───────────────┐
│    标题     │
├───────────────┤
│   区块 1    │
├───────────────┤
│   区块 2    │
├───────────────┤
│   区块 3    │
├───────────────┤
│   区块 4    │
├───────────────┤
│    页脚     │
└───────────────┘
```

**应用：**
- 垂直滚动内容
- 逐步流程
- 时间线信息图表
- 移动端优先设计

### 多列布局

最适合：对比、特性列表、复杂数据

```
┌─────────────────────────────────────┐
│           标题/名称              │
├──────────────┬──────────────────────┤
│   列 1       │      列 2            │
│   --------   │      --------        │
│   内容       │      内容            │
│   内容       │      内容            │
│   内容       │      内容            │
├──────────────┴──────────────────────┤
│            页脚                   │
└─────────────────────────────────────┘
```

**应用：**
- 并排对比
- 优缺点列表
- 特性矩阵
- 两类信息

### 网格布局

最适合：多个同等权重项目、统计数据、图标网格

```
┌─────────────────────────────────────┐
│           标题/名称              │
├───────────┬───────────┬─────────────┤
│   项目 1  │   项目 2  │   项目 3    │
├───────────┼───────────┼─────────────┤
│   项目 4  │   项目 5  │   项目 6    │
├───────────┼───────────┼─────────────┤
│   项目 7  │   项目 8  │   项目 9    │
├───────────┴───────────┴─────────────┤
│            页脚                   │
└─────────────────────────────────────┘
```

**应用：**
- 多个统计数据（2x2、3x3、2x3网格）
- 图标集合
- 特性亮点
- 团队成员展示

### 模块/卡片布局

最适合：多样内容类型、灵活信息、现代设计

```
┌─────────────────────────────────────┐
│           标题/名称              │
├───────────────────┬─────────────────┤
│                   │      卡片 2    │
│     卡片 1        ├─────────────────┤
│     （大）        │      卡片 3    │
├───────────────────┼─────────────────┤
│      卡片 4       │      卡片 5    │
└───────────────────┴─────────────────┘
```

**应用：**
- 混合内容类型
- 不同重要程度
- 现代仪表盘风格
- 杂志风格布局

## 60-40法则

最佳信息图表平衡视觉和文本内容：

- **60% 视觉元素**：图标、图表、插图、图片、形状
- **40% 文本内容**：标题、标签、描述、数据

### 为什么这很重要

- 文本过多：感觉像文档而不是信息图表
- 视觉过多：缺乏实质和清晰度
- 适当平衡：既有吸引力又信息丰富

### 应用法则

**视觉元素（60%）**
- 图表和图形
- 图标和符号
- 插图
- 照片
- 装饰形状
- 色块
- 线条和连接线

**文本元素（40%）**
- 标题和大字标题
- 副标题
- 数据标签
- 简要描述
- 来源引用
- 行动号召

## 留白（负空间）

留白是元素之间和周围的空白区域。它不是浪费的空间——而是一种设计工具。

### 留白的功能

1. **提高可读性**：让眼睛在内容之间得到休息
2. **创造焦点**：孤立的元素吸引注意力
3. **分组内容**：相关项目看起来是相连的
4. **增添优雅**：给设计高端感
5. **减少杂乱**：防止观者应接不暇

### 留白指南

**边距**：整个信息图表周围的空格
- 最小为宽度/高度的5-10%
- 越多边距越有高端感
- 各边保持一致

**内边距**：元素内部的空格（盒子、卡片）
- 最小等于文本行高
- 重要元素增加内边距
- 相似元素内部保持一致

**间距**：元素之间的空格
- 相关项目：小间距（8-16px）
- 不相关项目：大间距（24-48px）
- 章节：最大间距（48-72px）

**行距**：文本行之间的空格
- 正文：字号的1.4-1.6倍
- 标题：字号的1.1-1.3倍
- 列表：字号的1.5-2倍

## 字体排版

### 字体选择

**无衬线字体**（推荐用于信息图表）
- 简洁、现代的外观
- 屏幕可读性更好
- 专业感
- 示例：Arial、Helvetica、Open Sans、Roboto、Montserrat

**衬线字体**（谨慎使用）
- 传统、权威感
- 适合正式场合的标题
- 示例：Georgia、Times New Roman、Playfair Display

**展示字体**（仅用于标题）
- 标题的高冲击力
- 不适合正文
- 示例：Impact、Bebas Neue、Oswald

### 字体搭配规则

1. 每个信息图表最多使用2-3种字体
2. **对比是关键**：搭配不同风格（衬线+无衬线）
3. **明确角色**：一种用于标题，一种用于正文，一种用于强调
4. **保持一致**：同一用途始终使用相同字体

**安全搭配：**
- Montserrat（标题）+ Open Sans（正文）
- Playfair Display（标题）+ Roboto（正文）
- Bebas Neue（标题）+ Lato（正文）
- Oswald（标题）+ Source Sans Pro（正文）

### 字体大小

| 元素 | 大小范围 | 字重 |
|---------|------------|--------|
| 主标题 | 36-72pt | 粗体 |
| 章节标题 | 24-36pt | 粗体/中粗 |
| 副标题 | 18-24pt | 中粗 |
| 正文 | 12-16pt | 常规 |
| 说明文字/标签 | 10-14pt | 常规/细 |
| 小字 | 8-10pt | 细 |

### 排版最佳实践

1. **正文左对齐**（比居中更易读）
2. **标题居中**（更有冲击力）
3. **限制行宽**为45-75个字符
4. **谨慎使用粗体**强调
5. **正文避免全大写**（难以阅读）
6. **全大写可用于**短标题/标签
7. **保持文字与背景的对比**（最小4.5:1）

## 故事结构

每个有效的信息图表都讲述一个包含三部分的故事：

### 1. 引言（吸引点）

**目的**：吸引注意力，确立主题

**元素：**
- 引人注目的标题
- 吸睛的主图
- 关键统计数据或问题
- 主题介绍

**最佳实践：**
- 让它不可能被忽视
- 承诺价值（"了解如何..."）
- 创造好奇心
- 占总空间的10-15%

### 2. 主体（内容）

**目的**：传递主要信息

**元素：**
- 数据和统计数据
- 逐步内容
- 对比和分析
- 辅助视觉元素

**最佳实践：**
- 合乎逻辑的流程（按时间、按重要性或按类别）
- 清晰的章节分隔
- 平衡视觉和文字
- 占总空间的70-80%

### 3. 结论（要点）

**目的**：总结，行动号召

**元素：**
- 关键要点或总结
- 行动号召
- 来源引用
- 品牌/署名

**最佳实践：**
- 强化主要信息
- 为观者明确下一步
- 不要引入新信息
- 占总空间的10-15%

## 对齐和网格

### 网格系统

使用不可见网格一致地对齐元素：

**列网格**（最常见）
- 2、3、4或6列
- 元素跨越一列或多列
- 列之间的 gutter（间距）
- 创造有序、专业的外观

**模块网格**
- 列+行=模块
- 对多样化内容更灵活
- 适合复杂布局
- 仪表盘式设计

### 对齐类型

**左对齐**
- 文本最常用
- 创造强左边缘
- 易于扫描
- 专业外观

**居中对齐**
- 适合标题
- 创造对称感
- 文本谨慎使用
- 适合单一元素

**右对齐**
- 很少用于主要内容
- 适合表格中的数字
- 西方设计中可能感觉不寻常
- 刻意使用

### 对齐最佳实践

1. **选择一种主要对齐方式**并坚持使用
2. **相关元素对齐**到同一边缘或中心
3. **使用不可见的网格线**保持一致性
4. **避免随机放置**——一切都应该对齐到某处
5. **通过对齐创造视觉连接**

## 颜色运用

### 信息图表中颜色的功能

1. **建立层级**：亮色用于重要项目
2. **分组相关项目**：相同颜色=相同类别
3. **创造对比**：区分元素
4. **唤起情感**：颜色具有心理含义
5. **强化品牌**：与品牌身份一致

### 颜色分布

**60-30-10法则：**
- **60%** 主色（背景、大区域）
- **30%** 辅助色（辅助元素）
- **10%** 强调色（高亮、行动号召）

### 颜色心理学

| 颜色 | 联想 | 最适合 |
|-------|-------------|----------|
| 蓝色 | 信任、专业、平静 | 企业、科技、医疗 |
| 绿色 | 成长、自然、金钱 | 环境、金融、健康 |
| 红色 | 紧迫、活力、热情 | 警报、销售、食品 |
| 橙色 | 友好、自信、创意 | 行动号召、年轻品牌 |
| 黄色 | 乐观、谨慎、关注 | 高亮、警告 |
| 紫色 | 奢华、创意、智慧 | 高端品牌、教育 |
| 黑色 | 精致、力量、优雅 | 奢华、正式 |
| 白色 | 简洁、简单、空间 | 背景、呼吸空间 |

### 对比度要求

无障碍要求（WCAG 2.1 AA）：
- **正常文字**：最小4.5:1对比度
- **大号文字**（18pt+）：最小3:1对比度
- **图形和UI**：最小3:1对比度

检查对比度的工具：
- WebAIM Contrast Checker
- Coolors Contrast Checker
- Adobe Color Accessibility Tools

## 图标运用

### 图标风格

**线框图标**（轮廓）
- 简洁、现代外观
- 小尺寸效果也很好
- 极简设计最佳
- 保持线条粗细一致很重要

**填充图标**（实心）
- 视觉冲击力更强
- 快速识别效果好
- 作为焦点效果很好
- 小尺寸更易读

**插图式图标**
- 更多个性和独特性
- 视觉重量更大
- 活泼设计最佳
- 可能缩放效果不佳

### 图标最佳实践

1. **整个信息图表中保持一种风格**
2. **确保可识别性**——图标应能立即被理解
3. **相同层级的图标保持一致尺寸**
4. **图标含义不明确时添加标签**
5. **匹配图标的视觉重量与周围元素**
6. **谨慎考虑颜色**——单色通常更简洁
7. **避免图标过载**——不是所有东西都需要图标

### 图标大小指南

| 场景 | 推荐尺寸 |
|---------|------------------|
| 主角/特色图标 | 64-128px |
| 章节图标 | 32-48px |
| 列表项图标 | 24-32px |
| 行内图标 | 16-24px |

## 数据可视化最佳实践

### 选择图表类型

| 数据类型 | 最佳图表 |
|-----------|------------|
| 对比（少数项目） | 柱状图 |
| 对比（多数项目） | 横向柱状图 |
| 整体的一部分 | 饼图/环形图 |
| 随时间变化 | 折线图 |
| 分布 | 直方图 |
| 关系 | 散点图 |
| 地理 | 地图/等值区域图 |
| 层级 | 树状图 |
| 流程/过程 | 桑基图 |

### 图表最佳实践

1. **标注一切**：坐标轴、数据点、图例
2. **柱状图的Y轴从零开始**（避免误导）
3. **饼图切片最多5-7个**
4. **相同类别在不同图表中使用一致颜色**
5. **去除图表垃圾**：无3D效果，最小化网格线
6. **突出关键数据**：用颜色强调重要点

### 数字呈现

- **大数字**：使用缩写（1.2M，而非1,200,000）
- **百分比**：包含%符号，最多一位小数
- **比较**：使用一致的单位和精度
- **上下文**：始终提供参考点（"是行业平均的2倍"）

## 无障碍考虑

### 视觉无障碍

1. **颜色不应单独传达含义**
   - 添加图案、标签或形状
   - 适合色盲用户

2. **足够的对比度**
   - 正常文字4.5:1
   - 大号文字和图形3:1

3. **文字大小**
   - 印刷最小10pt
   - 数字最小12px

4. **不依赖颜色图例**
   - 可能时直接标注数据

### 色盲安全设计

- 使用色盲安全调色板（见color_palettes.md）
- 用色盲模拟器测试
- 添加图案或纹理来区分
- 使用标签和直接标注

### 阅读无障碍

- 清晰的层级和流程
- 简洁的文字
- 简单的语言
- 足够的间距
- 合乎逻辑的阅读顺序

## 质量检查清单

在最终确定您的信息图表之前，请验证：

### 布局
- [ ] 清晰的视觉层级
- [ ] 一致的对齐（基于网格）
- [ ] 足够的留白
- [ ] 合乎逻辑的阅读流程
- [ ] 平衡的构图

### 字体排版
- [ ] 最多使用2-3种字体
- [ ] 可读的字体大小
- [ ] 足够的文字对比度
- [ ] 相同元素样式一致
- [ ] 正文左对齐

### 颜色
- [ ] 60-30-10分布
- [ ] 色盲安全调色板
- [ ] 足够对比度（文字4.5:1）
- [ ] 一致的颜色含义
- [ ] 不过于繁杂

### 内容
- [ ] 清晰的故事结构（引言、主体、结论）
- [ ] 约60%视觉、40%文字
- [ ] 关键信息突出
- [ ] 数据准确且有来源
- [ ] 包含行动号召

### 图标和图形
- [ ] 一致的图标风格
- [ ] 适当的尺寸
- [ ] 可识别的含义
- [ ] 不滥用

### 无障碍
- [ ] 灰度模式下可用
- [ ] 图案/标签补充颜色
- [ ] 在预期尺寸下可读
- [ ] 无视觉线索时也有逻辑流程

---

以这些原则为基础，根据您的具体内容和受众进行调整。
