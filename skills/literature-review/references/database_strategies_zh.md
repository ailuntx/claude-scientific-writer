# 文献数据库检索策略

本文件为系统有效检索多个文献数据库提供全面指导。

## 可用数据库与技能

### 生物医学与生命科学

#### PubMed / PubMed Central
- **访问方式**：使用 `gget` 技能或 WebFetch 工具
- **覆盖范围**：生物医学文献中超过 3500 万条引文
- **最佳用途**：临床研究、生物医学研究、遗传学、分子生物学
- **检索技巧**：使用 MeSH 术语、布尔运算符（AND、OR、NOT）、字段标签 [Title]、[Author]
- **示例**：`"CRISPR"[Title] AND "gene editing"[Title/Abstract] AND 2020:2024[Publication Date]`

#### bioRxiv / medRxiv
- **访问方式**：使用 `gget` 技能或直接 API
- **覆盖范围**：生物学和医学预印本
- **最佳用途**：最新未发表的研究、前沿发现
- **注意**：未经同行评审；谨慎验证发现
- **检索技巧**：按类别搜索（生物信息学、基因组学等）

### 通用科学文献

#### arXiv
- **访问方式**：直接 API 访问
- **覆盖范围**：物理、数学、计算机科学、定量生物学预印本
- **最佳用途**：计算方法、生物信息学算法、理论工作
- **类别**：q-bio（定量生物学）、cs.LG（机器学习）、stat.ML（统计学）
- **检索格式**：`cat:q-bio.QM AND title:"single cell"`

#### Semantic Scholar
- **访问方式**：直接 API（需要 API 密钥）
- **覆盖范围**：各领域超过 2 亿篇论文
- **最佳用途**：跨学科检索、引文图谱、论文推荐
- **功能**：有影响力的引文、论文摘要、相关论文
- **速率限制**：使用 API 密钥每 5 分钟 100 次请求

#### Google Scholar
- **访问方式**：网页抓取（谨慎使用）或手动搜索
- **覆盖范围**：全领域综合覆盖
- **最佳用途**：查找高被引论文、会议论文、学位论文
- **限制**：无官方 API，存在速率限制
- **导出**：使用"引用"功能获取格式化引文

### 专业数据库

#### ChEMBL / PubChem
- **访问方式**：使用 `gget` 技能或 `bioservices` 技能
- **覆盖范围**：化合物、生物活性数据、药物分子
- **最佳用途**：药物发现化学生物学、药物化学
- **ChEMBL**：超过 200 万化合物、生物活性数据
- **PubChem**：超过 1.1 亿化合物、实验数据

#### UniProt
- **访问方式**：使用 `gget` 技能或 `bioservices` 技能
- **覆盖范围**：蛋白质序列和功能信息
- **最佳用途**：蛋白质研究、序列分析、功能注释
- **检索方式**：蛋白质名称、基因名称、生物体、功能

#### KEGG（京都基因与基因组百科全书）
- **访问方式**：使用 `bioservices` 技能
- **覆盖范围**：通路、疾病、药物、基因
- **最佳用途**：通路分析、系统生物学、代谢研究

#### COSRIC（癌症体细胞突变目录）
- **访问方式**：使用 `gget` 技能或直接下载
- **覆盖范围**：癌症基因组、体细胞突变
- **最佳用途**：癌症研究、突变分析

#### AlphaFold 数据库
- **访问方式**：使用 `gget` 技能与 `alphafold` 命令
- **覆盖范围**：超过 2 亿个蛋白质结构预测
- **最佳用途**：结构生物学、蛋白质建模

#### PDB（蛋白质数据库）
- **访问方式**：使用 `gget` 或直接 API
- **覆盖范围**：蛋白质、核酸的实验 3D 结构
- **最佳用途**：结构生物学、药物设计、分子建模

### 引文与参考文献管理

#### OpenAlex
- **访问方式**：直接 API（免费，无需密钥）
- **覆盖范围**：超过 2.5 亿作品，综合元数据
- **最佳用途**：引文分析、作者消歧、机构研究
- **功能**：开放获取，非常适合文献计量学

#### Dimensions
- **访问方式**：提供免费套餐
- **覆盖范围**：出版物、资助、专利、临床试验
- **最佳用途**：研究影响、资助分析、转化研究

## 检索策略框架

### 1. 定义研究问题（PICO 框架）

对于临床/生物医学综述：
- **P**opulation（人群）：研究是关于谁的？
- **I**ntervention（干预）：正在测试什么？
- **C**omparison（比较）：与什么比较？
- **O**utcome（结局）：结果是什么？

**示例**："CRISPR-Cas9 基因疗法（I）治疗镰状细胞病（P）与标准护理（C）相比在改善患者结局（O）方面的疗效如何？"

### 2. 开发检索词

#### 主要概念
从研究问题中识别 2-4 个主要概念。

**示例**：
- 概念 1：CRISPR、Cas9、基因编辑
- 概念 2：镰状细胞病、SCD、血红蛋白疾病
- 概念 3：基因疗法、治疗性编辑

#### 同义词与相关术语
列出替代术语、缩写和相关概念。

**工具**：使用 MeSH（医学主题词）浏览器获取标准化术语

#### 布尔运算符
- **AND**：缩小检索范围（必须包含两个术语）
- **OR**：扩大检索范围（包含任一术语）
- **NOT**：排除术语

**示例**：`(CRISPR OR Cas9 OR "gene editing") AND ("sickle cell" OR SCD) AND therapy`

#### 通配符与截词
- `*` 或 `%`：匹配任意字符
- `?`：匹配单个字符

**示例**：`genom*` 匹配 genomic、genomics、genome

### 3. 设定纳入/排除标准

#### 纳入标准
- **日期范围**：例如 2015-2024（最近 10 年）
- **语言**：英语（或指定多语言）
- **出版物类型**：同行评审文章、综述、预印本
- **研究设计**：随机对照试验、队列研究、荟萃分析
- **人群**：人类、动物模型、体外

#### 排除标准
- 病例报告（n<5）
- 无全文的会议摘要
- 非原创研究（社论、评论）
- 重复出版物
- 撤回的文章

### 4. 数据库选择策略

#### 多数据库方法
至少检索 3 个互补数据库：

1. **主要数据库**：PubMed（生物医学）或 arXiv（计算）
2. **预印本服务器**：bioRxiv/medRxiv 或 arXiv
3. **综合数据库**：Semantic Scholar 或 Google Scholar
4. **专业数据库**：ChEMBL、UniProt 或领域特定数据库

#### 数据库特定语法

| 数据库 | 字段标签 | 示例 |
|----------|-----------|---------|
| PubMed | [Title], [Author], [MeSH] | "CRISPR"[Title] AND 2020:2024[DP] |
| arXiv | ti:, au:, cat: | ti:"machine learning" AND cat:q-bio.QM |
| Semantic Scholar | title:, author:, year: | title:"deep learning" year:2020-2024 |

## 检索执行工作流程

### 阶段 1：试点检索
1. 使用广泛术语运行初步检索
2. 查看前 50 个结果的相关性
3. 记录常见关键词和 MeSH 术语
4. 优化检索策略

### 阶段 2：全面检索
1. 在所有选定的数据库中执行优化后的检索
2. 以标准格式导出结果（RIS、BibTeX、JSON）
3. 记录每个数据库的检索字符串和日期
4. 记录每个数据库的结果数量

### 阶段 3：去重
1. 将所有结果导入单个文件
2. 使用 `search_databases.py --deduplicate` 去除重复项
3. 按 DOI（主要）或标题（备用）识别重复项
4. 保留元数据最完整的版本

### 阶段 4：筛选
1. **标题筛选**：审查标题，排除明显无关的内容
2. **摘要筛选**：阅读摘要，应用纳入/排除标准
3. **全文筛选**：获取并审查全文
4. 在每个阶段记录排除原因

### 阶段 5：质量评估
1. 使用适当的工具评估研究质量：
   - **随机对照试验**：Cochrane 偏倚风险工具
   - **观察性研究**：纽卡斯尔-渥太华量表
   - **系统综述**：AMSTAR 2
2. 评定证据质量（高、中、低、极低）
3. 考虑排除质量极低的研究

## 高级检索技术

### 优先关注高影响力论文（关键）

**始终根据引文数量、期刊质量和作者声誉优先考虑论文。质量比数量更重要。**

#### 数据库检索中的引文指标

使用引文数量识别有影响力的工作：

| 论文年龄 | 引文数 | 分类 |
|-----------|-----------|----------------|
| 0-3 年 | 20+ | 值得注意 |
| 0-3 年 | 100+ | 高度有影响力 |
| 3-7 年 | 100+ | 重要 |
| 3-7 年 | 500+ | 里程碑 |
| 7+ 年 | 500+ | 开创性 |
| 7+ 年 | 1000+ | 奠基性 |

**数据库特定的引文功能：**
- **Google Scholar**：按引文数量排序，使用"被引用"功能
- **Semantic Scholar**："高度有影响力引文"指标，引文速度
- **OpenAlex**：引文数量、引文上下文分析
- **PubMed**：在 PMC 中使用"被引用"，通过 Google Scholar 检查引文数量

#### 按期刊质量筛选

优先选择来自更高层次期刊的论文：

**第一层次（始终首选）：**
- Nature、Science、Cell、NEJM、Lancet、JAMA、PNAS
- Nature Medicine、Nature Biotechnology、Nature Methods
- 检索技巧：在 Google Scholar 中使用 `source:Nature` 或 `journal:Nature`

**第二层次（高优先级）：**
- 高影响力专业期刊（影响因子 >10）
- 顶级会议：NeurIPS、ICML、ICLR、CVPR、ACL

**第三层次（相关时纳入）：**
- 值得尊重的领域特定期刊（IF 5-10）

**PubMed 期刊筛选：**
```
"Nature"[Journal] OR "Science"[Journal] OR "Cell"[Journal]
```

**Google Scholar 期刊筛选：**
```
source:Nature source:Science source:Cell
```

#### 利用"被引用"功能

**发现有影响力的工作：**
1. 从一篇已知的关键论文开始
2. 点击"被引用"找到引用它的论文
3. 按引用数量排序引用论文
4. 高被引用的引用论文表明重要的后续工作

**识别开创性论文：**
1. 广泛搜索您的主题
2. 注意哪些论文在参考文献列表中反复出现
3. 被多个结果引用的论文可能是开创性的
4. 检查引文数量以确认影响力

**Semantic Scholar 功能：**
- "高度有影响力引文"显示显著建立在该论文基础上的引文
- "引文速度"显示最近的引文增长
- 基于引文网络的论文推荐

### 引文链

#### 前向引文检索
查找引用关键论文的论文：
- 使用 Google Scholar"被引用"功能
- 使用 OpenAlex 或 Semantic Scholar API
- 识别建立在开创性工作上的较新研究
- **技巧**：按引文数量排序以找到最有影响力的后续工作

#### 后向引文检索
审查关键论文中的参考文献：
- 从纳入的论文中提取参考文献
- 搜索高被引的参考文献（较旧的论文 500+ 引文）
- 识别基础研究
- **技巧**：关注在多篇论文参考文献中出现的参考文献

### 滚雪球抽样
1. 从 3-5 篇来自第一层次期刊的高度相关论文开始
2. 提取所有参考文献
3. 检查哪些参考文献被多篇论文引用
4. 审查那些高重复率的参考文献——这些可能是开创性的
5. 对新识别的关键论文重复此过程
6. **在每一步优先选择高被引论文**

### 作者检索
关注该领域多产且有声望的作者：
- 在数据库中按作者姓名搜索
- 检查作者档案（ORCID、Google Scholar）的 h 指数和发表期刊
- 审查最近的出版物和预印本
- **优先选择多篇第一层次出版物且高 h 指数（>40）的作者**
- 寻找被认可的领域资深领导者

### 相关文章功能
许多数据库推荐相关文章：
- PubMed"类似文章"
- Semantic Scholar"推荐论文"
- 用于发现关键词检索遗漏的论文
- **按引文数量和期刊质量筛选推荐**

## 应避免的常见陷阱

1. **检索过于狭窄**：遗漏相关论文
   - 解决方案：包含同义词、相关术语、更广泛的概念

2. **检索过于宽泛**：数千个无关结果
   - 解决方案：使用 AND 添加特定概念，使用字段标签

3. **单一数据库**：覆盖不完整
   - 解决方案：至少检索 3 个互补数据库

4. **忽略预印本**：遗漏最新发现
   - 解决方案：包含 bioRxiv、medRxiv 或 arXiv

5. **无文档记录**：检索不可重复
   - 解决方案：记录每个检索字符串、日期和结果数量

6. **手动去重**：耗时且容易出错
   - 解决方案：使用 search_databases.py 脚本

7. **未验证的引文**：损坏的 DOI、错误的元数据
   - 解决方案：在最终参考文献列表上运行 verify_citations.py

8. **发表偏倚**：仅纳入已发表的正向结果
   - 解决方案：检索试验注册库，联系作者获取未发表数据
