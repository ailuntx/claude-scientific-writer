# 科学批评性评估与同行评审

## 概述

同行评审是评估科学手稿的系统化过程。评估方法论、统计、设计、可重复性、伦理和报告标准。应用此技能进行跨学科的手稿和资助评审，提供建设性、严格的评估。

## 何时使用此技能

此技能应在以下情况下使用：
- 为期刊进行科学手稿的同行评审
- 评估资助提案和研究申请
- 评估方法论和实验设计的严谨性
- 审查统计分析和报告标准
- 评估可重复性和数据可用性
- 检查是否符合报告指南（CONSORT、STROBE、PRISMA）
- 就科学写作提供建设性反馈

**相关资源：** **venue-templates** 技能提供 `reviewer_expectations.md`，其中包含不同场所（Nature/Science、Cell Press、医学期刊、ML会议）审稿人关注点的详细指导。使用此资源来校准您的评审标准以适应目标场所。

## 同行评审工作流程

通过以下阶段系统地进行同行评审，根据手稿类型和学科调整深度和重点。

### 阶段 1：初步评估

首先进行高层评估，以确定手稿的范围、新颖性和整体质量。

**关键问题：**
- 中心研究问题或假设是什么？
- 主要发现和结论是什么？
- 工作在科学上是否合理且有意义？
- 工作是否适合目标场所？
- 是否有任何会阻止发表的立即重大缺陷？

**输出：** 简要总结（2-3句话），捕捉手稿的本质和初步印象。

### 阶段 2：详细的逐部分评审

对手稿的每个部分进行彻底评估，记录具体问题和优点。

#### 摘要和标题
- **准确性：** 摘要是否准确反映研究内容和结论？
- **清晰度：** 标题是否具体、准确且信息丰富？
- **完整性：** 关键发现和方法是否被适当总结？
- **可接近性：** 摘要是否对广大科学受众可理解？

#### 引言
- **背景：** 背景信息是否充分且最新？
- **理由：** 研究问题是否清晰有据？
- **新颖性：** 工作的原创性和意义是否清晰阐述？
- **文献：** 是否有适当引用相关先前研究？
- **目标：** 研究目标/假设是否清晰陈述？

#### 方法
- **可重复性：** 另一位研究人员能否根据所提供描述复制该研究？
- **严谨性：** 方法是否适合解决研究问题？
- **细节：** 方案、试剂、设备和参数是否充分描述？
- **伦理：** 伦理批准、同意和数据处理是否适当记录？
- **统计：** 统计方法是否适当、清晰描述和有依据？
- **验证：** 对照、重复和验证方法是否充分？

**需要验证的关键要素：**
- 样本量和功效计算
- 随机化和盲法程序
- 纳入/排除标准
- 数据收集方案
- 计算方法和软件版本
- 统计检验和多重比较校正

#### 结果
- **呈现：** 结果是否逻辑清晰呈现？
- **图表：** 可视化是否适当、清晰且正确标注？
- **统计：** 统计结果是否正确报告（效应量、置信区间、p值）？
- **客观性：** 结果呈现是否没有过度解读？
- **完整性：** 是否包含所有相关结果，包括阴性结果？
- **可重复性：** 是否提供原始数据或汇总统计？

**需要识别的问题：**
- 选择性报告结果
- 不适当的统计检验
- 缺少误差线或变异性度量
- 过度拟合或循环分析
- 批次效应或混杂变量
- 缺少对照或验证实验

#### 讨论
- **解释：** 结论是否由数据支持？
- **局限性：** 是否承认并讨论研究局限性？
- **背景：** 发现是否适当地放在现有文献中？
- **推测：** 是否将推测与数据支持的结论明确区分？
- **意义：** 是否清晰阐述含义和重要性？
- **未来方向：** 是否讨论了下一步或未解决的问题？

**危险信号：**
- 结论过甚
- 忽略矛盾证据
- 从相关性数据得出因果声明
- 未充分讨论局限性
- 没有机制证据的机制声明

#### 参考文献
- **完整性：** 是否引用了关键相关论文？
- **时效性：** 是否包含最近的重要研究？
- **平衡性：** 是否适当引用了相反观点？
- **准确性：** 引用是否准确和适当？
- **自引：** 是否存在过度或不适当自引？

### 阶段 3：方法论和统计严谨性

评估研究的技术质量和严谨性，特别注意常见陷阱。

**统计评估：**
- 是否满足统计假设（正态性、独立性、同方差性）？
- 是否报告了效应量以及p值？
- 是否适当应用多重检验校正？
- 是否提供置信区间？
- 是否用功效分析解释样本量？
- 是否适当选择参数与非参数检验？
- 是否适当处理缺失数据？
- 是否区分探索性与验证性分析？

**实验设计：**
- 对照是否适当和充分？
- 重复是否充分（生物学和技术）？
- 是否识别和控制潜在混杂因素？
- 是否正确实施随机化？
- 盲法程序是否充分？
- 实验设计是否最适合研究问题？

**计算/生物信息学：**
- 计算方法是否清晰描述和有依据？
- 是否记录软件版本和参数？
- 是否为可重复性提供代码？
- 算法和模型是否适当验证？
- 是否满足计算方法的假设？
- 是否适当应用批次校正？

### 阶段 4：可重复性和透明度

评估研究是否满足现代可重复性和开放科学标准。

**数据可用性：**
- 原始数据是否存放在适当的仓库？
- 是否为公共数据库提供 accession 编号？
- 数据共享限制是否有依据（如患者隐私）？
- 数据格式是否标准且可访问？

**代码和材料：**
- 分析代码是否可用（GitHub、Zenodo等）？
- 独特材料是否可用或描述充分以供复制？
- 方案是否足够详细？

**报告标准：**
- 手稿是否遵循学科特定的报告指南（CONSORT、PRISMA、ARRIVE、MIAME、MINSEQE等）？
- 参见 `references/reporting_standards.md` 了解常见指南
- 是否解决了适当清单的所有要素？

### 阶段 5：图表和数据呈现

评估数据可视化的质量、清晰度和完整性。

**质量检查：**
- 图表是否高分辨率且清晰标注？
- 轴是否正确标注单位？
- 误差线是否界定（SD、SEM、CI）？
- 统计显著性指标是否解释？
- 颜色方案是否适当且无障碍（适合色盲）？
- 图像是否包含比例尺？
- 数据可视化是否适合数据类型？

**完整性检查：**
- 是否有图像处理的迹象（重复、拼接）？
- Western印迹和凝胶是否适当呈现？
- 代表性图像是否真正具有代表性？
- 是否显示所有条件（无选择性呈现）？

**清晰度：**
- 图表是否可独立于图例理解？
- 每个图表的信息是否立即清晰？
- 是否有冗余图表或面板？
- 数据是否更适合以表格或图表呈现？

### 阶段 6：伦理考虑

验证研究是否满足伦理标准指南。

**人类受试者：**
- 是否记录IRB/伦理批准？
- 是否描述知情同意？
- 是否适当保护弱势群体？
- 患者隐私是否充分保护？
- 是否披露潜在利益冲突？

**动物研究：**
- 是否记录IACUC或同等批准？
- 程序是否人道和有依据？
- 是否考虑3R原则（替代、减少、优化）？
- 安乐死方法是否适当？

**研究诚信：**
- 是否有数据伪造或篡改的担忧？
- 作者身份是否适当和有依据？
- 是否披露竞争利益？
- 是否披露资金来源？
- 是否有抄袭或重复发表的担忧？

### 阶段 7：写作质量和清晰度

评估手稿的清晰度、组织性和可接近性。

**结构和组织：**
- 手稿是否逻辑组织？
- 各部分是否连贯过渡？
- 想法之间的过渡是否清晰？
- 叙述是否有说服力和清晰？

**写作质量：**
- 语言是否清晰、精确、简洁？
- 是否尽量减少行话和缩写并加以定义？
- 语法和拼写是否正确？
- 句子是否不必要的复杂？
- 是否过度使用被动语态？

**可接近性：**
- 非专业人员能否理解主要发现？
- 技术术语是否解释？
- 意义是否对广大受众清晰？

## 构建同行评审报告

以分层结构组织反馈，优先处理问题并提供可操作的指导。

### 总结陈述

提供简明的整体评估（1-2段）：
- 研究简要概述
- 总体建议（接受、小修、大修、拒绝）
- 主要优点（2-3点）
- 主要缺点（2-3点）
- 对意义和科学性的底线评估

### 主要评论

列出显著影响手稿有效性、可解释性或意义的关键问题。编号以便轻松参考。

**主要评论通常包括：**
- 根本性的方法论缺陷
- 不适当的统计分析
- 无依据或过甚的结论
- 缺少关键对照或实验
- 严重的可重复性问题
- 文献覆盖的主要空白
- 伦理问题

**对于每个主要评论：**
1. 明确陈述问题
2. 解释为什么有问题
3. 建议具体解决方案或额外实验
4. 表明解决它对发表是否必要

### 次要评论

列出不太关键的问题，这些问题会提高清晰度、完整性或呈现效果。编号以便参考。

**次要评论通常包括：**
- 不清晰的图表标签或图例
- 缺少方法论细节
- 拼写或语法错误
- 改进数据呈现的建议
- 次要的统计报告问题
- 加强结论的补充分析
- 澄清请求

**对于每个次要评论：**
1. 识别具体位置（部分、段落、图表）
2. 明确陈述问题
3. 建议如何解决

### 具体逐行评论（可选）

对于需要详细反馈的手稿，提供部分特定或逐行评论：
- 引用具体页/行号或章节
- 注意事实错误、不清晰陈述或缺少引用
- 为清晰起见建议具体编辑

### 给作者的问题

列出需要澄清的具体问题：
- 不清晰的方法论细节
- 看似矛盾的结果
- 评估工作所需的缺失信息
- 请求额外数据或分析

## 语气和方法

在整个评审过程中保持建设性、专业性和协作性的语气。

**最佳实践：**
- **建设性：** 将批评框定为改进的机会
- **具体：** 提供具体示例和可操作的建议
- **平衡：** 认可优点和缺点
- **尊重：** 记得作者投入了大量努力
- **客观：** 聚焦于科学，而非科学家
- **彻底：** 不要忽视问题，但适当优先排序
- **清晰：** 避免模糊或模糊的批评

**避免：**
- 人身攻击或轻蔑的语言
- 讽刺或傲慢
- 没有具体示例的模糊批评
- 要求超出范围的不必要实验
- 要求坚持个人偏好而非最佳实践
- 如果评审是双盲的，透露您的身份

## 手稿类型的特殊考虑

### 原创研究文章
- 强调严谨性、可重复性和新颖性
- 评估意义和影响
- 验证结论是由数据驱动的
- 检查方法是否完整和对照是否适当

### 综述和荟萃分析
- 评估文献覆盖的全面性
- 评估搜索策略和纳入/排除标准
- 验证系统方法且无偏见
- 检查是否有关键分析而非仅仅总结
- 对于荟萃分析，评估统计方法和异质性

### 方法论文
- 强调与现有方法的验证和比较
- 评估方案/代码的可重复性和可用性
- 评估相对于现有方法的改进
- 检查是否有足够细节以便实施

### 简短报告/信函
- 调整对简洁的期望
- 确保核心发现仍然严谨且有意义
- 验证格式是否适合发现

### 预印本
- 认识到这些尚未经过正式同行评审
- 可能不如期刊投稿那么完善
- 仍然应用严格的科学有效性标准
- 考虑提供建设性反馈以帮助作者在期刊投稿前改进

### 演示文稿和幻灯片

**⚠️ 关键：对于演示文稿，切勿直接读取PDF。务必先转换为图像。**

在审阅科学演示文稿（PowerPoint、Beamer、幻灯片）时：

#### 强制性基于图像的评审工作流程

**切勿尝试直接读取演示文稿PDF** - 这会导致缓冲区溢出错误且不会显示视觉格式问题。

**必需流程：**
1. 使用Python将PDF转换为图像：
   ```bash
   python skills/scientific-slides/scripts/pdf_to_images.py presentation.pdf review/slide --dpi 150
   # 创建：review/slide-001.jpg、review/slide-002.jpg等
   ```
2. 按顺序读取并检查每个幻灯片图像文件
3. 用具体幻灯片编号记录问题
4. 提供关于视觉格式和内容的反馈

**开始审阅时打印：**
```
[HH:MM:SS] PEER REVIEW: 检测到演示文稿 - 正在转换为图像进行审阅
[HH:MM:SS] PDF REVIEW: 切勿直接读取PDF - 使用基于图像的检查
```

#### 演示文稿特定评估标准

**视觉设计和可读性：**
- [ ] 文本足够大（最小18pt，正文文本最好24pt+）
- [ ] 文本与背景之间的高对比度（最低4.5:1，最好7:1）
- [ ] 颜色方案专业且适合色盲
- [ ] 所有幻灯片视觉设计一致
- [ ] 空白空间适当（不拥挤）
- [ ] 字体清晰专业

**布局和格式（检查每个幻灯片图像）：**
- [ ] 幻灯片边缘无文本溢出或截断
- [ ] 无元素重叠（文本在图像上、形状重叠）
- [ ] 标题位置一致
- [ ] 内容正确对齐
- [ ] 项目符号和文本未被截断
- [ ] 图表在幻灯片边界内
- [ ] 图例和标签可见可读

**内容质量：**
- [ ] 每个幻灯片一个主要想法（不过载）
- [ ] 文本最少（每个幻灯片最多3-6个项目符号）
- [ ] 项目符号简洁（每个5-7个词）
- [ ] 图表简化清晰（不是从论文复制粘贴）
- [ ] 数据可视化有大而可读的标签
- [ ] 引用存在且格式正确
- [ ] 结果/数据幻灯片占主导（内容的40-50%）

**结构和流程：**
- [ ] 清晰的叙事弧线（引言→方法→结果→讨论）
- [ ] 幻灯片之间的逻辑进展
- [ ] 幻灯片数量适合演讲时长（约每分钟1张幻灯片）
- [ ] 标题幻灯片包含作者、单位、日期
- [ ] 引言引用相关背景文献（3-5篇论文）
- [ ] 讨论引用比较论文（3-5篇论文）
- [ ] 结论幻灯片总结关键发现
- [ ] 致谢/资助幻灯片在最后

**科学内容：**
- [ ] 研究问题清晰陈述
- [ ] 方法充分总结（不过多细节）
- [ ] 结果逻辑呈现并有清晰可视化
- [ ] 统计显著性适当标注
- [ ] 结论由显示的数据支持
- [ ] 局限性适当承认
- [ ] 讨论未来方向或更广泛影响

**常见演示文稿问题标记：**

**关键问题（必须修复）：**
- 文本溢出使内容不可读
- 字体太小（<18pt）
- 元素重叠遮挡数据
- 对比度不足（文本难以阅读）
- 图表太复杂或难以辨认
- 无引用（完全无支持的声明）
- 幻灯片数量与时长严重不匹配

**主要问题（应该修复）：**
- 幻灯片之间设计不一致
- 文本太多（整面文本，而非项目符号）
- 图表简化不当（轴标签太小）
- 布局拥挤，空白空间不足
- 缺少关键结构元素（无结论幻灯片）
- 颜色选择不当（不适合色盲）
- 结果内容最少（<30%幻灯片）

**次要问题（改进建议）：**
- 可以使用更多视觉/图表
- 有些幻灯片文字稍多
- 轻微对齐不一致
- 可以从更多空白空间受益
- 额外引用将加强声明
- 颜色方案可以更现代

#### 演示文稿的评审报告格式

**总结陈述：**
- 演示文稿质量的整体印象
- 对目标受众和时长的适当性
- 主要优点（视觉设计、内容、清晰度）
- 主要缺点（格式问题、内容差距）
- 建议（可以演示、小修、大修）

**布局和格式问题（按幻灯片编号）：**
```
Slide 3: 文本溢出 - 第4个项目符号超出右边缘
Slide 7: 元素重叠 - 图表与图例文本重叠
Slide 12: 字体大小 - 轴标签太小，距离太远看不清
Slide 18: 对齐 - 标题未居中
```

**内容和结构反馈：**
- 背景和引用的充分性
- 研究问题和目标的清晰度
- 方法总结的质量
- 结果呈现的有效性
- 结论和含义的强度

**设计和无障碍性：**
- 整体视觉吸引力和专业性
- 颜色对比度和可读性
- 色盲无障碍性
- 幻灯片之间的一致性

**时间和范围：**
- 幻灯片数量是否匹配预期时长
- 细节水平是否适合演讲类型
- 各部分之间的平衡
