# 科技论文中的常见方法学和统计学问题

本文档整理了同行评审中经常遇到的问题，按类别组织。作为参考，用于识别潜在问题并提供建设性反馈。

## 统计学问题

### 1. P值的误用和误解

**常见问题：**
- P值操纵（选择性地报告显著结果）
- 未进行校正的多重检验（族错误率膨胀）
- 将不显著解释为无效果的证据
- 只关注p值而忽略效应量
- 在任意阈值处将连续p值二分化（p=0.049 vs p=0.051）
- 混淆统计显著性与人/临床意义

**如何识别：**
- 可疑的高比例p值刚好低于0.05
- 进行了许多检验但未提及校正
- 使用"未发现差异"等表述来描述不显著结果
- 未报告效应量或置信区间
- 语言暗示p值代表效果强度

**建议措施：**
- 报告带置信区间的效应量
- 应用适当的多重检验校正（Bonferroni、FDR、Holm-Bonferroni）
- 谨慎解释不显著结果（缺乏证据≠证据缺乏）
- 预注册分析以避免p值操纵
- 对于"无差异"主张考虑等效性检验

### 2. 不恰当的统计检验

**常见问题：**
- 违反假设时使用参数检验（非正态数据、方差不齐）
- 对配对数据使用独立检验
- 对多组使用t检验而非方差分析事后检验
- 将有序数据当作连续数据处理
- 忽视重复测量结构
- 相关性适用时却使用回归

**如何识别：**
- 未提及假设检验
- 小样本量使用参数检验
- 多次配对t检验而非方差分析
- Likert量表用t检验分析
- 时间序列数据未考虑重复测量

**建议措施：**
- 明确检查假设（正态性检验、Q-Q图）
- 适当时使用非参数替代方法
- 方差分析后应用适当的多重比较校正
- 重复测量使用混合效应模型
- 有序结果考虑有序回归

### 3. 样本量和功效问题

**常见问题：**
- 无样本量依据或功效计算
- 效能不足的研究声称"无效果"
- 事后功效计算（信息量不足）
- 未预先规定停止规则
- 组间样本量不等且无理由

**如何识别：**
- 小样本量（典型设计每组n<30）
- 方法中未提及功效分析
- 有关于事后功效的表述
- 宽置信区间暗示不精确
- 大p值和小n声称"无效果"

**建议措施：**
- 基于预期效应量进行先验功效分析
- 报告达到的功效或精确度（置信区间宽度）
- 承认研究效能不足
- 解释时考虑效应量和置信区间
- 预注册样本量和停止规则

### 4. 缺失数据问题

**常见问题：**
- 完全案例分析无依据（列表删除）
- 未报告缺失的程度或模式
- 未检验假设数据完全随机缺失（MCAR）
- 不恰当的插补方法
- 未进行敏感性分析

**如何识别：**
- 不同分析的n值不同且无解释
- 未讨论缺失数据
- 参与者"从分析中排除"
- 使用简单均值插补
- 无比较完整数据与插补数据的敏感性分析

**建议措施：**
- 报告缺失的程度和模式
- 检验MCAR假设（Little检验）
- 使用适当的方法（多重插补、最大似然）
- 进行敏感性分析
- 考虑试验的意向性分析

### 5. 循环分析和双重使用数据

**常见问题：**
- 使用同一数据进行选择和推断
- 基于对比定义感兴趣区后在该区检验该对比
- 选择异常值后进行差异检验
- 事后亚组分析作为计划内分析呈现
- HARKing（结果已知后假设）

**如何识别：**
- 基于结果选择感兴趣区或特征
- 意外的事后亚组分析
- 事后分析未明确标注为探索性
- 无数据独立的验证
- 引言完美预测结果

**建议措施：**
- 使用独立数据集进行选择和检验
- 预注册分析和假设
- 明确区分验证性分析与探索性分析
- 使用交叉验证或保留数据集
- 校正选择偏倚

### 6. 伪重复

**常见问题：**
- 技术重复被视为生物学重复
- 来自同一受试者的多次测量被视为独立
- 聚类数据未考虑聚类
- 空间或时间数据中的非独立性

**如何识别：**
- n定义为测量数量而非生物学单位
- 来自同一动物的多个细胞被视为独立
- 未承认重复测量
- 未提及随机效应或聚类

**建议措施：**
- 将n定义为生物学重复（动物、患者、独立样本）
- 对嵌套或聚类数据使用混合效应模型
- 明确考虑重复测量
- 分析前平均技术重复
- 报告技术和生物学重复

## 实验设计问题

### 7. 缺乏适当的对照

**常见问题：**
- 缺少阴性对照
- 缺少验证的阳性对照
- 药物研究无溶剂对照
- 纵向研究无时间匹配对照
- 无批次对照

**如何识别：**
- 方法部分仅列出实验组
- 图表中未提及对照
- 基线或参考条件不清楚
- 跨批次比较无对照

**建议措施：**
- 包括阴性对照以评估特异性
- 包括阳性对照以验证方法
- 使用与实验处理匹配的溶剂对照
- 手术干预包括假手术对照
- 跨批次比较包括批次对照

### 8. 混杂变量

**常见问题：**
- 组间干预外的系统性差异
- 批次效应未控制或未校正
- 顺序实验中的顺序效应
- 未控制的时间效应
- 未设盲的实验者效应

**如何识别：**
- 组间在多个特征上有差异
- 样本按组在不同批次中处理
- 未随机化样本顺序
- 未提及设盲
- 组间基线特征不同

**建议措施：**
- 将实验单位随机分配到条件
- 按已知混杂因素进行区组化
- 随机化样本处理顺序
- 使用设盲最小化偏倚
- 如需要执行批次校正
- 报告并调整基线差异

### 9. 重复不足

**常见问题：**
- 单次实验无重复
- 技术重复误认为生物学重复
- 小n用"领域典型"辩护
- 关键发现无独立验证
- 挑选代表性例子

**如何识别：**
- 方法说明"实验进行一次"
- n=3无依据
- "显示代表性图像"
- 关键主张基于单次实验
- 独立数据集中无验证

**建议措施：**
- 进行独立生物学重复（通常≥3）
- 在独立队列中验证关键发现
- 报告所有重复，而非仅代表性例子
- 进行功效分析以说明样本量
- 显示单个数据点，而非仅汇总统计

## 可重复性问题

### 10. 方法学细节不足

**常见问题：**
- 方法描述不足以重复
- 关键试剂未说明（供应商、目录号）
- 未报告软件版本和参数
- 抗体未验证
- 细胞系身份未验证

**如何识别：**
- 模糊描述（"使用标准方案"）
- 无试剂来源信息
- 提及软件但无版本
- 无抗体验证信息
- 细胞系未验证

**建议措施：**
- 提供详细方案或引用特定方案
- 包括试剂供应商、目录号、批号
- 报告软件版本和所有参数
- 包括抗体验证（Western blot、特异性测试）
- 报告细胞系验证方法（STR分型）
- 提供方案（protocols.io、补充材料）

### 11. 数据和代码可用性

**常见问题：**
- 无数据可用性声明
- "数据根据要求提供"（通常未兑现）
- 计算分析无代码
- 自定义软件不可用
- 无清晰文档

**如何识别：**
- 缺少数据可用性声明
- 无仓库登录号
- 计算方法无代码
- 自定义流程无法访问
- 无README或文档

**建议措施：**
- 将原始数据存入适当仓库（GEO、SRA、Dryad、Zenodo）
- 在GitHub或类似平台分享分析代码
- 提供清晰的文档和README文件
- 包括requirements.txt或环境文件
- 提供自定义软件及安装说明
- 使用DOI进行永久数据引用

### 12. 缺乏方法验证

**常见问题：**
- 新方法未与金标准比较
- 检测未验证特异性、灵敏度、线性
- 无加标对照
- 未测试交叉反应
- 未建立检测限

**如何识别：**
- 呈现新检测方法无验证
- 未与现有方法比较
- 无阳性/阴性对照显示
- 特异性主张无证据
- 无标准曲线或对照

**建议措施：**
- 用已建立方法验证新方法
- 展示特异性（敲低/敲除对照）
- 展示线性和动态范围
- 包括阳性和阴性对照
- 报告检测和定量限
- 展示跨重复和操作人员的可重复性

## 解释问题

### 13. 结果陈述过度

**常见问题：**
- 对相关性数据使用因果语言
- 无机制证据却有机制主张
- 超出数据推断（物种、条件、人群）
- 未进行充分文献综述就声称"首次证明"
- 从有限样本过度概括

**如何识别：**
- 从观察数据得出"X导致Y"
- 未直接检验就提出机制
- 小鼠数据作为与人类相关呈现（无注意事项）
- 声称创新但缺少引用
- 窄样本做宽泛主张

**建议措施：**
- 使用适当语言（"相关"vs"导致"）
- 区分相关与因果
- 承认模型系统的局限性
- 提供充分的文献背景
- 对可推广性具体说明
- 将机制作为假设而非结论提出

### 14. 挑选和选择性报告

**常见问题：**
- 仅报告显著结果
- 显示可能不典型的"代表性"图像
- 无理由排除异常值
- 未报告阴性或矛盾发现
- 在不同统计方法间切换

**如何识别：**
- 报告的所有结果都显著
- "代表3次实验"但无量化
- 结果中提及数据排除但方法中未说明
- 补充数据与主要发现矛盾
- 多种分析方法只报告一种

**建议措施：**
- 报告所有计划分析（无论结果）
- 量化并显示跨重复的变异性
- 预先规定异常值排除标准
- 包括阴性结果
- 预注册分析计划
- 报告所有比较的效应量和置信区间

### 15. 忽视替代解释

**常见问题：**
- 呈现首选解释未考虑其他可能
- 未讨论就否定矛盾证据
- 未考虑脱靶效应
- 未承认混杂变量
- 局限性部分极少或缺失

**如何识别：**
- 将单一解释作为事实呈现
- 未引用或讨论先前矛盾发现
- 未考虑替代机制
- 未讨论局限性
- 假定特异性但无对照

**建议措施：**
- 讨论替代解释
- 处理文献中的矛盾发现
- 包括适当的特异性对照
- 充分承认并讨论局限性
- 考虑并检验替代假设

## 图表和数据呈现问题

### 16. 不恰当的数据可视化

**常见问题：**
- 对连续数据使用条形图（隐藏分布）
- 无误差线或未定义误差线
- 截断y轴夸大差异
- 双y轴造成误导性比较
- 有效数字过多
- 颜色不适合色盲

**如何识别：**
- 数据点少的条形图
- 不清楚误差线代表什么（SD、SEM、CI？）
- 比率/百分比数据的y轴不从零开始
- 左右y轴刻度不同
- 报告过高的精确度（p=0.04562）
- 红绿配色方案

**建议措施：**
- 用散点图/箱线图/小提琴图显示单个数据点
- 始终定义误差线（SD、SEM、95% CI）
- y轴从零开始或清晰标注断点
- 避免双y轴；用单独面板
- 报告适当的有效数字
- 使用色盲友好调色板（viridis、colorbrewer）
- 图例中包括样本量

### 17. 图像处理问题

**常见问题：**
- 过度调整对比度/亮度
- 未标注的拼接凝胶或图像
- 重复的图像或面板
- Western blot背景不均匀
- 选择性裁剪
- 过度处理的显微镜图像

**如何识别：**
- 可疑的模式或中断
- 很高对比度但无背景
- 不同面板中有相似特征
- 暗示拼接的直线
- 不一致的背景
- 暗示过度处理的细节丢失

**建议措施：**
- 对图像均匀应用调整
- 用分隔线标注拼接凝胶
- 在补充材料中显示完整未裁剪图像
- 如需要提供原始图像
- 遵循期刊图像完整性政策
- 使用适当的图像分析工具

## 研究设计问题

### 18. 假设和结果定义不清

**常见问题：**
- 未陈述清晰假设
- 未指定主要结果
- 多结果未校正
- 数据收集后改变结果
- 将探索性调查呈现为假设驱动

**如何识别：**
- 引言未陈述可检验的假设
- 多结果层级不清
- 结果中的结果与方法中的不匹配
- 探索性研究作为验证性研究呈现
- 许多检验但无多重检验校正

**建议措施：**
- 陈述清晰可检验的假设
- 预先指定主要和次要结果
- 可能时预注册研究
- 对多结果应用适当校正
- 明确区分探索性与验证性分析
- 报告所有预设结果

### 19. 基线不平衡和选择偏倚

**常见问题：**
- 组间基线不同
- 选择标准差异化应用
- 健康志愿者偏倚
- 生存者偏倚
- 观察研究中的指征偏倚

**如何识别：**
- 表1显示显著基线差异
- 组间入组标准不同
- 响应率<50%且无分析
- 分析仅包括完成者
- 组是自选而非随机

**建议措施：**
- 在表1中报告基线特征
- 使用随机化确保平衡
- 在分析中调整基线差异
- 报告响应率并比较响应者与非响应者
- 对观察数据考虑倾向评分匹配
- 使用意向性分析

### 20. 时间和批次效应

**常见问题：**
- 按条件分批处理样本
- 未考虑时间趋势
- 仪器随时间漂移
- 不同组用不同操作员
- 组间试剂批次变化

**如何识别：**
- 所有处理样本同一天处理
- 对照来自不同时期
- 未提及批次或时间效应
- 不同组技术人员不同
- 长期研究无时间分析

**建议措施：**
- 跨批次/时间随机化样本
- 在分析中包括批次作为协变量
- 执行批次校正（ComBat、limma）
- 跨批次包括质量控制样本
- 报告并检验时间趋势
- 平衡跨条件的操作员

## 报告问题

### 21. 不完整的统计报告

**常见问题：**
- 未报告检验统计量
- 缺失自由度
- 精确p值替换为不等式（p<0.05）
- 无置信区间
- 无效应量
- 未报告每组样本量

**如何识别：**
- 仅给出p值无检验统计量
- p值报告为p<0.05而非精确值
- 无不确定性度量
- 效应大小不清楚
- n报告总数但非每组

**建议措施：**
- 报告完整检验统计量（t、F、χ²等及df）
- 报告精确p值（除p<0.001外）
- 包括95%置信区间
- 报告效应量（Cohen's d、优势比、相关系数）
- 在每次分析中报告每组n
- 考虑CONSORT式流程图

### 22. 方法-结果不匹配

**常见问题：**
- 方法描述了未进行的分析
- 结果包括方法中未描述的分析
- 方法与结果中样本量不同
- 方法提及的对照未显示
- 统计方法与实际所做的不匹配

**如何识别：**
- 结果中有方法中无描述的分析
- 方法描述了结果中未出现的实验
- 部分数字不一致
- 提及对照但未显示
- 提及的软件与使用的不同

**建议措施：**
- 确保方法与结果完全一致
- 在方法中描述所有进行的分析
- 移除未进行实验的方法描述
- 验证所有数字一致
- 更新方法以匹配实际进行的分析

## 如何使用本参考

审稿时：
1. 系统阅读方法和结果
2. 检查每个类别的常见问题
3. 记录具体问题及证据
4. 提供改进的建设性建议
5. 区分主要问题（影响有效性）和次要问题（影响清晰度）
6. 优先考虑可重复性和透明度

这不是详尽清单，但涵盖最常遇到的问题。评估潜在问题时始终考虑具体背景和学科。
