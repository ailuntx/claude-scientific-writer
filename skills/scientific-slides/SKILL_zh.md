# 科学幻灯片

## 概览

科学演示是传播研究、分享成果以及与学术和专业受众交流的关键媒介。本技能涵盖从结构和内容开发到视觉设计和演讲准备的全面指导，帮助创建有效的科学演示。

**核心关注点**: 会议、研讨会、答辩及专业演讲中的口头报告。

**关键设计理念**: 科学演示应**视觉上引人入胜**且**有研究支撑**。不惜一切代价避免枯燥、文字堆砌的幻灯片。优秀的科学演示结合了：
- **引人注目的视觉效果**: 高质量图表、图片、示意图（而非仅仅要点列表）
- **研究背景**: 通过文献查询确立可信度的恰当引用
- **极简文字**: 要点仅作为提示，由你口头提供解释
- **专业设计**: 现代配色方案、强烈的视觉层次、充裕的留白
- **故事驱动**: 清晰的叙事弧线，而非单纯的数据堆砌

**请记住**: 枯燥的演示 = 被遗忘的科学。让幻灯片在视觉上令人难忘，同时通过恰当的引用来保持科学严谨性。

## 何时使用本技能

在以下情况下应使用本技能：
- 准备会议报告（5-20 分钟）
- 开发学术研讨会（45-60 分钟）
- 创建论文或学位答辩演示
- 设计基金申请陈述
- 准备期刊俱乐部演示
- 在机构或公司进行研究演讲
- 科学主题的教学或辅导演示

## 使用 Nano Banana Pro 生成幻灯片

**本技能使用 Nano Banana Pro AI 自动生成惊艳的演示幻灯片。**

根据输出格式的不同，有两种工作流程：

### 默认工作流程：PDF 幻灯片（推荐）

使用 Nano Banana Pro 将每张幻灯片生成为完整图片，然后组合成 PDF。这种方式能产生最视觉效果出色的结果。

**工作方式：**
1. **规划演示稿**: 为每张幻灯片制定详细计划（标题、要点、视觉元素）
2. **生成幻灯片**: 为每张幻灯片调用 Nano Banana Pro 创建完整幻灯片图片
3. **合并为 PDF**: 将幻灯片图片组合成单个 PDF 演示文稿

**步骤 1: 规划每张幻灯片**

在生成之前，为你的演示创建一个详细计划：

```markdown
# 演示计划：机器学习入门

## 幻灯片 1: 标题页
- 标题: "机器学习：从理论到实践"
- 副标题: "AI 大会 2025"
- 演讲者: 史密斯博士，XYZ 大学
- 视觉: 现代抽象神经网络背景

## 幻灯片 2: 引言
- 标题: "机器学习为何重要"
- 要点: 行业采用、突破性应用、未来潜力
- 视觉: 展示不同机器学习应用（医疗、金融、机器人）的图标

## 幻灯片 3: 核心概念
- 标题: "三种学习类型"
- 内容: 监督学习、无监督学习、强化学习
- 视觉: 每类展示示例的三部分图示

... (为所有幻灯片继续)
```

**步骤 2: 生成每张幻灯片**

使用 `generate_slide_image.py` 脚本创建每张幻灯片。

**关键：格式一致性协议**

为确保整个演示文稿格式统一：

1. **在演示开始时就定义格式目标**，并将其包含在每个提示中：
   - 配色方案（例如："深蓝背景、白色文字、金色强调色"）
   - 字体风格（例如："粗体无衬线标题、干净的正文字体"）
   - 视觉风格（例如："简约、专业、企业审美"）
   - 布局方式（例如："充裕留白、左对齐内容"）

2. **生成后续幻灯片时始终附加上一张幻灯片**，使用 `--attach`：
   - 允许 Nano Banana Pro 查看并匹配现有风格
   - 在整个演示稿中创建视觉连续性
   - 确保颜色、字体和设计语言一致

3. **默认作者为 "K-Dense"**，除非指定了其他名称

4. **在提示中直接包含引用**，用于引用研究结果的幻灯片：
   - 在提示文本中添加引用，使其出现在生成的幻灯片上
   - 使用格式："Include citation: (Author et al., Year)" 或 "Show reference: Author et al., Year"
   - 对于多个引用，在提示中全部列出
   - 引用应以小号字体显示在幻灯片底部或相关内容附近

5. **为结果幻灯片附加现有图表/数据**（对于数据驱动的演示至关重要）：
   - 创建关于结果的幻灯片时，**始终**检查以下位置是否有现有图表：
     - 工作目录（例如 `figures/`, `results/`, `plots/`, `images/`）
     - 用户提供的输入文件或目录
     - 与演示相关的任何数据可视化、图表或图形
   - 使用 `--attach` 包含这些图表，以便 Nano Banana Pro 可以将其融入：
     - 为结果幻灯片附加实际数据图表
     - 为方法幻灯片附加相关示意图
     - 为标题幻灯片附加徽标或机构图片
   - 附加数据图表时，在提示中描述所需内容：
     - "创建一张幻灯片，展示所附结果图表，并突出关键发现"
     - "围绕所附图表构建幻灯片，添加标题和解释数据的要点"
     - "将所附图表融入结果幻灯片，并附上解读"
   - **生成结果幻灯片之前**：列出工作目录中的文件以找到相关图表
   - 可以附加多个图表：`--attach fig1.png --attach fig2.png`

**示例：包含格式一致性、引用和图表附件**

```bash
# 标题幻灯片（第一张 - 奠定风格）
python scripts/generate_slide_image.py "标题幻灯片：'机器学习：从理论到实践'。副标题：'AI 大会 2025'。演讲者：K-Dense。格式目标：深蓝背景 (#1a237e)、白色文字、金色强调色 (#ffc107)、简约设计、无衬线字体、页边距充裕、无装饰元素。" -o slides/01_title.png

# 带引用的内容幻灯片（附加上一张以保持一致性）
python scripts/generate_slide_image.py "标题为 '机器学习为何重要' 的演示幻灯片。三个带简单图标的关键要点：1) 行业采用，2) 突破性应用，3) 未来潜力。引用：在底部小字显示：(LeCun et al., 2015; Goodfellow et al., 2016)。格式目标：完全匹配所附幻灯片风格 - 深蓝背景、白色文字、金色强调色、简约专业设计、无视觉杂乱。" -o slides/02_intro.png --attach slides/01_title.png

# 带多个引用的背景幻灯片
python scripts/generate_slide_image.py "标题为 '深度学习革命' 的演示幻灯片。关键里程碑：ImageNet 突破 (2012)、transformer 架构 (2017)、GPT 模型 (2018-至今)。引用：在底部显示参考文献：(Krizhevsky et al., 2012; Vaswani et al., 2017; Brown et al., 2020)。格式目标：完全匹配所附幻灯片风格 - 相同颜色、字体、极简设计。" -o slides/03_background.png --attach slides/02_intro.png

# 结果幻灯片 - 附加工作目录中的实际数据图表
# 首先，检查有哪些图表：ls figures/  或 ls results/
python scripts/generate_slide_image.py "标题为 '模型性能结果' 的演示幻灯片。创建一张展示所附准确率图表的幻灯片。要突出的关键发现：1) 达到 95% 准确率，2) 比基线高 12%，3) 跨测试集一致。引用：底部包含：(Our results, 2025)。格式目标：完全匹配所附幻灯片风格。" -o slides/04_results.png --attach slides/03_background.png --attach figures/accuracy_chart.png

# 结果幻灯片 - 多图对比
python scripts/generate_slide_image.py "标题为 '前后对比' 的演示幻灯片。使用所附的两张图表构建并排对比幻灯片。左侧：基线结果，右侧：我们的改进结果。添加简要标签解释改进。格式目标：完全匹配所附幻灯片风格。" -o slides/05_comparison.png --attach slides/04_results.png --attach figures/baseline.png --attach figures/improved.png

# 方法幻灯片 - 附加现有示意图
python scripts/generate_slide_image.py "标题为 '系统架构' 的演示幻灯片。展示所附架构示意图，并加上简要解释要点：1) 输入处理，2) 模型推理，3) 输出生成。格式目标：完全匹配所附幻灯片风格。" -o slides/06_architecture.png --attach slides/05_comparison.png --attach diagrams/system_architecture.png
```

**重要：创建结果幻灯片前，务必：**
1. 列出工作目录中的文件：`ls -la figures/` 或 `ls -la results/`
2. 检查用户提供的目录中是否有相关图表
3. 附加所有应出现在幻灯片上的相关图表
4. 描述 Nano Banana Pro 应如何整合所附图表

**提示模板：**

在每个提示中包含以下元素（根据需要自定义）：
```
[幻灯片内容描述]
引用：底部包含：(Author1 et al., Year; Author2 et al., Year)
格式目标：[背景颜色]、[文字颜色]、[强调色]、简约专业设计、无装饰元素、与所附幻灯片风格一致。
```

- `--api-key`: OpenRouter API 密钥（或设置环境变量 OPENROUTER_API_KEY）
- `-v, --verbose`: 详细输出

- "创建关于此数据的幻灯片" + 附加数据图表
- "制作带此徽标的标题幻灯片" + 附加徽标
- "将这些图表合并到一张幻灯片" + 附加多个图像
- "在一张幻灯片中解释此示意图" + 附加示意图

**环境设置：**

## 核心能力

### 1. 演示结构与组织

构建具有清晰叙事情节且适合不同场景的演示结构。详细指导请参阅 `references/presentation_structure.md`。

**通用故事弧线**:
1. **吸引注意**: 抓住注意力（30-60 秒）
2. **背景**: 确立重要性（演讲时间 5-10%）
3. **问题/差距**: 指出未知之处（5-10%）
4. **方法**: 解释你的解决方案（15-25%）
5. **结果**: 展示关键发现（40-50%）
6. **意义**: 讨论含义（15-20%）
7. **结尾**: 令人难忘的结论（1-2 分钟）

**特定演讲结构**:
- **会议报告（15 分钟）**: 聚焦 1-2 个关键发现，方法部分尽量精简
- **学术研讨会（45 分钟）**: 全面覆盖，详细方法，多个研究
- **论文答辩（60 分钟）**: 完整论文概述，涵盖所有研究
- **基金陈述（15 分钟）**: 强调重要性、可行性和影响力
- **期刊俱乐部（30 分钟）**: 对已发表工作的批判性分析

### 2. 幻灯片设计原则

创建专业、可读且无障碍的幻灯片，以促进理解。完整设计指南请参阅 `references/slide_design_principles.md`。

**反面模式：避免枯燥、文字堆砌的演示**

❌ **什么导致演示枯燥且容易被遗忘：**
- 文字墙（每张幻灯片超过 6 个要点）
- 小号字体（<24pt 正文）
- 仅黑字白底（无视觉趣味）
- 无图片或图形（仅要点）
- 无定制的通用模板
- 密集、段落式的要点
- 缺失研究背景（无引用）
- 所有幻灯片看起来都一样（重复）

✅ **什么使演示引人入胜且印象深刻：**
- **高质量视觉元素**占主导（图表、照片、示意图、图标）
- 大号、清晰的文字作为点缀（而非主要内容）
- 现代、有目的的配色方案（非默认主题）
- 充裕的留白（幻灯片有呼吸感）
- 有研究支撑的上下文（通过文献查询获得恰当引用）
- 幻灯片布局多样（不全是要点列表）
- 故事驱动且有视觉锚点的流程
- 专业、精致的外观

**核心设计原则**:

**视觉优先方法**（关键）：
- 从视觉元素开始（图表、图片、示意图），以文字为辅助
- 每张幻灯片都应有**强的**视觉元素（图表、图形、照片、示意图）
- 文字解释或补充视觉内容，而非替代它
- 思考："如何展示这一点，而不仅仅是讲述？"
- 目标：60-70% 视觉内容，30-40% 文字

**简约而有冲击力**：
- 每张幻灯片一个主旨
- **极少**文字（3-4 个要点，每个 4-6 词为佳）
- 充裕留白（幻灯片 40-50%）
- 清晰的视觉焦点
- 大胆、自信的设计选择

**字体排印增强参与感**：
- 无衬线字体（Arial, Calibri, Helvetica）
- **大号**字体：正文 24-28pt（而非不低于 18pt）
- 幻灯片标题 36-44pt（加粗）
- 高对比度（最低 4.5:1，建议 7:1）
- 用大小建立层次，而非仅用字重

**色彩增强冲击力**：
- **现代**配色方案（非默认蓝/灰）
- 考虑你的主题：生物技术？用鲜艳色彩。物理？用深色系。健康？用暖色调。
- 有限色板（共 3-5 种颜色）
- 高对比度组合
- 色盲安全（避免红绿组合）
- 有目的地使用颜色（而非装饰）

**布局增加视觉趣味**：
- 变化布局（不全是要点列表）
- 使用两栏布局（文字 + 图表）
- 关键结果使用全幻灯片图表
- 不对称构图（比居中更有趣）
- 重点的三分法
- 一致但不重复

### 3. 幻灯片中的数据可视化

为演示场景改编科学图表。详细指导请参阅 `references/data_visualization_slides.md`。

**与期刊图表的区别**：
- 简化，不复刻
- 更大字体（最低 18-24pt）
- 更少的分面（拆分到不同幻灯片）
- 直接标注（而非图例）
- 通过颜色和大小强调
- 对复杂数据采用渐进揭示

**可视化最佳实践**：
- **条形图**：比较离散类别
- **折线图**：趋势和轨迹
- **散点图**：关系和关联
- **热图**：矩阵数据和模式
- **网络图**：关系和连接

**需避免的常见错误**：
- 极小字体 (<18pt)
- 一张幻灯片太多面板
- 复杂图例
- 对比度不足
- 布局杂乱

### 4. 演讲类型专项指导

不同演示场景需要不同方法。各类型的全面指导请参阅 `references/talk_types_guide.md`。

**会议报告** (10-20 分钟)：
- 结构：简短引言 → 精简方法 → 关键结果 → 快速结论
- 焦点：仅 1-2 个主要发现
- 风格：引人入胜、快节奏、令人难忘
- 目标：引起兴趣、建立联系、获得邀请

**学术研讨会** (45-60 分钟)：
- 结构：详细方法的全面覆盖
- 焦点：多个发现、深度分析
- 风格：学术性、互动性、以讨论为导向
- 目标：展示专业知识、获取反馈、寻求合作

**论文答辩** (45-60 分钟)：
- 结构：完整论文概述，所有研究
- 焦点：展示掌握程度和独立思考
- 风格：正式、全面、为严格提问做好准备
- 目标：通过考核，为研究决策辩护

**基金陈述** (10-20 分钟)：
- 结构：问题 → 重要性 → 方法 → 可行性 → 影响
- 焦点：创新性、前期数据、团队资质
- 风格：有说服力、聚焦成果和影响
- 目标：获得资助、展示可行性

**期刊俱乐部** (20-45 分钟)：
- 结构：背景 → 方法 → 结果 → 批判性分析
- 焦点：理解和评析已发表工作
- 风格：教育性、批判性、促进讨论
- 目标：学习、批评、讨论意义

### 5. 实现方式选项

#### Nano Banana Pro PDF（默认 - 推荐）

**最适用于**：视觉震撼的幻灯片、快速创建、非专业受众

**这是默认且推荐的方法。** 使用 AI 将每张幻灯片生成为完整图片。

**工作流程**：
1. 计划每张幻灯片（标题、内容、视觉元素）
2. 使用 `generate_slide_image.py` 生成每张幻灯片
3. 使用 `slides_to_pdf.py` 组合为 PDF

```bash
# 生成幻灯片
python scripts/generate_slide_image.py "标题：引言..." -o slides/01.png
python scripts/generate_slide_image.py "标题：方法..." -o slides/02.png

# 组合为 PDF
python scripts/slides_to_pdf.py slides/*.png -o presentation.pdf
```

**优势**：
- 视觉上最令人印象深刻
- 创建快速（描述并生成）
- 无需设计技能
- 一致、专业的外观
- 非常适合普通受众

**最适用于**：
- 会议演讲
- 商务演示
- 一般科学演讲
- 提案陈述

#### 通过 PPTX 技能的 PowerPoint

**最适用于**：可编辑幻灯片、定制设计、基于模板的工作流程

**参考**: 完整文档见 `scientific-skills/pptx/SKILL.md`

使用 Nano Banana Pro 的 `--visual-only` 生成图片，然后用文本构建 PPTX。

**关键资源**：
- `assets/powerpoint_design_guide.md`: 完整的 PowerPoint 设计指南
- PPTX 技能的 `html2pptx.md`: 编程创建流程
- PPTX 技能脚本: `rearrange.py`, `inventory.py`, `replace.py`, `thumbnail.py`

**工作流程**：
1. 使用 `generate_slide_image.py --visual-only` 生成视觉元素
2. 设计 HTML 幻灯片（编程方式）或使用模板
3. 使用 html2pptx 或模板编辑创建演示文稿
4. 添加生成的图片和文本内容
5. 生成缩略图进行视觉验证
6. 基于视觉检查进行迭代

**优势**：
- 可编辑幻灯片（后续可修改文本）
- 复杂动画和过渡效果
- 交互元素
- 公司模板兼容性

#### LaTeX Beamer

**最适用于**：数学内容、一致格式、版本控制

**参考**: 完整文档见 `references/beamer_guide.md`

**可用模板**：
- `assets/beamer_template_conference.tex`: 15 分钟会议演讲
- `assets/beamer_template_seminar.tex`: 45 分钟学术研讨会
- `assets/beamer_template_defense.tex`: 学位论文答辩

**工作流程**：
1. 选择合适的模板
2. 定制主题和颜色
3. 添加内容（LaTeX 原生：公式、代码、算法）
4. 编译为 PDF
5. 转换为图片进行视觉验证

**优势**：
- 精美的数学和公式
- 一致、专业的外观
- 版本控制友好（纯文本）
- 非常适合算法和代码
- 可复现、可编程

### 6. 视觉审查与迭代

通过视觉检查实现迭代改进。完整工作流程请参阅 `references/visual_review_workflow.md`。

**视觉验证工作流程**：

**步骤 1: 生成 PDF**（如果还不是 PDF）
- PowerPoint：导出为 PDF
- Beamer：编译 LaTeX 源码

**步骤 2: 转换为图片**
```bash
# 使用 pdf_to_images 脚本
python scripts/pdf_to_images.py presentation.pdf review/slide --dpi 150

# 或使用 pptx 技能的缩略图工具
python scientific-skills/pptx/scripts/thumbnail.py presentation.pptx review/thumb
```

**步骤 3: 系统检查**

逐张检查幻灯片：
- **文字溢出**: 文字在边缘被截断
- **元素重叠**: 文字与图片或其他文字重叠
- **字号**: 文字太小 (<18pt)
- **对比度**: 文字与背景对比度不足
- **布局问题**: 未对齐、间距不佳
- **视觉质量**: 图像像素化、渲染不佳

**步骤 4: 记录问题**

创建问题日志：
```
幻灯片 # | 问题类型 | 描述 | 优先级
--------|-----------|-------------|----------
3       | 文字溢出 | 第4个要点超出文本框 | 高
7       | 重叠 | 图表与图例重叠 | 高
12      | 字号 | 坐标轴标签太小 | 中
```

**步骤 5: 应用修复**

对源文件进行修正：
- PowerPoint：编辑文本框、调整元素大小
- Beamer：调整 LaTeX 代码，重新编译

**步骤 6: 重新验证**

重复步骤 1-5，直到没有严重问题残留。

**停止标准**：
- 无文字溢出
- 无不恰当重叠
- 所有文字可读（等效 ≥18pt）
- 对比度充足（≥4.5:1）
- 外观专业

### 7. 时间与节奏

确保演示符合分配的时间。全面的时间指导请参阅 `assets/timing_guidelines.md`。

**一分钟一张幻灯片法则**：
- 一般准则：~1 张幻灯片/分钟
- 对复杂幻灯片调整（2-3 分钟）
- 对简单幻灯片调整（15-30 秒）

**时间分配**：
- 引言：15-20%
- 方法：15-20%
- 结果：40-50%（**最多时间**）
- 讨论：15-20%
- 结论：5%

**练习要求**：
- 5 分钟演讲：练习 5-7 次
- 15 分钟演讲：练习 3-5 次
- 45 分钟演讲：练习 3-4 次
- 答辩：练习 4-6 次

**时间检查点**：

对于 15 分钟演讲：
- 3-4 分钟：完成引言
- 7-8 分钟：结果部分进行到一半
- 12-13 分钟：开始结论

**应急策略**：
- 时间滞后：跳过备用幻灯片（提前准备）
- 时间超前：扩展示例，稍微放慢
- 无论如何不要跳过结论

## 演示开发流程

### 阶段 1: 计划（制作幻灯片前）

**定义背景**：
1. 什么类型的演讲？（会议、研讨会、答辩等）
2. 时长？（分钟数）
3. 听众是谁？（专家、普通、混合）
4. 场地是什么？（房间大小、影音设备、线上/线下）
5. 演讲后有什么？（问答、讨论、交流）

**研究与文献综述**（使用 research-lookup 技能）：
1. **搜索背景文献**: 找到 5-10 篇建立背景的关键论文
2. **识别知识空白**: 使用 research-lookup 找出未知点
3. **定位对比研究**: 找到方法或结果相似的论文
4. **收集支持性引用**: 收集支持你解释的论文
5. **建立参考文献列表**: 创建用于幻灯片的 .bib 文件或引用列表
6. **记下要引用的关键发现**: 记录要参考的具体结果

**制定内容大纲**：
1. 明确 1-3 个核心信息
2. 选择要展示的关键发现
3. 挑选必要的图表（15 分钟演讲通常 3-6 个）
4. 规划带有恰当地引用的叙事弧线
5. 按章节分配时间

**15 分钟演讲大纲示例**：
```
1. 标题 (30 秒)
2. 吸引注意：引人入胜的问题 (60 秒) [通过 research-lookup 引用 1-2 篇论文]
3. 背景 (90 秒) [引用 3-4 篇建立背景的关键论文]
4. 研究问题 (45 秒) [引用显示空白的论文]
5. 方法概述 (2 分钟)
6-8. 主要结果 1 (3 分钟, 3 张幻灯片)
9-10. 主要结果 2 (2 分钟, 2 张幻灯片)
11-12. 结果 3 或验证 (2 分钟, 2 张幻灯片)
13-14. 讨论与意义 (2 分钟) [与 2-3 项前期研究比较]
15. 结论 (45 秒)
16. 致谢 (15 秒)

注意：创建幻灯片前，使用 research-lookup 为背景（幻灯片 2-4）
和讨论（幻灯片 13-14）查找论文。
```

### 阶段 2: 设计与创建

**选择实现方式**：

**选项 A: PowerPoint（通过 PPTX 技能）**
1. 阅读 `assets/powerpoint_design_guide.md`
2. 阅读 `scientific-skills/pptx/SKILL.md`
3. 选择方法（编程或基于模板）
4. 创建具有一致设计的主幻灯片
5. 按大纲构建演示文稿

**选项 B: LaTeX Beamer**
1. 阅读 `references/beamer_guide.md`
2. 从 `assets/` 中选择适当模板
3. 定制主题和颜色
4. 用 LaTeX 撰写内容
5. 编译为 PDF

**设计考量**（使其视觉美观）：
- **选择现代配色方案**: 匹配你的主题（生物技术=鲜艳、物理=极简、健康=温暖）
  - 使用 pptx 技能的颜色板示例（蓝绿&珊瑚、大胆红、深紫&翡翠等）
  - 不仅仅是默认的蓝/灰主题
  - 3-5 种高对比度颜色
- **选择干净字体**: 无衬线、大号（正文 24pt+）
- **规划视觉元素**: 每张幻灯片有什么图片、示意图、图标？
- **创建多样布局**: 混合全图、两栏、文字叠加（不全是要点）
- **设计章节分隔页**: 带有醒目图形的视觉停顿
- **规划动画/构建**: 控制复杂幻灯片的信息流
- **增加视觉趣味**: 背景图片、色块、形状、图标

### 阶段 3: 内容开发

**填充幻灯片**（视觉优先策略）：
1. **从视觉元素开始**: 为每个关键点规划图表、图片、示意图
2. **广泛使用 research-lookup**: 找到 8-15 篇论文获取恰当引用
3. **首先创建视觉骨架**: 添加所有图表、图形、图片、示意图
4. **添加最少文字作为辅助**: 要点补充视觉，而非取代它们
5. **设计章节分隔页**: 带有图片或图形的视觉休息（而非纯文字）
6. **打磨标题/结尾**: 视觉上醒目，包含联系方式
7. **添加过渡/构建**: 控制信息流

**视觉内容要求**（让幻灯片引人入胜）：
- **图片**: 使用高质量照片、插画、概念图
- **图标**: 概念的可视化表示（非装饰）
- **示意图**: 流程图、原理图、过程图
- **图表**: 简化后的研究图表，标签大（18-24pt）
- **图形**: 清晰的数据可视化，信息明确
- **图形元素**: 视觉隐喻、概念插画
- **色块**: 用彩色形状组织内容
- 目标：每张幻灯片至少 1-2 个强视觉元素

**科学内容**（有研究支撑）：
- **引用**: 广泛使用 research-lookup 找到相关论文
  - 引言：引用 3-5 篇建立背景和指出的空白论文
  - 背景：可视化展示关键前期工作（而不仅仅是引用）
  - 讨论：引用 3-5 篇论文与你的结果进行比较
  - 使用作者-年份格式（Smith et al., 2023）增强可读性
  - 引用建立可信度和科学严谨性
- **图表**: 从论文简化，标签大（最低 18-24pt）
- **公式**: 大号、清晰，解释每个术语（谨慎使用）
- **表格**: 极简，突出关键对比（非数据堆砌）
- **代码/算法**: 使用语法高亮，保持简短

**文字指南**（少即是多）：
- 要点，绝非段落
- 每张幻灯片 3-4 个要点（最多仅在有必要时 6 个）
- 每个要点 4-6 个词（比 6×6 规则更少）
- 关键术语用粗体
- 文字是**辅助角色**，视觉是明星
- 使用构建控制节奏

### 阶段 4: 视觉验证

**生成图片**：
```bash
# 将 PDF 转换为图片
python scripts/pdf_to_images.py presentation.pdf review/slides

# 或创建缩略图网格
python scientific-skills/pptx/scripts/thumbnail.py presentation.pptx review/grid
```

**系统审查**：
1. 查看每张幻灯片图片
2. 对照问题清单检查
3. 记录问题及幻灯片编号
4. 测试远距离可读性（以 50% 尺寸查看）

**需修复的常见问题**：
- 文字超出边界
- 图表与文字重叠
- 字号太小
- 对比度差
- 未对齐

**迭代**：
1. 在源文件中修复识别的问题
2. 重新生成 PDF/演示文稿
3. 再次转换为图片
4. 再次检查
5. 重复直到清晰

### 阶段 5: 练习与优化

**练习计划**：
- 第1遍：草稿（可能会超时）
- 第2遍：流畅过渡
- 第3遍：精准计时
- 第4遍：最终打磨
- 第5遍及以上：保持（前一天、当天早上）

**练习内容**：
- 带计时器的完整演讲
- 困难解释的部分
- 章节间的过渡
- 开场和结尾（直到完美）
- 预期的提问

**基于练习的优化**：
- 若超时则删减幻灯片
- 若不清晰则扩展解释
- 调整措辞以增进清晰度
- 标记时间检查点
- 准备备用幻灯片

### 阶段 6: 最终准备

**技术检查**：
- [ ] 多份拷贝已保存（电脑、云端、U盘）
- [ ] 在演示电脑上可运行
- [ ] 适配器/线缆已备
- [ ] 备有 PDF 版本
- [ ] 已用投影仪测试（如可能）

**内容定稿**：
- [ ] 无拼写或错误

## 需避免的常见陷阱

### 内容错误

**枯燥无味的演示**（务必避免）：
- 问题：文字堆砌的幻灯片，无视觉趣味，缺少研究背景
- 迹象：全是要点、无图片、默认模板、无引用
- 解决方案：
  - 使用 research-lookup 找到 8-15 篇论文以获取可靠背景
  - 为每张幻灯片添加高质量视觉元素（图表、照片、示意图、图标）
  - 选择反映主题的现代配色方案
  - 变化幻灯片布局（不全是要点列表）
  - 用视觉讲故事，谨慎使用文字

**内容过多**：
- 问题：试图包含论文中的所有内容
- 解决方案：对于短演讲，聚焦 1-2 个关键发现，可视化展示

**文字过多**：
- 问题：幻灯片上是整段文字、密集要点、照本宣科
- 解决方案：3-4 个要点，每个 4-6 词，让视觉承载信息

**缺失研究背景**：
- 问题：无引用、无支持的声明、定位不明确
- 解决方案：使用 research-lookup 查找论文，引言中引用 3-5 篇，讨论中引用 3-5 篇

**叙事不佳**：
- 问题：主题间跳跃，无清晰故事线，无流畅感
- 解决方案：遵循故事弧线，使用视觉过渡，保持主线

**仓促掠过结果**：
- 问题：方法简短，结果简短，讨论很长
- 解决方案：花 40-50% 时间在结果上，可视化展示数据

### 设计错误

**通用、默认外观**：
- 问题：使用默认 PowerPoint/Beamer 主题无定制，看起来过时
- 解决方案：选择现代配色，定制字体/布局，增添视觉个性

**文字堆砌、视觉贫乏**：
- 问题：全是要点列表幻灯片，无图片或图形，视觉乏味
- 解决方案：为每张幻灯片添加图表、照片、示意图、图标，使其视觉有趣

**小号字体**：
- 问题：正文 <18pt，后排无法阅读，显得不专业
- 解决方案：正文 24-28pt（而非仅 18pt 最低），标题 36-44pt

**低对比度**：
- 问题：浅色文字在浅色背景上，可见性差，难以阅读
- 解决方案：高对比度（建议 7:1，而非仅 4.5:1 最低），用对比检查器测试

**杂乱幻灯片**：
- 问题：元素过多，无留白，令人窒息
- 解决方案：每张幻灯片一个主旨，40-50% 留白，间距充裕

**格式不一致**：
- 问题：不同幻灯片字体、颜色、布局不同，显得业余
- 解决方案：使用母版幻灯片，维护设计系统，保持专业一致性

**缺失视觉层次**：
- 问题：所有元素同样大小和颜色，无强调，焦点不清
- 解决方案：大小差异（标题大，正文中等），颜色强调，清晰焦点

### 时间错误

**不练习**：
- 问题：实际演示时才第一次过
- 解决方案：至少带计时器练习 3 次

**无时间检查点**：
- 问题：发现时间滞后时为时已晚
- 解决方案：设定 3-4 个检查点，全程监控

**超时**：
- 问题：极其不专业，挤占问答时间
- 解决方案：练习到精准时间，准备备用方案（可跳过幻灯片）

**跳过结论**：
- 问题：时间不够，仓促或跳过结尾
- 解决方案：任何情况下不跳过结论，改为裁剪前面内容

## 快速入门指南

### 15 分钟会议演讲（PDF 工作流程 - 推荐）

1. **研究与计划** (45 分钟):
   - **使用 research-lookup** 找到 8-12 篇相关引用论文
   - 建立参考文献列表（背景、对比研究）
   - 拟出内容提纲（引言 → 方法 → 2-3 个关键结果 → 结论）
   - **为每张幻灯片制定详细计划**（标题、要点、视觉元素）
   - 目标 15-18 张幻灯片

2. **使用 Nano Banana Pro 生成幻灯片** (1-2 小时):
   
   **重要：使用一致格式，附加前张幻灯片，并包含引用！**
   
   ```bash
   # 标题幻灯片（奠定风格 - 默认作者：K-Dense）
   python scripts/generate_slide_image.py "标题幻灯片：'你的研究标题'。会议名称，K-Dense。格式目标：[你的配色方案]，极简专业设计，无装饰元素，干净企业风格。" -o slides/01_title.png
   
   # 带引用的引言幻灯片（附加前张以保持一致性）
   python scripts/generate_slide_image.py "标题为 '这为何重要' 的幻灯片。三个带简单图标的关键点。引用：在底部包含：(Smith et al., 2023; Jones et al., 2024)。格式目标：完全匹配所附幻灯片风格。" -o slides/02_intro.png --attach slides/01_title.png
   
   # 继续每张幻灯片（始终附加前张，相关处包含引用）
   python scripts/generate_slide_image.py "标题为 '方法' 的幻灯片。关键方法要点。引用：(Based on Chen et al., 2022)。格式目标：完全匹配所附幻灯片风格。" -o slides/03_methods.png --attach slides/02_intro.png
   
   # 合并为 PDF
   python scripts/slides_to_pdf.py slides/*.png -o presentation.pdf
   ```

3. **审查与迭代** (30 分钟):
   - 打开 PDF 并审查每张幻灯片
   - 重新生成需要改进的幻灯片
   - 重新合并为 PDF

4. **练习** (2-3 小时):
   - 带计时器练习 3-5 次
   - 目标是 13-14 分钟（留出缓冲）
   - 录下自己，回看
   - **为提问做准备**（使用 research-lookup 预测问题）

5. **定稿** (30 分钟):
   - 如有需要，生成备用/附录幻灯片
   - 保存多份拷贝
   - 在演示电脑上测试

总时间：制作高质量 AI 生成的演示文稿约需 5-6 小时

## 总结：关键原则

1. **视觉优先设计**: 每张幻灯片都需要强的视觉元素（图表、图片、示意图）——避免纯文字幻灯片
2. **研究支撑**: 使用 research-lookup 找到 8-15 篇论文，引言中引用 3-5 篇，讨论中引用 3-5 篇
3. **现代审美**: 选择与主题匹配的当代配色方案，非默认主题
4. **极简文字**: 3-4 个要点，每个 4-6 词（24-28pt 字体），让视觉讲故事
5. **结构**: 遵循故事弧线，花 40-50% 时间在结果上
6. **高对比度**: 建议 7:1 以实现专业外观
7. **多样布局**: 混合全图、两栏、视觉叠加（不全是要点）
8. **时间**: 练习 3-5 次，~1 张幻灯片/分钟，任何情况下不跳过结论
9. **验证**: 视觉审查流程以发现溢出和重叠
10. **留白**: 幻灯片 40-50% 留空以提供视觉呼吸感

**记住**： 
- **枯燥 = 被遗忘**：干涩、文字堆砌的幻灯片无法有效传播你的科学
- **视觉 + 研究 = 影响力**：结合引人注目的视觉效果和有研究支撑的上下文
- **你是演示，幻灯片是视觉辅助**：它们应增强而非取代你的演讲
