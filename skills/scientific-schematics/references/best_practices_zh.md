# 科学图表制作最佳实践

## 概述

本指南提供发表标准、无障碍指南和最佳实践，帮助创建符合期刊要求的高质量科学图表，确保对所有读者都能有效传达信息。

## 发表标准

### 1. 文件格式要求

**矢量格式（推荐）**
- **PDF**：通用支持，保持质量，适用于LaTeX
  - 适用于：线条图、流程图、框图、电路图
  - 优点：可缩放、文件体积小、嵌入字体
  - LaTeX工作流程的标准格式

- **EPS（封装PostScript）**：传统格式，仍被接受
  - 适用于：较老的出版系统
  - 与大多数期刊兼容
  - 可以从PDF转换

- **SVG（可缩放矢量图形）**：网页友好，逐渐被接受
  - 适用于：在线出版物、交互式图表
  - 可在矢量图形软件中编辑
  - 并非所有期刊都接受SVG

**光栅格式（必要时使用）**
- **TIFF**：光栅图形的专业标准
  - 适用于：显微镜图像、与图表结合的照片
  - 最终打印尺寸下最低300 DPI
  - 无损压缩（LZW）

- **PNG**：网页友好，无损压缩
  - 适用于：在线补充材料、演示文稿
  - 打印最低300 DPI
  - 支持透明背景

**禁止使用**
- **JPEG**：有损压缩会在图表中产生伪影
- **GIF**：颜色有限，不适合科学图表
- **BMP**：未压缩，文件过大

### 2. 分辨率要求

**矢量图形**
- 无限分辨率（可缩放）
- **推荐**：尽可能使用矢量格式

**光栅图形（无法使用矢量时）**
- **出版质量**：300-600 DPI
- **线条图**：600-1200 DPI
- **网页/屏幕**：150 DPI可接受
- **禁止**：打印低于300 DPI

**计算DPI**
```
DPI = 像素 / （最终尺寸的英寸数）

示例：
图像尺寸：2400 × 1800 像素
最终打印尺寸：8 × 6 英寸
DPI = 2400 / 8 = 300 ✓ （可接受）
```

### 3. 尺寸和规格

**期刊特定栏宽**
- **Nature**：单栏89毫米（3.5英寸），双栏183毫米（7.2英寸）
- **Science**：单栏55毫米（2.17英寸），双栏120毫米（4.72英寸）
- **Cell**：单栏85毫米（3.35英寸），双栏178毫米（7英寸）
- **PLOS**：单栏83毫米（3.27英寸），双栏173毫米（6.83英寸）
- **IEEE**：单栏3.5英寸，双栏7.16英寸

**最佳实践**
- 按最终打印尺寸设计（避免缩放）
- 使用期刊提供的模板
- 预留裁切边距
- 提交前在最终尺寸下测试显示效果

### 4. 排版标准

**字体选择**
- **推荐**：Arial、Helvetica、Calibri（无衬线体）
- **可接受**：Times New Roman（衬线体）用于数学密集内容
- **避免**：装饰字体、手写字体、可能无法嵌入的系统字体

**字号（最终打印尺寸下）**
- **最小值**：6-7磅（视期刊而定）
- **坐标轴标签**：8-9磅
- **图表标签**：10-12磅
- **面板标签（A、B、C）**：10-14磅，加粗
- **正文**：应与手稿正文匹配

**文字清晰度**
- 使用句首大写："Time (seconds)"而非"TIME (SECONDS)"
- 单位放在括号内："Temperature (°C)"
- 在图注中拼写全缩写
- 尽可能避免旋转文字（Y轴标签除外）
- **图表内不要包含图号** - 不要加入"Figure 1:"、"Fig. 1"等（这些由LaTeX/文档添加）

### 5. 线宽和描边

**推荐线宽**
- **图表轮廓**：0.5-1.0磅
- **连接线/箭头**：1.0-2.0磅
- **强调元素**：2.0-3.0磅
- **最小可视**：最终尺寸下0.25磅

**一致性**
- 对相似元素使用相同线宽
- 通过线宽变化显示层级结构
- 避免极细线条（过细可能无法可靠打印）

## 无障碍和色盲友好

### 1. 色盲安全配色方案

**Okabe-Ito配色方案（推荐）**
各类色盲患者都能最清晰区分：

```latex
% RGB值
Orange:     #E69F00 (230, 159,   0)
Sky Blue:   #56B4E9 ( 86, 180, 233)
Green:      #009E73 (  0, 158, 115)
Yellow:     #F0E442 (240, 228,  66)
Blue:       #0072B2 (  0, 114, 178)
Vermillion: #D55E00 (213,  94,   0)
Purple:     #CC79A7 (204, 121, 167)
Black:      #000000 (  0,   0,   0)
```

**替代方案：ColorBrewer配色方案**
- **定性**：Set2、Paired、Dark2
- **渐变**：Blues、Greens、Oranges（避免红绿同时使用）
- **双极**：RdBu（红-蓝）、PuOr（紫-橙）

**避免同时使用的颜色**
- 红绿组合（8%的男性无法区分）
- 蓝紫组合
- 黄浅绿组合

### 2. 冗余编码

不要仅依赖颜色。使用多个视觉通道：

**形状 + 颜色**
```
圆形 + 蓝色   = 条件A
方形 + 橙色   = 条件B
三角形 + 绿色 = 条件C
```

**线型 + 颜色**
```
实线 + 蓝色   = 处理1
虚线 + 橙色   = 处理2
点线 + 绿色   = 对照
```

**填充图案 + 颜色**
```
实心填充 + 蓝色   = 组A
对角线 + 橙色     = 组B
交叉线 + 绿色     = 组C
```

### 3. 灰度兼容性

**测试要求**：所有图表必须在灰度下可解读

**策略**
- 使用不同深浅（浅、中、深）
- 为填充区域添加图案或纹理
- 变化线型（实线、虚线、点线）
- 直接在元素上添加标签
- 包含文字注释

**灰度测试**
```bash
# 转换为灰度进行测试
convert diagram.pdf -colorspace gray diagram_gray.pdf
```

### 4. 对比度要求

**最低对比度比率（WCAG指南）**
- **正常文本**：4.5:1
- **大文本**（≥18磅）：3:1
- **图形元素**：3:1

**高对比度实践**
- 深色文本配浅色背景（或反之）
- 避免低对比度颜色对（白色上的黄色、白色上的浅灰色）
- 对关键文本使用黑色或深灰色
- 深色背景上的白色文本需要更大的字号

### 5. 替代文本和描述

**图注必须包含**
- 图表类型描述
- 所有缩写拼写全称
- 符号和颜色说明
- 样本量（n）相关时
- 统计注释说明
- 如适用，参考详细方法

**图注示例**
"遵循CONSORT指南的参与者流程图。矩形代表研究阶段，显示参与者数量（n）。排除标准列于每个筛选阶段。最终分析纳入两个组的n=350名参与者。"

## 设计原则

### 1. 简洁清晰

**图表的奥卡姆剃刀**
- 删除每个不增加信息的元素
- 简化复杂关系
- 将复杂图表拆分为多个面板
- 相关图表使用一致布局

**视觉层级**
- 最重要元素：最大、最深色、居中
- 辅助元素：较小、较浅色、周边
- 注释：最少，仅清晰标签

### 2. 一致性

**同一图表内**
- 相同形状/颜色代表相同概念
- 相同关系的箭头样式一致
- 间距和对齐均匀
- 相似元素的字号匹配

**论文各图表间**
- 复用配色方案
- 保持节点样式一致
- 使用相同符号系统
- 应用相同布局原则

### 3. 专业外观

**对齐**
- 使用网格放置节点
- 水平或垂直对齐节点
- 元素间距均匀
- 标签在形状内居中

**留白**
- 不要使图表过于拥挤
- 元素周围留出呼吸空间
- 用留白对相关项目分组
- 整个图表周围留边距

**精修**
- 无锯齿线条或错位元素
- 曲线平滑、角度精确
- 连接点干净
- 无文字重叠

## 常见问题和解决方案

### 问题1：图表过于复杂

**问题**：一个图表中信息过多
**解决方案**：
- 拆分为多个面板（A、B、C）
- 创建总览图+详细图
- 将细节移至补充图
- 使用层级 presentation

### 问题2：样式不一致

**问题**：不同图表中相同元素样式不同
**解决方案**：
- 创建并使用样式模板
- 整个论文使用相同配色方案
- 记录您的样式选择

### 问题3：标签位置不佳

**问题**：标签与元素重叠或难以阅读
**解决方案**：
- 尽可能将标签放在形状外部
- 对远距离标签使用引出线
- 仅在必要时旋转文字
- 确保与背景有足够对比度

### 问题4：文字过小

**问题**：最终打印尺寸下文字过小无法阅读
**解决方案**：
- 从一开始就按最终尺寸设计
- 在最终尺寸下测试打印
- 最小7-8磅字体
- 空间有限时简化标签

### 问题5：箭头不明确

**问题**：箭头代表什么或指向哪里不清楚
**解决方案**：
- 对不同含义使用不同箭头样式
- 在箭头上添加标签
- 为箭头类型提供图例
- 使用锚点精确连接

### 问题6：颜色滥用

**问题**：颜色过多，造成混淆或无法访问
**解决方案**：
- 最多限制3-5种颜色
- 有目的性地使用颜色（分类、强调）
- 使用色盲安全配色方案
- 提供冗余编码

## 期刊特定指南

### Nature

**图表要求**
- **尺寸**：89毫米（单栏）或183毫米（双栏）
- **格式**：PDF、EPS或高分辨率TIFF
- **字体**：推荐无衬线体
- **文件大小**：每个文件<10 MB
- **分辨率**：光栅图像最低300 DPI

**样式说明**
- 面板标签：小写加粗（a、b、c）
- 简洁、清晰设计
- 最少颜色
- 图注清晰

### Science

**图表要求**
- **尺寸**：55毫米（单栏）或120毫米（双栏）
- **格式**：PDF、EPS、TIFF或JPEG（高质量）
- **分辨率**：照片300 DPI，线条图600 DPI
- **文件大小**：<10 MB
- **字体**：最小6-7磅

**样式说明**
- 面板标签：大写加粗（A、B、C）
- 高对比度
- 小尺寸下可读

### Cell

**图表要求**
- **尺寸**：85毫米（单栏）或178毫米（双栏）
- **格式**：推荐PDF，TIFF、EPS可接受
- **分辨率**：最低300 DPI
- **字体**：标签8-10磅
- **线宽**：最小0.5磅

**样式说明**
- 简洁、专业
- 彩色或灰度
- 面板标签大写（A、B、C）

### IEEE

**图表要求**
- **尺寸**：3.5英寸（单栏）或7.16英寸（双栏）
- **格式**：PDF、EPS（推荐矢量）
- **分辨率**：线条图600 DPI，半调图像300 DPI
- **字体**：最小8-10磅
- **颜色**：印刷为灰度，数字版为彩色

**样式说明**
- 遵循IEEE图形手册
- 电路标准符号
- 技术精确
- 坐标轴标签清晰

## 软件特定导出设置

### AI生成的图像

AI生成的图表导出为PNG图像，可以使用以下方式包含在LaTeX文档中：

```latex
\includegraphics[width=\textwidth]{diagram.png}
```

### Python（Matplotlib）导出

```python
import matplotlib.pyplot as plt

# 设置出版质量
plt.rcParams['font.family'] = 'sans-serif'
plt.rcParams['font.sans-serif'] = ['Arial']
plt.rcParams['font.size'] = 8
plt.rcParams['pdf.fonttype'] = 42  # PDF中的TrueType字体

# 使用正确的DPI和裁切保存
fig.savefig('diagram.pdf', dpi=300, bbox_inches='tight', 
            pad_inches=0.1, transparent=False)
fig.savefig('diagram.png', dpi=300, bbox_inches='tight')
```

### Schemdraw导出

```python
import schemdraw

d = schemdraw.Drawing()
# ... 构建电路 ...

# 导出
d.save('circuit.svg')  # 矢量
d.save('circuit.pdf')  # 矢量
d.save('circuit.png', dpi=300)  # 光栅
```

### Inkscape命令行

```bash
# PDF转高分辨率PNG
inkscape diagram.pdf --export-png=diagram.png --export-dpi=300

# SVG转PDF
inkscape diagram.svg --export-pdf=diagram.pdf
```

## 版本控制最佳实践

**保留源文件**
- 保存原始.tex、.py或.svg文件
- 使用带版本的描述性文件名
- 记录配色方案和样式选择
- 包含README说明如何重新生成

**目录结构**
```
figures/
├── source/          # 可编辑的源文件
│   ├── diagram1.tex
│   ├── circuit.py
│   └── pathway.svg
├── generated/       # 自动生成的输出
│   ├── diagram1.pdf
│   ├── circuit.pdf
│   └── pathway.pdf
└── final/          # 最终提交版本
    ├── figure1.pdf
    └── figure2.pdf
```

**Git跟踪**
- 跟踪源文件（.tex、.py）
- 考虑将生成的PDF放入.gitignore（大型文件）
- 使用releases/tags标记提交版本
- 在README中记录生成过程

## 测试和验证

### 提交前测试

**视觉测试**
1. **打印测试**：按最终尺寸打印，检查可读性
2. **灰度测试**：转换为灰度，验证可理解性
3. **缩放测试**：在400%和25%下查看，检查可缩放性
4. **屏幕测试**：在不同设备上查看（手机、平板、电脑）

**技术测试**
1. **字体嵌入**：检查PDF属性
2. **分辨率检查**：验证DPI是否符合要求
3. **文件大小**：确保在期刊限制内
4. **格式合规**：验证接受的格式

**无障碍测试**
1. **色盲模拟**：使用Color Oracle等工具
2. **对比度检查**：WCAG对比度工具
3. **屏幕阅读器**：测试alt文本（网页图表）

### 测试工具

**色盲模拟**
- Color Oracle（免费，跨平台）
- Coblis（色盲模拟器）
- Photoshop/GIMP色盲预览模式

**PDF检查**
```bash
# 检查PDF属性
pdfinfo diagram.pdf

# 检查字体
pdffonts diagram.pdf

# 检查图像分辨率
identify -verbose diagram.pdf
```

**对比度检查**
- WebAIM对比度检查器：https://webaim.org/resources/contrastchecker/
- Colorable：https://colorable.jxnblk.com/

## 总结：黄金法则

1. **优先矢量**：尽可能使用矢量格式
2. **按最终尺寸设计**：创建后避免缩放
3. **色盲安全配色方案**：使用Okabe-Ito或类似方案
4. **灰度测试**：图表必须在无颜色情况下可用
5. **最小7-8磅文字**：最终打印尺寸下
6. **样式一致**：论文中所有图表保持一致
7. **保持简洁**：删除不必要的元素
8. **高对比度**：确保可读性
9. **对齐元素**：专业外观很重要
10. **完整图注**：解释一切
