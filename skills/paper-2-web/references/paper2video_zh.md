# Paper2Video：演示视频生成

## 概述

Paper2Video 从 LaTeX 源文件生成演示视频，将学术论文转化为引人入胜的视频演示。该系统通过多个专业模块处理论文，创建包含幻灯片、解说和可选主持人视频的专业演示视频。

## 核心组件

### 1. 幻灯片生成模块
- 从论文结构中提取关键内容
- 创建视觉吸引力强的演示幻灯片
- 按逻辑流程组织内容
- 包含图表、表格和公式
- 优化文本密度以提高可读性

### 2. 字幕生成模块
- 生成自然的演示文稿脚本
- 文字与幻灯片切换同步
- 创建演讲者备注和时间信息
- 支持多种语言
- 优化语音合成效果

### 3. 语音合成模块
- 将字幕转换为自然语音
- 支持多种声音和口音
- 控制语速和重音
- 生成视频音频轨道
- 处理专业术语

### 4. 光标移动模块
- 模拟演讲者光标移动
- 突出幻灯片上的关键点
- 引导观众注意力
- 创建自然的演示流程
- 与解说同步

### 5. 主持人视频生成（可选）
- 使用 Hallo2 生成逼真的演讲者视频
- 与生成的音频口型同步
- 需要参考图片或视频
- GPU 密集型（最低 NVIDIA A6000 48GB）
- 创建引人入胜的主持人形象

## 使用方法

### 基本视频生成（无主持人）

```bash
python pipeline_light.py \
  --model_name_t gpt-4.1 \
  --model_name_v gpt-4.1 \
  --result_dir /path/to/output \
  --paper_latex_root /path/to/paper
```

### 完整视频生成（含主持人）

```bash
python pipeline_all.py \
  --input-dir "path/to/papers" \
  --output-dir "path/to/output" \
  --model-choice 1 \
  --enable-talking-head
```

### 参数

**模型配置：**
- `--model_name_t`：文本/字幕生成模型（默认：gpt-4.1）
- `--model_name_v`：视觉/幻灯片生成模型（默认：gpt-4.1）
- `--model-choice`：预设模型配置（1=GPT-4，2=GPT-4.1）

**输入/输出：**
- `--paper_latex_root`：LaTeX 论文源的根目录
- `--result_dir` 或 `--output-dir`：生成视频的输出目录
- `--input-dir`：包含多篇论文待处理的目录

**视频选项：**
- `--enable-talking-head`：启用主持人视频生成（需要 GPU）
- `--video-duration`：目标视频时长（秒）（默认：自动计算）
- `--slides-per-minute`：控制演示节奏（默认：2-3）
- `--voice`：语音合成的语音选择

**质量设置：**
- `--video-resolution`：输出分辨率（默认：1920x1080）
- `--video-fps`：帧率（默认：30）
- `--audio-quality`：音频比特率（默认：192kbps）

## 输入要求

### LaTeX 源文件结构
```
paper_directory/
├── main.tex              # 主论文文件
├── sections/             # 章节文件（如果分开）
│   ├── introduction.tex
│   ├── methods.tex
│   └── results.tex
├── figures/              # 图片文件
│   ├── fig1.pdf
│   ├── fig2.png
│   └── ...
├── tables/               # 表格文件
└── bibliography.bib      # 参考文献
```

### 必需元素
- 可编译的有效 LaTeX 源文件
- 正确的章节结构（摘要、引言、方法、结果、结论）
- 高质量的图片（优先使用矢量格式）
- 完整的参考文献

### 可选元素
- 主持人头像图片用于生成主持人视频
- 自定义幻灯片模板
- 背景音乐或音效
- 机构品牌素材

## 输出结构

```
output/paper_name/video/
├── final_video.mp4           # 完整的演示视频
├── slides/                   # 生成的幻灯片图片
│   ├── slide_001.png
│   ├── slide_002.png
│   └── ...
├── audio/                    # 音频组件
│   ├── narration.mp3         # 语音合成输出
│   └── background.mp3        # 可选的背景音频
├── subtitles/                # 字幕文件
│   ├── subtitles.srt         # 标准字幕格式
│   └── subtitles.vtt         # WebVTT 格式
├── script/                   # 演示脚本
│   ├── full_script.txt       # 完整解说文本
│   └── slide_notes.json      # 幻灯片逐张备注
└── metadata/                 # 视频元数据
    ├── timings.json          # 幻灯片时间信息
    └── video_info.json       # 视频属性
```

## 视频生成流程

### 阶段一：内容分析
1. 解析 LaTeX 源文件结构
2. 提取关键概念和发现
3. 识别重要的图表和公式
4. 确定逻辑演示流程

### 阶段二：幻灯片创建
1. 根据内容设计幻灯片布局
2. 将内容分配到适当数量的幻灯片
3. 融入图表和视觉元素
4. 应用一致的样式和品牌标识

### 阶段三：脚本生成
1. 撰写自然的演示解说词
2. 将脚本章节与幻灯片时间对应
3. 添加转场和重音
4. 优化语音合成效果

### 阶段四：音频制作
1. 从脚本生成语音
2. 添加重音和节奏
3. 包含幻灯片切换的停顿
4. 与可选背景音频混合

### 阶段五：视频合成
1. 将幻灯片与时间信息组合
2. 同步音频轨道
3. 添加光标移动和高亮效果
4. 生成主持人视频（如启用）
5. 渲染最终视频文件

## 最佳实践

### 输入准备
1. **清理 LaTeX 源文件**：移除不必要的注释和冗余内容
2. **高质量图片**：尽可能使用矢量格式
3. **清晰结构**：组织良好的章节和子章节
4. **完整内容**：包含所有必要的文件和参考文献

### 模型选择
- **文本生成 (model_name_t)**：GPT-4.1 获得最佳脚本质量
- **视觉生成 (model_name_v)**：GPT-4.1 获得最佳幻灯片设计
- 对于更快处理且可接受质量：GPT-3.5-turbo

### 视频优化
1. **目标时长**：会议演讲 10-15 分钟，详细演示 30-45 分钟
2. **节奏**：技术内容每分钟 2-3 张幻灯片
3. **分辨率**：标准 1920x1080，高质量 3840x2160
4. **音频**：最低 192kbps 以获得清晰语音

### 质量检查
最终确定前：
1. 观看完整视频检查内容准确性
2. 检查音频与幻灯片的同步
3. 验证图片质量和可读性
4. 测试字幕准确性和时间
5. 检查光标移动的自然流畅性

## 性能注意事项

### 处理时间
- **无主持人**：每篇论文 10-30 分钟（取决于长度）
- **有主持人**：每篇论文 30-120 分钟
- **因素**：论文长度、图片数量、模型速度、GPU 可用性

### 资源要求
- **CPU**：建议多核用于并行处理
- **RAM**：最低 16GB，大型论文 32GB
- **GPU**：标准可选，主持人需要（A6000 48GB）
- **存储**：每个视频 1-5GB，取决于长度和质量

## 故障排除

### 常见问题

**1. LaTeX 解析错误**
- 确保 LaTeX 源文件成功编译
- 检查特殊包或自定义命令
- 验证所有引用的文件都存在

**2. 语音合成问题**
- 检查音频质量设置
- 验证文本格式正确
- 用不同的语音选项测试

**3. 视频渲染失败**
- 检查可用磁盘空间
- 验证所有依赖项已安装
- 查看错误日志获取具体问题

**4. 主持人视频生成错误**
- 确认 GPU 内存（需要 48GB）
- 检查 CUDA 驱动是否最新
- 验证参考图片质量和格式
