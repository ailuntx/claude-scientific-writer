# Paper2Web：学术主页生成工具

## 概述

Paper2Web 将学术论文转换为可交互、可探索的学术主页。与传统方法（直接生成、模板化或HTML转换）不同，Paper2Web 通过迭代优化流程创建具有布局感知的交互式网站。

## 核心功能

### 1. 布局感知生成
- 分析论文结构和内容组织
- 创建响应式、多板块布局
- 根据论文类型调整设计（研究文章、综述、预印本等）

### 2. 交互元素
- 可展开板块显示详细内容
- 交互式图表
- 嵌入的引用和参考文献
- 便于浏览的导航菜单
- 移动端响应式设计

### 3. 内容优化
系统采用迭代流程：
1. 初始内容提取和结构化
2. 布局生成与视觉层次
3. 交互元素集成
4. 美学优化
5. 质量评估和验证

## 使用方法

### 基本网站生成

```bash
python pipeline_all.py \
  --input-dir "path/to/papers" \
  --output-dir "path/to/output" \
  --model-choice 1
```

### 参数

- `--input-dir`：包含论文文件的目录（PDF或LaTeX）
- `--output-dir`：生成网站文件的输出目录
- `--model-choice`：LLM模型选择（1=GPT-4, 2=GPT-4.1）
- `--enable-logo-search`：使用Google Search API查找机构Logo（可选）

### 输入格式要求

**支持的输入格式：**
1. **LaTeX源文件**（首选，效果最佳）
   - 主文件：`main.tex`
   - 包含所有引用的图表、表格和参考文献文件
   - 每个论文组织在一个目录中

2. **PDF文件**
   - 高质量、可选择文本的PDF
   - 内嵌图形应为高分辨率
   - 正确的章节标题和结构

**目录结构：**
```
input/
└── paper_name/
    ├── main.tex           # LaTeX源文件
    ├── bibliography.bib   # 参考文献
    ├── figures/           # 图片文件
    │   ├── fig1.png
    │   └── fig2.pdf
    └── tables/            # 表格文件
```

## 输出结构

生成的网站包括：

```
output/paper_name/website/
├── index.html          # 主网页
├── styles.css          # 样式
├── script.js          # 交互功能
├── assets/             # 图片和媒体
│   ├── figures/
│   └── logos/
└── data/               # 结构化数据（可选）
```

## 自定义选项

### 视觉设计
生成的网站自动包含：
- 基于论文内容的专业配色方案
- 为可读性优化的字体排印
- 一致的间距和视觉层次
- 深色模式支持（可选）

### 内容板块
标准板块包括：
- 摘要
- 主要发现/贡献
- 方法概述
- 结果和可视化
- 讨论和意义
- 参考文献和引用
- 作者信息和单位

根据论文内容自动添加的额外板块：
- 代码仓库
- 数据集链接
- 补充材料
- 相关出版物

## 质量评估

Paper2Web 内置评估功能：

### 美学指标
- 布局平衡和间距
- 色彩和谐
- 字体排印一致性
- 视觉层次效果

### 信息性指标
- 内容完整性
- 主要发现清晰度
- 方法解释充分性
- 结果呈现质量

### 技术指标
- 页面加载时间
- 移动端响应能力
- 浏览器兼容性
- 无障碍合规性

## 高级功能

### Logo发现
启用Google Search API时：
- 自动查找机构Logo
- 匹配作者单位
- 下载并优化Logo图片
- 集成到网站头部

### 引用集成
- 交互式参考文献列表
- 引用悬停预览
- 链接到DOI和外部来源
- 引用计数跟踪（如可用）

### 图表增强
- 高分辨率图表渲染
- 缩放和平移功能
- 说明文字和描述集成
- 多面板图表导航

## 最佳实践

### 输入准备
1. **尽可能使用LaTeX**：提供最佳的结构提取
2. **包含所有素材**：图表、表格和参考文献文件
3. **清晰的格式**：移除编译产物和临时文件
4. **高质量图片**：尽可能使用矢量格式（PDF、SVG）

### 模型选择
- **GPT-4**：质量和成本的最佳平衡
- **GPT-4.1**：最新功能，成本较高
- **GPT-3.5-turbo**：处理更快，适合简单论文

### 输出优化
1. 检查生成内容的准确性
2. 确认所有图表正确渲染
3. 测试交互元素功能
4. 验证移动端响应能力
5. 检查外部链接有效性

## 局限性

- 复杂的数学公式可能需要手动检查
- PDF中的多栏布局可能影响提取质量
- 大型论文（>50页）可能需要更长的处理时间
- 某些特殊类型的图表可能需要手动调整

## 与其他组件的集成

Paper2Web可与以下工具结合使用：
- **Paper2Video**：为网站生成配套视频
- **Paper2Poster**：创建匹配的海报设计
- **AutoPR**：生成链接到网站的宣传内容
