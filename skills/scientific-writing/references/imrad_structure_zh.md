# IMRAD结构指南

## 概述

IMRAD（引言、方法、结果与讨论）是原创研究科学期刊论文的主要组织结构。自1970年代成为主流格式以来，现已成为医学、健康、生物、化学、工程和计算机科学领域的标准格式。

## 为什么要用IMRAD？

IMRAD结构模仿科学方法：
- **引言**：你提出了什么问题？
- **方法**：你是如何研究的？
- **结果**：你发现了什么？
- **讨论**：这意味着什么？

这种逻辑流程使科学论文更容易撰写、阅读和评估。

## 完整稿件组成部分

完整的科学稿件通常按以下顺序包含这些部分：

1. **标题**
2. **摘要**
3. **引言**
4. **方法**（也称为材料与方法、方法学）
5. **结果**
6. **讨论**（有时与结果合并）
7. **结论**（有时属于讨论的一部分）
8. **致谢**
9. **参考文献**
10. **补充材料**（如适用）

## 标题

### 目的
吸引读者并准确代表论文内容。

### 指南
- 简洁但具有描述性（通常10-15个词）
- 包含关键变量和研究的关系
- 避免缩写、术语和问句格式（除非期刊允许）
- 足够具体以区别于其他研究
- 包含关键搜索词以提高可发现性

### 示例
- 良好："高强度间歇训练对老年人心血管功能的影响"
- 太模糊："运动与健康"
- 过于详细："一项随机对照试验：比较高强度间歇训练与中等强度持续训练对60-75岁成人通过VO2最大值测量的心血管功能的影响"

## 摘要

### 目的
提供完整的独立摘要，使读者能够判断这篇论文是否与其相关。

### 格式：流畅段落（默认）

**⚠️ 关键：摘要应写成流畅段落，而非带标签的部分。**

大多数科学论文使用**非结构化摘要**，写成一到两个连贯的段落。这是包括Nature、Science、Cell、PNAS以及大多数专业期刊的标准格式。

❌ **错误——带标签的结构化摘要：**
```
背景：医院获得性感染仍然是发病的主要原因。
方法：我们进行了一项12个月的前后研究...
结果：干预后，表面污染减少了47%...
结论：UV-C消毒显著降低了感染率。
```

✅ **正确——流畅段落风格：**
```
医院获得性感染仍然是发病的主要原因，然而最佳消毒策略仍不清楚。我们在一家500张床位的教学医院进行了一项12个月的前后研究，以评估在标准清洁方案中添加UV-C消毒的效果。每月对环境表面进行培养，并通过监测数据追踪感染率。干预后，表面污染减少了47%（95% CI：38-56%，p<0.001），导尿管相关尿路感染从每1000导尿日3.2例降至1.8例（RR=0.56，95% CI：0.38-0.83，p=0.004）。未观察到不良影响。这些发现表明，UV-C消毒显著降低了环境污染和感染率，表明它可能成为医院感染控制方案的有价值的补充。
```

### 摘要结构（作为统一段落）

虽然写成流畅的 prose，摘要应按顺序涵盖以下要素：

1. **背景与问题**（1-2句）：为什么这项研究重要，存在什么空白
2. **研究描述**（1-2句）：做了什么以及如何做的（研究设计、方法）
3. **主要发现**（2-4句）：主要结果，包含具体定量数据
4. **意义**（1-2句）：解释、含义和结论

### 长度
- 通常150-300词（查看期刊要求）
- 一些期刊允许最多350词

### 关键规则
- **最后**撰写摘要（在完成所有其他部分之后）
- **写成流畅段落**——不带标签的部分
- 无需阅读论文即可完全理解
- 摘要中不引用参考文献
- 避免缩写或在首次使用时定义
- 方法和结果使用过去时，结论使用现在时
- 包含关键定量结果和统计指标
- 使用过渡词自然连接句子

### 何时使用结构化摘要（例外情况）

仅在以下情况使用带标签的部分（背景/目的、方法、结果、结论）：
- 期刊**明确要求**结构化摘要（见作者指南）
- 一些医学期刊常见（JAMA、BMJ、Annals of Internal Medicine）
- 始终在格式化前检查期刊要求

即使是结构化摘要，也应将每个部分写成完整句子，而非片段。

### 示例：流畅段落摘要

```
转录组年龄时钟具有独特优势，能够通过捕捉动态细胞状态和对扰动的急性反应来评估生物学年龄。使用包含超过120万个人类样本的统一处理RNA-seq数据的ARCHS4数据库，我们开发了深度神经网络模型来从基因表达谱预测时序年龄。我们表现最佳的模型在留出测试数据上实现了4.2年的平均绝对误差（R² = 0.89），显著优于传统机器学习方法，包括弹性网络回归（MAE = 6.8年）和随机森林（MAE = 5.9年）。特征重要性分析确定衰老、炎症和线粒体功能通路中的基因是最强的预测因子。跨组织验证显示，肺和血液样本产生最准确的预测，而肝脏显示最高方差。这些发现确立了深度学习作为转录组年龄预测的强大方法，并确定了生物学年龄评估的候选生物标志物。
```

## 引言

### 目的
让读者相信该研究使用适当的方法解决了一个重要问题。

### 结构与内容

**第一段：宏观背景**
- 建立广泛的研究领域
- 解释为什么这个主题重要
- 对已知事实使用现在时
- 使非专业人士也能理解

**第二至三段：逐步缩小范围**
- 回顾相关先前研究
- 展示已知内容
- 识别现有工作中的争议或局限性
- 为空白创造逻辑推进

**第四段：空白**
- 明确识别仍未知晓的内容
- 解释为什么这个知识空白是有问题的
- 将空白与宏观重要性联系起来

**最后一段：本研究**
- 陈述具体的研究问题或假设
- 简要描述整体方法
- 解释本研究如何填补空白
- 可选：预览主要发现（一些期刊不鼓励这样做）

### 长度
- 通常1.5-2页（取决于期刊）
- 通常4-5段
- 通信/简短来信更短

### 动词时态
- **现在时**：已知事实（"运动改善心血管健康"）
- **过去时**：先前研究及其发现（"Smith等人发现..."）
- **现在时/过去时**：你的研究目的（"本研究探讨..."或"本研究探讨了..."）

### 常见错误要避免
- 开头过于宽泛（例如，"自从时间开始..."）
- 详尽的文献综述（保留给综述文章）
- 引用无关或过时的参考文献
- 未能识别明确的空白
- 对研究的理由不充分
- 未陈述明确的研究问题或假设
- 包含方法或结果（这些属于后面的部分）

### 要回答的关键问题
1. 关于这个主题我们知道什么？
2. 我们不知道什么？（空白）
3. 为什么这个空白很重要？
4. 本研究旨在发现什么？

## 方法

### 目的
提供足够的细节，以便他人能够重复这项研究并评估其有效性。

### 关键原则
该领域的其他专家应该能够完全按照你的操作重复你的实验。

### 标准子部分

#### 研究设计
- 陈述整体设计（例如，随机对照试验、队列研究、横断面调查）
- 如果设计选择不明显，说明其理由
- 如适用，提及盲法、随机化或对照

#### 参与者/受试者/样本
- 定义感兴趣的人群
- 精确描述纳入和排除标准
- 报告样本量及其确定方式（功效分析）
- 解释招募方法和设置
- 对于动物：指定物种、品系、年龄、性别、饲养条件

#### 材料和设备
- 列出使用的所有材料、试剂和设备
- 包含制造商名称和地址（用括号括起）
- 指定专业物品的目录号
- 报告软件名称和版本

#### 程序
- 按时间顺序描述做了什么
- 包含足够的细节以便重复
- 使用副标题组织复杂程序
- 指定时间（例如，"在37°C孵育2小时"）
- 对于调查/访谈：描述工具、验证、管理

#### 测量和结果
- 定义所有测量的变量
- 指定主要和次要结果
- 描述测量工具及其有效性
- 包含测量单位

#### 统计分析
- 命名所有使用的统计检验
- 说明检验选择的理由
- 说明显著性水平（通常α = 0.05）
- 报告样本量的功效分析
- 命名统计软件及其版本
- 描述缺失数据的处理
- 如适用，提及多重比较的调整

#### 伦理考量
- 说明IRB/伦理委员会批准（附批准号）
- 说明知情同意程序
- 对于人类研究：说明遵守《赫尔辛基宣言》
- 对于动物研究：说明遵守相关指南（例如，ARRIVE）

### 长度
- 通常2-4页
- 与研究复杂程度成比例

### 动词时态
- **过去时**用于你执行的操作（"我们测量了..."，"参与者完成了..."）
- **现在时**用于既定程序（"PCR扩增..."，"问卷包含..."）

### 常见错误
- 重复实验的细节不足
- 方法首次出现在结果中
- 包含结果或讨论
- 缺少统计检验
- 未定义的缩写
- 缺少伦理批准声明

## 结果

### 目的
客观呈现发现，不加解释。

### 关键原则
展示，不要解释。将解释留给讨论部分。

### 结构与内容

**开篇段落**
- 描述参与者/样本特征
- 报告招募流程（例如，筛选、纳入、完成）
- 考虑包含CONSORT风格的流程图

**后续段落**
- 按逻辑顺序呈现结果（通常主要结果优先）
- 遵循引言中陈述的目标顺序
- 按主题或时间顺序组织，取决于哪种更清晰
- 按编号引用图表

**每个发现应包含：**
- 观察到的结果
- 效应的方向
- 效应的大小
- 统计显著性
- 置信区间

**示例**："干预组平均收缩压降低了12 mmHg，而对照组降低了3 mmHg（差异：9 mmHg，95% CI：4-14 mmHg，p=0.002）。"

### 与图表的整合

**何时使用：**
- **图**：趋势、模式、分布、比较、关系
- **表**：精确值、人口统计数据、多个变量

**如何引用：**
- "图1显示了...的分布"（而非"下图图1"）
- "表2呈现了基线特征"
- 不要在正文中重复所有表数据；突出关键发现
- 每个图/表应在正文中被引用

### 图表指南
- 按提及顺序连续编号
- 包含完整、独立的图例
- 在图例或脚注中定义所有缩写
- 报告样本量（n）
- 标明统计显著性（*，p值）
- 使用一致的格式

### 统计报告

**必需要素：**
- 检验统计量（t、F、χ²等）
- 自由度
- p值（如果p > 0.001则报告精确值，否则报告为"p < 0.001"）
- 效应量和置信区间
- 样本量

**示例**："组在测试表现上显著不同（t(48) = 3.21，p = 0.002，Cohen's d = 0.87，95% CI：0.34-1.40）。"

### 长度
- 通常2-4页
- 与方法部分大致相当

### 动词时态
- **过去时**用于你的发现（"平均值是..."，"参与者显示..."）

### 常见错误
- 解释结果（保留给讨论）
- 在正文中重复所有表/图数据
- 呈现新方法
- 统计细节不足
- 单位或符号不一致
- 未处理负面或意外发现
- 选择性报告（所有检验的假设都应报告）

### 组织策略

**按目标：**
```
干预对主要结果的影响
干预对次要结果A的影响
干预对次要结果B的影响
```

**按分析类型：**
```
描述性统计
单变量分析
多变量分析
```

**按时间顺序：**
```
基线特征
短期结果（1个月）
长期结果（6个月）
```

## 讨论

### 目的
解释发现，将其与现有知识联系起来，承认局限性，并提出未来方向。

### 结构与内容

**第一段：主要发现总结**
- 重申主要目标或假设
- 用2-4句话总结主要发现
- 避免重复结果中的细节
- 明确说明假设是否得到支持

**第二至四段：情境中的解释**
- 将你的发现与先前研究进行比较
- 解释与先前工作的一致和分歧
- 为发现提出机制或解释
- 讨论意外结果
- 考虑替代解释
- 讨论发现是支持还是反驳现有理论

**第五段：优势与局限性**
- 诚实地承认研究局限性
- 解释局限性如何影响解释
- 提及研究优势（设计、样本、方法）
- 避免笼统的局限性（"需要更大的样本"）——要具体

**第六段：意义**
- 临床意义（对于医学研究）
- 实际应用
- 政策意义
- 理论贡献

**最后一段：结论与未来方向**
- 总结要点信息
- 提出具体 future research 来解决空白或局限性
- 以强有力的结论陈述结束

### 长度
- 通常3-5页
- 通常是最长的部分

### 动词时态
- **过去时**：你的研究发现（"我们发现..."，"结果显示..."）
- **现在时**：既定事实和你的解释（"这表明..."，"这些发现表明..."）
- **将来时**：意义和未来研究（"未来研究应探讨..."）

### 讨论策略

**与先前工作比较：**
```
"我们发现症状减少30%与Smith等人（2023年）的发现一致，他们报告使用类似干预措施减少了28%。然而，Jones等人（2022年）未发现显著效果，这可能是因为他们使用了强度较低的治疗方案。"
```

**提出机制：**
```
"观察到的认知功能改善可能源于 cerebral blood flow 的增加，正如前额叶皮层功能MRI信号的并发增加所证明的。这一解释与认知增强的血管假说一致。"
```

**承认局限性：**
```
"横断面设计 prevents causal inference。此外，来自单一学术医学中心的便利样本可能限制了向社区环境的推广。自评 measures 可能引入回忆 bias，尽管我们试图通过结构化访谈 minimize 这一点。"
```

### 常见错误
- 只是重复结果而不解释
- 过度解释发现或在无依据的情况下声称因果关系
- 忽略不一致或负面发现
- 未能与现有文献比较
- 引入新数据或方法
- 笼统或浅显地讨论局限性
- 超出研究人群过度概括
- 缺少"那又如何？"——未能解释意义

### 要回答的关键问题
1. 这些发现意味着什么？
2. 它们与先前研究如何比较？
3. 为什么可能存在差异？
4. 替代解释是什么？
5. 局限性是什么？
6. 实际意义是什么？
7. 未来研究应该探讨什么？

## 结论

### 目的
提供关键发现及其意义的简洁总结。

### 位置
- 可能是单独的部分或讨论的最后一段（检查期刊要求）

### 内容
- 最多1-2段
- 重申主要发现
- 强调意义或含义
- 以强有力的、令人难忘的陈述结束
- **不要**引入新信息

### 示例
```
这项随机试验表明，为期12周的正念干预显著减少了大学生焦虑症状，效果在6个月随访时持续存在。这些发现支持将正念项目纳入大学心理健康服务。鉴于基于群体的正念训练的可扩展性和成本效益，这种方法为解决高等教育日益严重的心理健康危机提供了一种有前景的策略。
```

### 致谢
- 感谢资助来源（附资助编号）
- 感谢不符合作者资格但有实质性贡献的人员
- 感谢提供材料、设备或帮助的人员
- 声明任何利益冲突

### 参考文献
- 按期刊格式（参见`citation_styles.md`）
- 验证所有引用都准确
- 确保所有引用在正文中出现且反之亦然
- 原创研究的典型范围：20-50条参考文献

### 补充材料
- 附加的图、表或数据集
- 详细方案或问卷
- 视频或音频文件
- 大型数据集或代码库

## 时态使用总结

| 部分 | 动词时态 |
|---------|-----------|
| 摘要 - 背景 | 现在时（已知事实）或过去时（先前研究） |
| 摘要 - 方法 | 过去时 |
| 摘要 - 结果 | 过去时 |
| 摘要 - 结论 | 现在时 |
| 引言 - 一般背景 | 现在时 |
| 引言 - 先前研究 | 过去时 |
| 引言 - 你的目标 | 现在时或过去时 |
| 方法 | 过去时（你的操作），现在时（一般程序） |
| 结果 | 过去时 |
| 讨论 - 你的发现 | 过去时 |
| 讨论 - 解释 | 现在时 |
| 讨论 - 先前工作 | 现在时或过去时 |
| 结论 | 现在时 |

## IMRAD变体

### 合并结果与讨论
- 一些期刊允许或要求此格式
- 交织呈现和解释
- 每个结果呈现后立即讨论
- 对于具有多个实验的复杂研究有用

### 无独立结论的IMRaD
- 结论整合到讨论的最后一段
- 许多期刊常见

### 扩展IMRAD（ILMRaD）
- 添加"文献综述"作为独立部分
- 在论文和学位论文中更常见

## 针对不同研究类型调整IMRAD

### 临床试验
- 在结果中添加CONSORT流程图
- 在方法中包含试验注册号
- 在结果中报告不良事件

### 系统综述/荟萃分析
- 方法描述搜索策略和纳入标准
- 结果包含PRISMA流程图和综合
- 可能还有额外部分（偏倚风险评估）

### 病例报告
- 引言：病情背景
- 病例呈现：取代方法和结果
- 讨论：将病例与文献联系起来

### 观察性研究
- 遵守STROBE指南
- 仔细注意方法中的潜在混杂因素
- 讨论解决因果关系局限性

## 场所特定的结构期望

### 期刊 vs 会议格式

| 场所类型 | 长度 | 结构 | 方法位置 | 重点 |
|-----------|--------|-----------|-------------------|-----------|
| **Nature/Science** | 2,000-4,500词 | 改进的IMRAD | 补充材料 | 广泛意义 |
| **医学** | 2,700-3,500词 | 严格IMRAD | 正文 | 临床结果 |
| **专业期刊** | 3,000-6,000词 | 标准IMRAD | 正文 | 技术深度 |
| **ML会议** | 8-9页（约6,000词） | 引言-方法-实验-结论 | 正文（简洁） | 新颖贡献 |

### ML会议结构（NeurIPS/ICML/ICLR）

**典型8页结构：**
1. **摘要**（150-200词）：问题、方法、主要结果
2. **引言**（1页）：动机、贡献总结、相关工作概述
3. **方法**（2-3页）：技术方法、架构、算法
4. **实验**（2-3页）：设置、数据集、基线、结果、消融
5. **相关工作**（0.5-1页，常在附录）：详细文献比较
6. **结论**（0.25-0.5页）：总结、局限性、 future work
7. **参考文献**（在页数限制内或单独，取决于会议）
8. **附录/补充材料**（无限制）：额外实验、证明、细节

**与期刊的关键区别：**
- **贡献要点**：引言中常为编号列表（例如，"我们的贡献是：(1)... (2)... (3)..."）
- **无独立结果/讨论**：在实验部分整合
- **消融研究**：关键组成部分，显示什么重要
- **计算要求**：常需要（训练时间、GPU、内存）
- **代码可用性**：越来越被期望

### 部分长度比例

| 场所 | 引言 | 方法 | 结果/实验 | 讨论/结论 |
|-------|-------|---------|---------------------|----------------------|
| **Nature/Science** | 10% | 15%* | 40% | 35% |
| **医学（NEJM/JAMA）** | 10% | 25% | 30% | 35% |
| **专业期刊** | 20% | 25% | 30% | 25% |
| **ML会议** | 12-15% | 30-35% | 40-45% | 5-8% |

*Nature/Science的方法常在补充材料中

**关键医学期刊特点：**
- NEJM/Lancet/JAMA：严格IMRAD；临床重点；结构化讨论；遵守CONSORT/STROBE
- 明确的主要/次要结果；统计预先规定

**关键ML会议特点：**
- 引言中的编号贡献列表
- 带伪代码/方程的方法细节
- 广泛实验：主要结果、消融、分析
- 简短结论（注明局限性）
- 相关工作常在附录

### 场所写作风格

| 场所 | 读者 | 引言重点 | 方法细节 | 结果/实验 | 讨论/结论 |
|-------|----------|-------------|----------------|---------------------|----------------------|
| **Nature/Science** | 非专业人士 | 广泛意义 | 简洁，补充材料 | 故事驱动 | 广泛意义 |
| **医学** | 临床医生 | 临床问题 | 全面 | 首要结果优先 | 临床相关性 |
| **专业** | 专家 | 领域背景 | 完整技术 | 按实验 | 机制深度 |
| **ML会议** | ML研究者 | 新颖贡献 | 可重复 | 基线、消融 | 简短，局限性 |

**ML会议重点：**
- **引言**：清晰的问题陈述；编号贡献；与先前工作的定位
- **方法**：数学符号；伪代码；架构图；复杂度分析
- **实验**：描述数据集；多个基线；消融研究；误差分析
- **结论**：总结；承认局限性；更广泛影响（如要求）

### 场所评估

**检查内容：**
- **适合性**：适合场所范围和读者
- **长度**：在限制内（会议严格）
- **清晰度**：写作质量足够；声称有支持
- **可重复性**：方法支持重复
- **完整性**：报告所有结果；承认局限性

**常见拒绝原因：**
- 对场所意义不足
- 方法缺乏重复细节
- 结果不支持声称
- 讨论过度陈述发现
- 超出页/词限制（会议严格）

**ML会议特定评估：**
- 清晰的问题表述和动机
- 新颖性和贡献阐述清晰
- 基线全面且公平
- 消融研究展示什么有效
- 代码/数据可用性（越来越要求）
- 可重复性信息（种子、超参数）

### 快速适应指南

**期刊 → ML会议：**
- 精简引言；添加编号贡献
- 方法：保持简洁，添加伪代码
- 合并结果+讨论 → 实验部分
- 添加广泛消融和基线比较
- 简短结论，注明局限性

**ML会议 → 期刊：**
- 扩展引言，添加更多背景
- 将方法部分分开，提供完整细节
- 将实验分成结果和讨论
- 去除贡献编号
- 扩展局限性讨论

**专业 → 广泛期刊：**
- 简化引言；强调广泛意义
- 将技术方法移至补充材料
- 故事驱动的结果组织
- 以意义引领讨论

**广泛 → 专业：**
- 添加详细文献综述
- 完整方法放在正文
- 按实验组织结果
- 增加机制讨论深度
