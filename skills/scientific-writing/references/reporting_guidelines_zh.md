# 科学研究的报告指南

## 概述

报告指南是基于证据的建议，说明在报告特定类型的研究时应包含哪些信息。它们提供检查清单和流程图，以确保完整、准确和透明的报告，这对于读者评估研究有效性以及其他研究人员复制该工作至关重要。

EQUATOR网络（增强健康研究的质量和透明度）维护着一个全面的报告指南库。使用适当的报告指南可以提高手稿质量并增加被接受的可能性。

## 为什么要使用报告指南？

### 好处

**对于作者：**
- 确保不会遗漏重要内容
- 提高接受率
- 改善手稿组织
- 减少审稿人要求补充信息的要求

**对于读者和审稿人：**
- 能够Critical评估研究有效性
- 促进系统综述和Meta分析
- 改进对实际做了什么工作的理解

**对于科学：**
- 增强可重复性
- 减少研究浪费
- 提高透明度
- 能够进行更好的证据综合

### 使用时机

- **在研究设计期间**：许多指南包含方案版本（例如，SPIRIT用于试验方案）
- **在手稿撰写期间**：使用检查清单确保涵盖所有项目
- **在提交之前**：验证依从性，通常随稿件一起提交检查清单
- **许多期刊要求**：将报告指南检查清单作为提交的一部分

## 按研究类型分类的主要报告指南

### CONSORT - 随机对照试验

**全称：**  consolidated Standards of Reporting Trials

**使用时机：** 任何随机对照试验（RCT），包括试点和可行性试验

**最新版本：** CONSORT 2010（更新声明）

**主要组成部分：**
- **检查清单**：25个项目，涵盖标题、摘要、引言、方法、结果、讨论
- **流程图**：参与者的入组、分配、随访和分析流程

**主要检查清单项目：**
1. 标题表明研究为随机试验
2. 结构化摘要
3. 科学背景和理由
4. 具体目标和假设
5. 试验设计描述（平行、交叉、析因等）
6. 参与者的资格标准
7. 数据收集的设置和位置
8. 详细描述干预措施以便于复制
9. 定义主要和次要结局
10. 样本量确定和把握度计算
11. 随机序列生成
12. 分配隐藏机制
13. 盲法实施
14. 统计学方法
15. 参与者流程及退出原因
16. 招募日期和随访日期
17. 基线特征表
18. 每个结局的分析结果
19. 伤害和不良事件
20. 试验的局限性
21. 可推广性
22. 解释与结果一致
23. 试验注册号
24. 完整方案访问权限
25. 资金来源

**针对特定设计的扩展：**
- CONSORT 用于整群随机试验
- CONSORT 用于非劣效性和等效性试验
- CONSORT 用于实效性试验
- CONSORT 用于交叉试验
- CONSORT 用于N-of-1试验
- CONSORT 用于阶梯楔形设计

**获取地址：** http://www.consort-statement.org/

### STROBE - 观察性研究

**全称：** Strengthening the Reporting of Observational Studies in Epidemiology

**使用时机：** 队列研究、病例对照研究和横断面研究

**最新版本：** STROBE 2007（广泛采用的标准）

**涵盖的主要研究设计：**
- **队列**：随访暴露组和非暴露组
- **病例对照**：比较病例和对照的暴露史
- **横断面**：同时测量暴露和结局

**主要检查清单项目（22项）：**
1. 标题和摘要表明研究设计
2. 背景和理由
3. 目标
4. 研究设计及理由
5. 设置、位置和日期
6. 资格标准和选择方法
7. 清晰定义变量（结局、暴露、混杂因素）
8. 数据来源和测量方法
9. 偏倚管理策略
10. 研究规模论证
11. 定量变量的处理
12. 统计学方法，包括混杂和交互
13. 敏感性分析
14. 参与者流程及未参与原因
15. 描述性数据，包括随访时间
16. 结局数据
17. 主要结果，包括未调整和调整后的估计值
18. 其他分析（亚组、敏感性分析）
19. 关键结果总结
20. 局限性及潜在偏倚讨论
21. 解释和可推广性
22. 资金来源和作用

**扩展：**
- STROBE-ME（分子流行病学）
- RECORD（常规收集的健康数据）
- STROBE-RDS（受访者驱动抽样）

**获取地址：** https://www.strobe-statement.org/

### PRISMA - 系统综述和Meta分析

**全称：** Preferred Reporting Items for Systematic Reviews and Meta-Analyses

**使用时机：** 伴有或不伴Meta分析的系统综述

**最新版本：** PRISMA 2020（重要更新）

**主要组成部分：**
- **检查清单**：27个项目，涵盖所有部分
- **流程图**：研究选择过程

**主要部分：**
1. **标题**：识别为系统综述/Meta分析
2. **摘要**：结构化总结
3. **引言**：理由和目标
4. **方法**：
   - 资格标准
   - 信息来源（数据库、日期）
   - 搜索策略（至少一个数据库的完整策略）
   - 选择过程
   - 数据收集过程
   - 提取的数据项目
   - 偏倚风险评估
   - 效应量
   - 综合方法
   - 发表偏倚评估
   - 确定性评估（如GRADE）
5. **结果**：
   - 研究选择流程图
   - 研究特征
   - 偏倚风险评估结果
   - 综合结果（如果适用，进行Meta分析）
   - 发表偏倚
   - 证据的确定性
6. **讨论**：
   - 局限性
   - 解释
   - 意义

**扩展：**
- PRISMA 用于摘要
- PRISMA 用于方案（PRISMA-P）
- PRISMA 用于网络Meta分析
- PRISMA 用于范围综述（PRISMA-ScR）
- PRISMA 用于个体患者数据
- PRISMA 用于诊断准确性
- PRISMA 用于公平性综述

**获取地址：** http://www.prisma-statement.org/

### SPIRIT - 临床试验的研究方案

**全称：** Standard Protocol Items: Recommendations for Interventional Trials

**使用时机：** 随机试验和其他计划干预研究的方案

**最新版本：** SPIRIT 2013

**目的：** 确保试验方案在试验开始前包含完整描述

**主要检查清单项目（33项）：**
- 行政信息（标题、试验注册、资金）
- 引言（背景、理由、目标）
- 方法：试验设计
  - 研究设置
  - 资格标准
  - 详细干预措施
  - 结局（主要和次要）
  - 参与者时间表
  - 样本量计算
  - 招募策略
  - 分配和随机化
  - 盲法
  - 数据收集方法
  - 数据管理
  - 统计学方法
  - 监测（数据监测委员会）
  - 伤害报告
  - 审计
- 伦理和传播
  - 伦理批准
  - 知情同意程序
  - 保密
  - 传播计划

**获取地址：** https://www.spirit-statement.org/

### STARD - 诊断准确性研究

**全称：** Standards for Reporting of Diagnostic Accuracy Studies

**使用时机：** 评估诊断测试准确性的研究

**最新版本：** STARD 2015

**主要检查清单项目（30项）：**
1. 研究设计识别
2. 背景信息和目标
3. 研究设计描述
4. 参与者选择标准和招募
5. 数据收集方法
6. 指数测试描述和执行
7. 参考标准描述
8. 选择参考标准的理由
9. 测试结果定义和 cutoff 值
10. 参与者和时间流程
11. 基线人口学和临床特征
12. 指数测试结果与参考标准的交叉表
13. 诊断准确性估计值及置信区间
14. 不确定结果的处理
15. 测试的不良事件

**流程图：** 显示参与者流程和测试结果

**获取地址：** https://www.equator-network.org/reporting-guidelines/stard/

**全称：** Standards for Reporting Qualitative Research

## 如何使用报告指南

### 在研究规划期间

1. **根据研究设计识别相关指南**
2. **审查需要规划的项目**（如随机化、盲法）
3. **设计研究**以确保捕获所有必要元素
4. **考虑方案指南**（如SPIRIT用于试验）

### 在手稿撰写期间

1. **从指南网站下载检查清单**
2. **系统处理每个项目**
3. **注明每个项目在手稿中的位置**（页码/行号）
4. **修改手稿**以包含缺失的项目
5. **适当使用流程图**

### 在提交之前

1. **完成正式检查清单**，标注页码
2. **审查所有项目**是否充分解决
3. **如果期刊要求**，随提交材料一起提交检查清单
4. **在cover letter或方法部分**注明指南依从性

### 检查清单条目示例

```
Item 7: 参与者的资格标准，以及收集数据的环境和位置
第6页，第112-125行："参与者是60-85岁的社区居住成人，根据Petersen标准诊断为轻度认知障碍（MCI）。排除标准包括痴呆诊断、主要精神障碍或不稳定的医疗状况。招募于2022年1月至2023年12月在波士顿的三个记忆诊所进行。"
```

## 找到正确的指南

### EQUATOR网络搜索

**网站：** https://www.equator-network.org/

**使用方法：**
1. 从向导中选择您的研究设计
2. 按健康研究类别浏览
3. 搜索特定关键词
4. 按指南状态筛选（开发阶段）

### 按研究设计

| 如果您的研究是... | 使用此指南 |
|----------------------|-------------------|
| 随机对照试验 | CONSORT |
| 队列、病例对照或横断面研究 | STROBE |
| 系统综述或Meta分析 | PRISMA |
| 试验方案 | SPIRIT |
| 诊断准确性研究 | STARD |
| 预测模型研究 | TRIPOD |
| 动物研究 | ARRIVE |
| 病例报告 | CARE |
| 质量改进研究 | SQUIRE |
| 经济学评估 | CHEERS |
| 定性研究 | SRQR 或 COREQ |

### 多个指南

**某些研究可能需要多个指南：**

**示例1：** 带有定性成分的试点RCT
- CONSORT 用于定量部分
- SRQR 用于定性部分

**示例2：** 诊断测试的系统综述
- PRISMA 用于综述方法
- STARD 注意事项用于纳入的研究

1. **入组**：评估资格
2. **分配**：随机分配到组

**阶段：**
1. **识别**：来自数据库和注册处的记录

## 常见错误及避免方法

### 错误1：根本不使用指南

**影响：** 缺失关键信息，接受机会降低

**解决方案：** 从研究规划阶段识别并使用适当的指南

### 错误2：仅在手稿完成后才使用指南

**影响：** 可能意识到关键数据未收集或记录

**解决方案：** 在研究设计和数据收集期间审查指南

### 错误3：检查清单完成不完整

**影响：** 遗漏的项目仍未报告

**解决方案：** 系统地解决每个检查清单项目

### 错误4：使用过时的指南

**影响：** 错失报告标准的最新改进

**解决方案：** 始终在官方指南网站检查最新版本

### 错误5：为研究设计使用错误的指南

**影响：** 重要的设计特定元素未报告

**解决方案：** 仔细将研究设计与适当的指南匹配

### 错误6：需要时未提交检查清单

**影响：** 编辑初审拒绝或延误

**解决方案：** 检查期刊提交指南并包含检查清单

### 错误7：_generic报告缺乏特异性

**影响：** 信息不足以进行复制或评估

**解决方案：** 为每个项目提供具体、详细的信息

## 期刊要求

### 许多期刊现在要求：

1. **在方法中声明** 依从于报告指南
2. **上传完成的检查清单** 作为补充文件
3. **在检查清单上标注页码/行号** 表明项目解决位置
4. **流程图** 作为手稿中的图形

### 方法声明示例：

```
"本研究报告遵循加强流行病学观察性研究报告（STROBE）声明。
完整的STROBE检查清单作为补充文件1提供。"
```

### 有严格要求的期刊：

- PLOS期刊（要求特定设计的检查清单）
- BMJ（要求CONSORT、PRISMA等）
- The Lancet（要求依从性声明）
- JAMA和JAMA网络期刊（要求检查清单）
- Nature期刊组合（鼓励使用指南）

## 检查清单：使用报告指南

**在开始研究之前：**
- [ ] 识别了适当的报告指南
- [ ] 审查了需要前瞻性规划的项目
- [ ] 设计研究以捕获所有必要元素
- [ ] 如适用，注册了方案

**在手稿撰写期间：**
- [ ] 下载了指南检查清单的最新版本
- [ ] 系统地解决了每个检查清单项目
- [ ] 创建了要求的流程图
- [ ] 注明了每个项目解决的位置（页码/行）

**在提交之前：**
- [ ] 完成了带页码的正式检查清单
- [ ] 验证所有项目已充分解决
- [ ] 在方法中包含了依从性声明
- [ ] 如需要，将检查清单准备为补充文件
- [ ] 检查了期刊特定要求
- [ ] 在cover letter中提到了指南依从性

## 按场所的特定报告要求

### 按场所类型的报告标准

| 场所类型 | 指南使用 | 透明度要求 |
|-----------|--------------|---------------------------|
| **医学期刊** | 强制（CONSORT、STROBE等） | 提交时需要检查清单 |
| **PLOS/BMC** | 强制（针对某些研究类型） | 检查清单作为补充上传 |
| **Nature/Science** | 推荐 | 强调方法完整性 |
| **ML会议** | 无正式指南 | 需要可重复性详情 |

### ML会议报告标准

**NeurIPS/ICML/ICLR可重复性要求：**
- **数据集**：名称、版本、访问方法、预处理
- **代码**：可用性声明；GitHub常见
- **超参数**：报告所有设置（学习率、批量大小等）
- **种子**：用于可重复性的随机种子
- **计算资源**：使用的GPU、训练时间
- **统计显著性**：误差线、置信区间、多次运行
- **更广泛影响声明**（NeurIPS）：社会影响

**要包含的内容（通常在附录中）：**
- 完整的超参数设置
- 训练细节和收敛标准
- 硬件规格
- 软件版本（PyTorch 2.0等）
- 数据集划分和任何预处理
- 评估指标和协议

### 执行和评估

**检查的内容：**
- **医学期刊**：上传检查清单；方法中的依从性声明；系统性完整性
- **PLOS/BMC**：某些设计的强制检查清单；强调可重复性
- **高影响力期刊**：方法足以进行复制（通常不需要检查清单）
- **ML会议**：可重复性检查清单（NeurIPS）；越来越期望代码可用性

**导致拒稿的常见问题：**
- 缺少要求的检查清单（医学期刊）
- 方法细节不足以进行复制
- 缺少关键信息（随机化、盲法、把握度计算）
- 缺少要求的数据/代码可用性声明

**方法声明示例：**

**期刊（STROBE）：**
```
本研究遵循STROBE报告指南。检查清单见补充文件1。
```

**ML会议（可重复性）：**
```
代码可在github.com/user/project获取。所有超参数在附录A中。
训练使用4×A100 GPU（~20小时）。种子：{42, 123, 456}。
```

### 提交前报告检查清单

**对于临床试验（医学期刊）：**
- [ ] CONSORT检查清单已完成并标注页码
- [ ] 试验注册号在摘要和方法中
- [ ] 包含CONSORT流程图
- [ ] 描述了统计分析计划
- [ ] 方法中有依从性声明

**对于观察性研究（医学/流行病学）：**
- [ ] STROBE检查清单已完成
- [ ] 清晰说明研究设计
- [ ] 详细说明统计学方法
- [ ] 解决了混杂因素
- [ ] 方法中有依从性声明

**对于系统综述：**
- [ ] PRISMA检查清单已完成
- [ ] 包含PRISMA流程图
- [ ] 已注册方案（PROSPERO）
- [ ] 记录了搜索策略
- [ ] 包含偏倚风险评估

**对于ML会议论文：**
- [ ] 所有数据集都已命名并标明版本
- [ ] 说明了代码可用性（如可用，提供GitHub链接）
- [ ] 列出超参数（附录可接受）
- [ ] 报告了随机种子
- [ ] 指定了计算资源
- [ ] 显示了误差线/置信区间
- [ ] 更广泛影响声明（如果要求）
