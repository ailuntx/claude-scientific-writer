# 科学写作

## 概述

**这是深度研究与写作工具的核心技能**——将 AI 驱动的深度研究与格式良好的书面输出相结合。通过 research-lookup 技能，每份生成的文档都经过全面的文献检索和验证的引用支持。

科学写作是以精确和清晰的方式传达研究的过程。使用 IMRAD 结构、引用（APA/AMA/Vancouver）、图表和报告规范（CONSORT/STROBE/PRISMA）撰写稿件。将本技能应用于研究论文和期刊投稿。

**关键原则：始终以完整的段落和流畅的文笔撰写。最终稿件中绝不要提交要点列表。** 使用两阶段流程：首先使用 research-lookup 创建带有要点的章节大纲，然后将这些大纲转化为完整的段落。

## 何时使用此技能

在以下情况下应使用此技能：
- 撰写或修改科学稿件的任何章节（摘要、引言、方法、结果、讨论）
- 使用 IMRAD 或其他标准格式构建研究论文结构
- 按特定风格格式化引用和参考文献（APA、AMA、Vancouver、Chicago、IEEE）
- 创建、格式化或改进图表和数据可视化
- 应用特定于研究类型的报告规范（试验用 CONSORT，观察性研究用 STROBE，系统综述用 PRISMA）
- 撰写符合期刊要求的摘要（结构化或非结构化）
- 为特定期刊准备投稿
- 提高写作清晰度、简洁性和精确性
- 确保正确使用领域特定的术语和命名法
- 回应审稿人意见并修改稿件

## 使用科学示意图增强视觉效果

**⚠️ 强制要求：每篇科学论文必须包含一幅图形摘要，并使用 scientific-schematics 技能额外生成 1-2 张 AI 生成的图片。**

这不是可选项。没有视觉元素的科学论文是不完整的。在最终定稿任何文件之前：
1. **始终生成一幅图形摘要** 作为首个视觉元素
2. 至少使用 scientific-schematics 生成**一幅**额外的示意图或图表
3. 对于综合性论文，最好总共使用 3-4 张图片（图形摘要 + 方法流程图 + 结果可视化 + 概念图）

### 图形摘要（必需）

**每篇科学论文都必须包含一幅图形摘要。** 这是你论文的视觉总结，它：
- 出现于文字摘要之前或紧随其后
- 用一张图片捕捉整篇论文的核心信息
- 适合期刊目录展示
- 使用横向（通常 1200x600px）

**首先生成图形摘要：**
```bash
python scripts/generate_schematic.py "Graphical abstract for [paper title]: [brief description showing workflow from input → methods → key findings → conclusions]" -o figures/graphical_abstract.png
```

**图形摘要要求：**
- **内容**：展示工作流程、关键方法、主要发现和结论的视觉总结
- **风格**：简洁、专业，适合期刊目录
- **元素**：包含 3-5 个关键步骤/概念，并用箭头或流程线连接
- **文字**：标签最少化，使用大而清晰的可读字体
- 日志记录：`[HH:MM:SS] GENERATED: Graphical abstract for paper summary`

## 核心能力

### 1. 稿件结构与组织

**IMRAD 格式**：指导论文按照大多数科学学科使用的标准引言、方法、结果、讨论结构撰写。包括：
- **引言**：确立研究背景、指出空白、陈述目标
- **方法**：详细说明研究设计、人群、程序和分析方法
- **结果**：客观呈现发现，不加解释
- **讨论**：解读结果、承认局限性、提出未来方向

有关 IMRAD 结构的详细指导，请参阅 `references/imrad_structure.md`。

**替代结构**：支持特定学科格式，包括：
- 综述文章（叙述性、系统性、范围综述）
- 病例报告和病例系列
- 荟萃分析和合并分析
- 理论/建模论文
- 方法学论文和研究方案

### 2. 各章节写作指导

**摘要撰写**：构思简洁、独立的摘要（100-250 字），概括论文的目的、方法、结果和结论。支持结构化摘要（带标签章节）和非结构化的单段格式。

**引言展开**：构建有力的引言，需：
- 确立研究问题的重要性
- 系统地回顾相关文献
- 指出现有知识空白或争议
- 陈述明确的研究问题或假设
- 解释研究的新颖性和意义

**方法学记录**：通过以下方式确保可重复性：
- 详细的参与者/样本描述
- 清晰的程序记录
- 统计方法及其理由
- 设备和材料规格
- 伦理批准和知情同意声明

**结果呈现**：展示结果时：
- 从主要结果到次要结果的逻辑流程
- 与图表有效结合
- 报告统计显著性及效应量
- 客观报告，不加解释

**讨论构建**：通过以下方式综合发现：
- 将结果与研究问题联系起来
- 与现有文献进行比较
- 诚恳地承认局限性
- 提出机制性解释
- 建议实际意义和未来研究

### 3. 引用和参考文献管理

正确应用各学科的引用风格。有关完整的风格指南，请参阅 `references/citation_styles.md`。

**主要引用风格：**
- **AMA（美国医学协会）**：数字上标引用，医学常用
- **Vancouver**：方括号数字引用，生物医学标准
- **APA（美国心理学会）**：作者-日期文内引用，社会科学常用
- **Chicago**：注释-书目或作者-日期制，人文学科与科学领域
- **IEEE**：方括号数字引用，工程与计算机科学

**最佳实践：**
- 尽可能引用原始来源
- 包含近期文献（活跃领域最近 5-10 年）
- 在引言和讨论中平衡引用分布
- 对照原始来源核实所有引用
- 使用参考文献管理软件（Zotero、Mendeley、EndNote）

### 4. 图表

创建有效的数据可视化，以增强理解。有关详细的最佳实践，请参阅 `references/figures_tables.md`。

**何时使用表格与图片：**
- **表格**：精确的数值数据、复杂数据集、需要精确值的多个变量
- **图片**：趋势、模式、关系、通过视觉最佳理解的对比

**设计原则：**
- 每张表/图应通过完整的标题做到自明
- 所有显示项使用一致的格式和术语
- 标注所有坐标轴、列和行及单位
- 包含样本量（n）和统计注释
- 遵循“每 1000 字一张表/图”的指引
- 避免在正文、表格和图片之间重复信息

**常见图表类型：**
- 柱状图：比较离散类别
- 折线图：显示时间趋势
- 散点图：展示相关性
- 箱线图：展示分布和异常值
- 热图：可视化矩阵和模式

### 5. 研究类型对应的报告规范

遵循已建立的报告标准以确保完整性和透明度。有关规范的详细信息，请参阅 `references/reporting_guidelines.md`。

**关键规范：**
- **CONSORT**：随机对照试验
- **STROBE**：观察性研究（队列、病例对照、横断面研究）
- **PRISMA**：系统性综述和荟萃分析
- **STARD**：诊断准确性研究
- **TRIPOD**：预测模型研究
- **ARRIVE**：动物研究
- **CARE**：病例报告
- **SQUIRE**：质量改进研究
- **SPIRIT**：临床试验方案
- **CHEERS**：经济评估

每种规范都提供了核对清单，以确保所有关键方法学要素被报告。

### 6. 写作原则与风格

应用基本的科学写作原则。有关详细指导，请参阅 `references/writing_principles.md`。

**清晰性**：
- 使用精确、无歧义的语言
- 在首次使用时定义技术术语和缩写
- 保持段落内和段落间的逻辑流畅
- 在有助于清晰时使用主动语态

**简洁性**：
- 删去冗余的词语和短语
- 倾向于较短的句子（平均 15-20 词）
- 删除不必要的修饰语
- 严格遵守字数限制

**准确性**：
- 以适当的精度报告确切数值
- 全文使用一致的术语
- 区分观察和解释
- 适当地承认不确定性

**客观性**：
- 不带偏见地呈现结果
- 避免夸大发现或推论
- 承认相互矛盾的证据
- 保持专业、中立的语气

### 7. 写作流程：从大纲到完整段落

**关键：始终以完整的段落撰写，科学论文中绝不要提交要点列表。**

科学论文必须以完整、流畅的散文书写。使用这种两阶段方法进行有效写作：

**第一阶段：创建带有要点的章节大纲**

开始新章节时：
1. 使用 research-lookup 技能收集相关文献和数据
2. 创建一个结构化大纲，用要点标注：
   - 要呈现的主要论点或发现
   - 要引用的关键研究
   - 要包含的数据点和统计数据
   - 逻辑流程和组织
3. 这些要点只是搭建用的脚手架——它们**不是**最终稿件

**示例大纲（引言部分）：**
```
- 背景：AI 在药物研发中日益受关注
  * 引用最新综述 (Smith 2023, Jones 2024)
  * 传统方法缓慢且昂贵
- 差距：在罕见病方面的应用有限
  * 仅 2 项先前研究 (Lee 2022, Chen 2023)
  * 小数据集仍是挑战
- 我们的方法：从常见病迁移学习
  * 结合 X 和 Y 的新颖架构
- 研究目标：在 3 个罕见病数据集上验证
```

**第二阶段：将要点转化为完整段落**

大纲完成后，将每个要点扩展为合适的散文：

1. **将要点转化为完整句子**，包含主语、谓语和宾语
2. **在句子和观点之间添加过渡词**（然而、而且、相比之下、随后）
3. **在句子中自然地整合引用**，而不是作为列表
4. **用要点中省略的上下文和解释进行扩展**
5. **确保每个段落内句子间的逻辑流畅**
6. **变化句子结构**以保持读者的兴趣

**转为散文的示例：**

```
在过去的十年里，人工智能方法在药物研发流程中取得了显著进展（Smith, 2023; Jones, 2024）。尽管这些计算方法显示出加速识别治疗候选药物的希望，传统的实验方法仍然缓慢且资源密集，通常需要多年的实验室工作和大量的资金投入。然而，人工智能在罕见病方面的应用仍然有限，仅有两项先前研究展示了概念验证结果（Lee, 2022; Chen, 2023）。主要障碍是针对小患者群体的疾病缺乏训练数据。

为了应对这一挑战，我们开发了一种迁移学习方法，利用已充分研究的常见病知识来预测罕见病的治疗靶点。我们新颖的神经架构结合了用于分子特征提取的卷积层和用于蛋白质-配体相互作用建模的注意力机制。本研究的目标是在三个独立的罕见病数据集上验证我们的方法，评估预测准确性和结果的生物学可解释性。
```

**大纲与最终文稿的关键区别：**

| 大纲（规划阶段） | 最终稿件 |
|--------------------------|------------------|
| 要点和片断 | 完整的句子和段落 |
| 电报式笔记 | 带上下文的完整解释 |
| 引用列表 | 整合在散文中的引用 |
| 缩略观点 | 带有过渡的展开论证 |
| 仅供自己查看 | 用于出版和同行评审 |

**常见错误避免：**

- ❌ **绝不要**在最终稿件中留有点要列表
- ❌ **绝不要**在要求段落的地方提交列表
- ❌ **不要**在结果或讨论部分使用编号或项目符号列表（特殊情况如研究假设或入排标准除外）
- ❌ **不要**写句子片段或不完整的想法
- ✅ **可以**仅在方法部分偶尔使用列表（例如，入排标准、材料列表）
- ✅ **确保**每个章节都以连贯的散文流动
- ✅ **朗读**段落以检查自然的流畅度

**列表可以接受的情况（有限情况）：**

科学论文中只有在特定情境下才可出现列表：
- **方法**：入排标准、材料和试剂、参与者特征
- **补充材料**：扩展方案、设备列表、详细参数
- **绝不出现在**：摘要、引言、结果、讨论、结论

**摘要格式规则：**
- ❌ **绝不要**使用带标签的章节（背景：、方法：、结果：、结论：）
- ✅ **始终**以流畅的段落书写，带自然的过渡
- 例外：仅在期刊的作者指南明确要求时才使用结构化格式

**与 Research Lookup 的集成：**

research-lookup 技能对第一阶段（创建大纲）至关重要：
1. 使用 research-lookup 检索相关论文
2. 提取关键发现、方法和数据
3. 将发现组织为大纲中的要点
4. 然后在第二阶段将大纲转化为完整段落

这一两阶段过程确保你：
- 系统地收集和组织信息
- 在写作前创建逻辑结构
- 生成润色的、可发表的散文
- 保持对叙事流程的专注

### 8. 专业报告格式化（非期刊文档）

对于研究报告、技术报告、白皮书以及那些**不是**期刊稿件的专业文档，使用 `scientific_report.sty` LaTeX 风格包以获得精美、专业的效果。

**何时使用专业报告格式化：**
- 研究报告和技术报告
- 白皮书和政策简报
- 基金报告和进展报告
- 行业报告和技术文档
- 内部研究总结
- 可行性研究和项目交付物

**何时不使用（改用特定场所的格式）：**
- 期刊稿件 → 使用 `venue-templates` 技能
- 会议论文 → 使用 `venue-templates` 技能
- 学位论文 → 使用机构模板

**`scientific_report.sty` 风格包提供：**

| 功能 | 描述 |
|---------|-------------|
| 字体 | Helvetica 字体族，现代专业外观 |
| 配色方案 | 专业的蓝色、绿色以及强调色 |
| 颜色框环境 | 彩色框用于关键发现、方法学、建议、局限性 |
| 表格 | 交替行色、专业表头 |
| 图片 | 统一的标题格式 |
| 科学命令 | p 值、效应量、置信区间的快捷命令 |

**用于内容组织的颜色框环境：**

```latex
% 关键发现（蓝色）- 用于主要发现
\begin{keyfindings}[标题]
包含关键发现和统计的内容。
\end{keyfindings}

% 方法学（绿色）- 用于方法重点
\begin{methodology}[研究设计]
方法和程序描述。
\end{methodology}

% 建议（紫色）- 用于行动项
\begin{recommendations}[临床意义]
\begin{enumerate}
    \item 具体建议 1
    \item 具体建议 2
\end{enumerate}
\end{recommendations}

% 局限性（橙色）- 用于需注意和告诫部分
\begin{limitations}[研究局限]
局限及其影响的描述。
\end{limitations}
```

**专业表格格式化：**

```latex
\begin{table}[htbp]
\centering
\caption{结果摘要}
\begin{tabular}{@{}lccc@{}}
\toprule
\textbf{变量} & \textbf{治疗组} & \textbf{对照组} & \textbf{p} \\
\midrule
结果 1 & \meansd{42.5}{8.3} & \meansd{35.2}{7.9} & <.001\sigthree \\
\rowcolor{tablealt} 结果 2 & \meansd{3.8}{1.2} & \meansd{3.1}{1.1} & .012\sigone \\
结果 3 & \meansd{18.2}{4.5} & \meansd{17.8}{4.2} & .58\signs \\
\bottomrule
\end{tabular}

{\small \siglegend}
\end{table}
```

**科学记号命令：**

| 命令 | 输出 | 用途 |
|---------|--------|---------|
| `\pvalue{0.023}` | *p* = 0.023 | P 值 |
| `\psig{< 0.001}` | ***p* = < 0.001** | 显著的 P 值（加粗） |
| `\CI{0.45}{0.72}` | 95% CI [0.45, 0.72] | 置信区间 |
| `\effectsize{d}{0.75}` | d = 0.75 | 效应量 |
| `\samplesize{250}` | *n* = 250 | 样本量 |
| `\meansd{42.5}{8.3}` | 42.5 ± 8.3 | 均值加标准差 |
| `\sigone`, `\sigtwo`, `\sigthree` | *, **, *** | 显著性星号 |

### 10. 领域特定语言和术语

调整语言、术语和惯例，以匹配具体的科学学科。每个领域都有既定的词汇、偏好的表达方式和领域特定的惯例，这些表明专业水平并确保目标受众的理解清晰。

**识别领域特定的语言惯例：**
- 审查目标期刊近期高影响力论文中使用的术语
- 注意领域特定的缩写、单位和记法系统
- 识别首选术语（例如，“participants”与“subjects”，“compound”与“drug”，“specimens”与“samples”）
- 观察方法、生物体或技术通常如何被描述

**生物医学和临床科学：**
- 使用精确的解剖和临床术语（例如，在正式写作中用“心肌梗死”而非“心脏病发作”）
- 遵循标准化的疾病命名法（ICD、DSM、SNOMED-CT）
- 药物名称首先使用通用名，如有需要可在括号中标注商品名
- 临床研究中使用“patients”，社区研究中用“participants”
- 基因变异遵循人类基因组变异学会（HGVS）命名法
- 以标准单位报告实验室值（大多数国际期刊使用国际单位制）

**分子生物学和遗传学：**
- 基因符号用斜体（例如，*TP53*），蛋白质用常规字体（例如，p53）
- 遵循物种特定的基因命名法（人类大写：*BRCA1*；小鼠首字母大写：*Brca1*）
- 首次提及时写出完整的生物体名称，之后使用公认的缩写（例如，*大肠杆菌*，然后*E. coli*）
- 使用标准遗传符号（例如，+/+、+/-、-/- 表示基因型）
- 使用既定的分子技术术语（例如，“定量 PCR”或“qPCR”，而非“实时 PCR”）

**化学和制药科学：**
- 遵循 IUPAC 化学化合物命名法
- 新化合物使用系统命名，熟知物质用通用名称
- 用标准记法描述化学结构（例如，用于数据库的 SMILES、InChI）
- 用适当的单位（mM、μM、nM 或 % w/v、v/v）报告浓度
- 使用公认的反应命名法描述合成路线
- 使用符合领域定义的“生物利用度”、“药代动力学”、“IC50”等术语

**生态学与环境科学：**
- 物种使用双名法（斜体：*智人*）
- 首次提及时在物种名后附上命名人（若相关）
- 使用标准化的栖息地和生态系统分类
- 对生态指标使用一致的术语（例如，“物种丰富度”、“Shannon 多样性指数”）
- 使用领域标准术语描述采样方法（例如，“样带”、“样方”、“标记重捕法”）

**物理学与工程学：**
- 除领域惯例另有要求外，始终使用国际单位制
- 使用标准记法表示物理量（标量与矢量、张量）
- 使用既定的术语描述现象（例如，“量子纠缠”、“层流”）
- 相关时指定设备型号和制造商
- 使用符合领域标准的数学记法（例如，ℏ 表示约化普朗克常数）

**神经科学：**
- 使用标准化的脑区命名法（例如，参考 Allen Brain Atlas 等脑图谱）
- 使用既定的脑立体定位系统指定脑区坐标
- 遵循神经术语惯例（例如，正式写作中用“动作电位”而非“spike”）
- 根据测量方法适当地使用“神经活动”、“神经元放电”、“大脑激活”
- 用适当的特异性描述记录技术（例如，“全细胞膜片钳”、“细胞外记录”）

**社会与行为科学：**
- 适当时使用以人为本的语言（例如，“精神分裂症患者”而非“精神分裂症病人”）
- 使用标准化的心理学构念和经过验证的评估工具名称
- 遵循 APA 减少语言偏见的指南
- 使用既定术语明确理论框架
- 对人类研究使用“participants”而非“subjects”

**一般原则：**

**匹配受众专业水平：**
- 专业期刊：自由使用领域特定术语，仅定义高度专业或新颖的术语
- 广泛影响力的期刊（如 *Nature*、*Science*）：定义更多技术术语，为专业概念提供背景
- 跨学科受众：在精确性和可理解性之间平衡，首次使用时定义术语

**策略性地定义技术术语：**
- 首次使用时定义缩写：“信使 RNA（mRNA）”
- 为更广泛的受众撰写时，对专业方法提供简要解释
- 避免过度定义目标受众熟悉的术语（表明对领域不熟悉）
- 如果不可避免使用大量专业术语，创建术语表

**保持一致性：**
- 全文对同一概念使用相同的术语（不要在“药物”、“药品”和“药剂”之间交替）
- 缩写遵循一致的系统（首次定义后，确定使用“PCR”还是“聚合酶链反应”）
- 全文应用相同的命名系统（尤其是基因、物种、化学品）

**避免领域混用错误：**
- 基础科学不使用临床术语（例如，不要称小鼠为“病人”）
- 避免用口语或过于笼统的术语代替精确的领域术语
- 在确保正确用法前，不要从相邻领域引入术语

**验证术语用法：**
- 查阅领域特定的风格指南和命名资源
- 查看目标期刊近期论文中术语的使用方式
- 使用领域特定的数据库和本体论（例如，基因本体、MeSH 词）
- 如有不确定，引用确立该术语的关键参考文献

### 11. 应避免的常见陷阱

**拒稿的主要原因：**
1. 统计方法不当、不完整或描述不足
2. 对结果的过度解读或无支撑的结论
3. 方法描述不佳，影响可重复性
4. 样本小、有偏倚或不适当
5. 写作质量差或文本难以理解
6. 文献综述或背景不足
7. 图表不清晰或设计不佳
8. 未遵循报告规范

**写作质量问题：**
- 时态混用（方法/结果使用过去时，已确立的事实使用现在时）
- 过多的行话或未定义的缩略词
- 段落分割破坏逻辑流畅性
- 章节之间缺少过渡
- 记法或术语不一致

## 稿件开发工作流程

**第一阶段：规划**
1. 确定目标期刊并审阅作者指南
2. 确定适用的报告规范（CONSORT、STROBE 等）
3. 规划稿件结构（通常为 IMRAD）
4. 将图表作为论文的骨干进行规划

**第二阶段：起草**（每个章节使用两阶段写作过程）
1. 从图表开始（核心数据故事）
2. 对于以下每个章节，遵循两阶段过程：
   - **首先**：使用 research-lookup 创建带有要点的提纲
   - **其次**：将要点转化为流畅散文的完整段落
3. 撰写方法（通常最容易先起草）
4. 起草结果（客观描述图表）
5. 撰写讨论（解读发现）
6. 撰写引言（提出研究问题）
7. 构思摘要（综合整个故事）
8. 拟定标题（简洁且描述性强）

**记住**：要点仅用于规划——最终稿件必须是完整的段落。

**第三阶段：修订**
1. 检查逻辑流程和贯穿全文的“红线”
2. 核实术语和记法的一致性
3. 确保图表是自明的
4. 确认遵守报告规范
5. 验证所有引用准确且格式正确
6. 检查每节字数
7. 校对语法、拼写和清晰度

**第四阶段：最终准备**
1. 根据期刊要求格式化
2. 准备补充材料
3. 撰写强调研究意义的投稿信
4. 完成投稿检查清单
5. 收集所有必需的声明和表格
