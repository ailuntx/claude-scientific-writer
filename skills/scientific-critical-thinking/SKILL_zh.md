# 科学批判性思维

## 概述

批判性思维是评估科学严谨性的系统性过程。使用 GRADE 和 Cochrane ROB 框架评估方法学、实验设计、统计有效性、偏倚、混杂和证据质量。将此技能应用于对科学声明的批判性分析。

## 何时使用此技能

此技能应在以下情况下使用：
- 评估研究方法学和实验设计
- 评估统计有效性和证据质量
- 识别研究中的偏倚和混杂
- 审查科学声明和结论
- 进行系统综述或元分析
- 应用 GRADE 或 Cochrane 偏倚风险评估
- 为研究论文提供批判性分析

## 核心能力

### 1. 方法学批评

评估研究方法学的严谨性、有效性和潜在缺陷。

**适用于：**
- 审查研究论文
- 评估实验设计
- 评价研究方案
- 规划新研究

**评估框架：**

1. **研究设计评估**
   - 设计是否适合研究问题？
   - 设计能否支持所做出的因果声明？
   - 比较组是否适当和充分？
   - 考虑实验性、准实验性或观察性设计是否合理

2. **有效性分析**
   - **内部有效性：** 我们能否信任因果推断？
     - 检查随机化质量
     - 评估混杂控制
     - 评估选择偏倚
     - 审查流失/退出模式
   - **外部有效性：** 结果是否可以推广？
     - 评估样本代表性
     - 考虑环境的生态有效性
     - 评估条件是否与目标应用匹配
   - **构念有效性：** 测量是否捕获了预期构念？
     - 审查测量验证
     - 检查操作定义
     - 评估测量是直接还是代理
   - **统计结论有效性：** 统计推断是否合理？
     - 验证足够的功效/样本量
     - 检查假设符合情况
     - 评估测试适当性

3. **对照和盲法**
   - 随机化是否正确实施（序列生成、分配隐藏）？
   - 盲法是否可行并已实施（参与者、提供者、评估者）？
   - 对照条件是否适当（安慰剂、阳性对照、无治疗）？
   - 表现或检测偏倚是否可能影响结果？

4. **测量质量**
   - 仪器是否经过验证且可靠？
   - 测量是否尽可能客观，或带有已知局限性的主观测量？
   - 结果评估是否标准化？
   - 是否使用多种测量来三角验证发现？

**参考：** 详细原则请参见 `references/scientific_method.md`，全面设计清单请参见 `references/experimental_design.md`。

### 2. 偏倚检测

识别和评估可能扭曲发现的潜在偏倚来源。

**适用于：**
- 审查已发表的研究
- 设计新研究
- 解释相互矛盾的证据
- 评估研究质量

**系统性偏倚审查：**

1. **认知偏倚（研究者）**
   - **确认偏倚：** 是否只突出支持性发现？
   - **HARKing：** 假设是事先陈述的还是看到结果后形成的？
   - **发表偏倚：** 阴性结果是否在文献中缺失？
   - **选择性报告：** 证据是否被选择性报告？
   - 检查预注册和分析计划透明度

2. **选择偏倚**
   - **抽样偏倚：** 样本是否代表目标人群？
   - **志愿者偏倚：** 参与者是否以系统性方式自我选择？
   - **流失偏倚：** 组间流失是否差异？
   - **生存偏倚：** 样本中是否只可见"幸存者"？
   - 检查参与者流程图并比较基线特征

3. **测量偏倚**
   - **观察者偏倚：** 期望是否可能影响观察？
   - **回忆偏倚：** 回顾性报告是否系统性不准确？
   - **社会期望：** 回答是否偏向可接受性？
   - **仪器偏倚：** 测量工具是否有系统性误差？
   - 评估盲法、验证和测量客观性

4. **分析偏倚**
   - **P值黑客：** 是否进行了多次分析直到出现显著性？
   - **结果转换：** 是否用显著结果替换非显著结果？
   - **选择性报告：** 是否报告了所有计划分析？
   - **亚组钓鱼：** 亚组分析是否未进行校正？
   - 检查研究注册并与已发表结果比较

5. **混杂**
   - 哪些变量可能同时影响暴露和结果？
   - 是否测量并控制了混杂因素（统计或通过设计）？
   - 未测量的混杂是否能解释发现？
   是否有合理的替代解释？

**参考：** 全面偏倚分类及检测和缓解策略请参见 `references/common_biases.md`。

### 3. 统计方法评估

批判性评估统计方法、解释和报告。

**适用于：**
- 审查定量研究
- 评估数据驱动的声明
- 评估临床试验结果
- 审查元分析

**统计审查清单：**

1. **样本量和功效**
   - 是否进行了先验功效分析？
   - 样本是否足以检测有意义的效应？
   - 研究是否功效不足（常见问题）？
   - 小样本的显著结果是否引发效应量膨胀的警示？

2. **统计检验**
   - 检验是否适合数据类型和分布？
   - 是否检查并满足检验假设？
   - 参数检验是否合理，是否应使用非参数替代？
   - 分析是否与研究设计匹配（如配对 vs 独立）？

3. **多重比较**
   - 是否检验了多个假设？
   - 是否应用了校正（Bonferroni、FDR、其他）？
   - 主要结果是否与次要/探索性结果区分？
   - 发现是否可能因多重检验而成为假阳性？

4. **P值解释**
   - P值是否被正确解释（如果零假设为真，数据的概率）？
   - 非显著性是否被错误解释为"无效应"？
   - 统计显著性是否与实际重要性混为一谈？
   - 是否报告了精确P值，还是仅"p < .05"？
   - 是否可疑地聚集在 .05 下方？

5. **效应量和置信区间**
   - 是否随显著性报告效应量？
   - 是否提供置信区间以显示精确度？
   - 效应量在实际上是否有意义？
   - 标准化效应量是否结合领域特定背景解释？

6. **缺失数据**
   - 缺失了多少数据？
   - 是否考虑了缺失数据机制（MCAR、MAR、MNAR）？
   - 如何处理缺失数据（删除、插补、最大似然）？
   - 缺失数据是否可能导致结果偏倚？

7. **回归和建模**
   - 模型是否过拟合（预测变量过多、无交叉验证）？
   - 是否在数据范围外进行预测（外推）？
   - 是否解决了多重共线性问题？
   - 是否检查了模型假设？

8. **常见陷阱**
   - 将相关性视为因果关系
   - 忽略回归到均值
   - 忽视基础率
   - 德州神枪手谬误（在噪声中找模式）
   - 辛普森悖论（亚组混杂）

**详细陷阱和正确实践请参见 `references/statistical_pitfalls.md`。

### 4. 证据质量评估

系统性评估证据的强度和质量。

**适用于：**
- 为决策权衡证据
- 进行文献综述
- 比较相互矛盾的发现
- 确定对结论的信心

**证据评估框架：**

1. **研究设计层级**
   - 系统综述/元分析（干预效果最高）
   - 随机对照试验
   - 队列研究
   - 病例对照研究
   - 横断面研究
   - 病例系列/报告
   - 专家意见（最低）

   **重要提示：** 更高层级的设计并非总是更好质量。设计良好的观察性研究可能比做得不好的 RCT 更强。

2. **设计类型内的质量**
   - 偏倚风险评估（使用适当工具：Cochrane ROB、Newcastle-Ottawa 等）
   - 方法学严谨性
   - 透明度和报告完整性
   - 利益冲突

3. **GRADE 注意事项（如适用）**
   - 从设计类型开始（RCT = 高，观察性 = 低）
   - **降级原因：**
     - 偏倚风险
     - 研究间不一致性
     - 间接性（错误的人群/干预/结果）
     - 不精确（宽置信区间、小样本）
     - 发表偏倚
   - **升级原因：**
     - 大效应量
     - 剂量-反应关系
     - 混杂因素会减少（而非增加）效应

4. **证据一致性**
   - **更强当：**
     - 多个独立重复
     - 不同研究小组和设置
     - 不同方法论得出相同结论
     - 机制和经验证据一致
   - **更弱当：**
     - 单个研究或研究小组
     - 文献中相互矛盾的发现
     - 明显的发表偏倚
     - 无重复尝试

5. **情境因素**
   - 生物学/理论合理性
   - 与已有知识的一致性
   - 时间性（原因先于结果）
   - 关系的特异性
   - 关联强度

**详细层级、GRADE 系统和质量评估工具请参见 `references/evidence_hierarchy.md`。

### 5. 逻辑谬误识别

检测并命名科学论证和声明中的逻辑错误。

**适用于：**
- 评估科学声明
- 审查讨论/结论部分
- 评估科普交流
- 识别有缺陷的推理

**科学中常见的谬误：**

1. **因果谬误**
   - **后此谬误：** "B 跟在 A 后面，所以 A 导致了 B"
   - **相关性 = 因果关系：** 将关联与因果关系混淆
   - **反向因果关系：** 将原因误认为结果
   - **单一原因谬误：** 将复杂结果归因于一个因素

2. **概括谬误**
   - **草率概括：** 从小样本得出广泛结论
   - **轶事谬误：** 以个人故事作为证据
   - **选择性报告：** 仅选择支持性证据
   - **生态谬误：** 将群体模式应用于个人

3. **权威和来源谬误**
   - **诉诸权威：** "专家说的，所以是真的"（无证据）
   - **人身攻击：** 攻击人而非论点
   - **起源谬误：** 根据来源而非优点判断
   - **诉诸自然：** "自然 = 好/安全"

4. **统计谬误**
   - **基础率忽视：** 忽略先验概率
   - **德州神枪手：** 在随机数据中找模式
   - **多重比较：** 不对多重检验进行校正
   - **检方谬误：** 将 P(E|H) 与 P(H|E) 混淆

5. **结构性谬误**
   - **错误二分法：** "要么 A 要么 B"而存在更多选项
   - **移动球门：** 达到标准后改变证据标准
   - **循环论证：** 乞求问题
   - **稻草人：** 歪曲论点以便攻击

6. **科学特有谬误**
   - **伽利略 gambit：** "他们嘲笑伽利略，所以我的非主流观点是正确的"
   - **无知论证：** "未被证伪，所以是真的"
   - **完美主义谬误：** 拒绝不完美的解决方案
   - **不可证伪性：** 做出不可检验的声明

**识别谬误时：**
- 命名具体谬误
- 解释推理为何有缺陷
- 识别需要什么证据才能有效推断
- 注意谬误推理并不能证明结论为假——只是这个论点不支持它

**全面谬误目录及示例和检测策略请参见 `references/logical_fallacies.md`。

### 6. 研究设计指导

为规划严谨研究提供建设性指导。

**适用于：**
- 帮助设计新实验
- 规划研究项目
- 审查研究提案
- 改进研究方案

**设计流程：**

1. **研究问题提炼**
   - 确保问题具体、可回答且可证伪
   - 验证它解决了文献中的空白或矛盾
   - 确认可行性（资源、伦理、时间）
   - 对变量进行操作定义

2. **设计选择**
   - 将设计与问题匹配（因果 → 实验；关联 → 观察性）
   - 考虑可行性和伦理约束
   - 选择组间、组内或混合设计
   - 如需测试多个因素则计划析因设计

3. **偏倚最小化策略**
   - 尽可能实施随机化
   - 在所有可行水平计划盲法（参与者、提供者、评估者）
   - 识别并计划控制混杂因素（随机化、匹配、分层、统计调整）
   - 标准化所有程序
   - 计划最小化流失

4. **样本规划**
   - 进行先验功效分析（指定预期效应、 desired power、alpha）
   - 在样本量中考虑流失
   - 明确纳入/排除标准
   - 考虑招募策略和可行性
   - 计划样本代表性

5. **测量策略**
   - 选择经过验证、可靠的仪器
   - 尽可能使用客观测量
   - 计划对关键构念进行多次测量（三角验证）
   - 确保测量对预期变化敏感
   - 建立评分者间信度程序

6. **分析规划**
   - 预先指定所有假设和分析
   - 明确指定主要结果
   - 计划统计检验并检查假设
   - 缺失数据将如何处理
   - 计划报告效应量和置信区间
   - 考虑多重比较校正

7. **透明度和严谨性**
   - 预注册研究和分析计划
   - 使用报告指南（CONSORT、STROBE、PRISMA）
   - 计划报告所有结果，而不仅仅是显著结果
   - 区分确认性分析和探索性分析
   - 承诺数据/代码共享

**从问题到传播的全面设计清单请参见 `references/experimental_design.md`。

### 7. 声明评估

系统性评估科学声明的有效性和支持度。

**适用于：**
- 评估论文中的结论
- 评估媒体对研究的报道
- 审查摘要或引言中的声明
- 检查数据是否支持结论

**声明评估流程：**

1. **识别声明**
   - 到底声明了什么？
   - 是因果声明、关联声明还是描述性声明？
   - 声明的强度如何（已证实、可能、建议、可能）？

2. **评估证据**
   - 提供了什么证据？
   - 证据是直接还是间接？
   - 证据是否足以支持声明的强度？
   - 是否排除了替代解释？

3. **检查逻辑联系**
   - 结论是否从数据得出？
   - 是否有逻辑跳跃？
   - 是否用关联数据支持因果声明？
   - 是否承认局限性？

4. **评估比例性**
   - 信心是否与证据强度成比例？
   - 是否适当使用模糊措辞？
   - 是否淡化局限性？
   - 是否明确标注推测？

5. **检查过度泛化**
   - 声明是否超出研究样本？
   - 是否承认人群限制？
   - 是否认识到情境依赖性？
   - 是否包含关于泛化的注意事项？

6. **警示信号**
   - 来自关联研究的因果语言
   - "证明"或绝对确定性
   - 选择性引用
   - 忽略矛盾证据
   - 轻视局限性
   - 数据外推

**提供具体反馈：**
- 引用有问题的声明
- 解释需要什么证据来支持它
- 如有需要，建议适当的模糊措辞
- 区分数据（发现了什么）和解释（意味着什么）

## 应用指南

### 一般方法

1. **保持建设性**
   - 识别优点和缺点
   - 建议改进而非仅批评
   - 区分致命缺陷和轻微局限性
   - 认识到所有研究都有局限性

2. **保持具体**
   - 指向具体实例（例如"表 2 显示..."或"在方法部分..."）
   - 引用有问题的陈述
   - 提供问题的具体示例
   - 引用具体违反的原则或标准

3. **保持比例**
   - 批评严重程度与问题重要性匹配
   - 区分对有效性的主要威胁和次要关注
   - 考虑问题是否影响主要结论
   - 承认自己评估中的不确定性

4. **应用一致标准**
   - 对所有研究使用相同标准
   - 不要对不喜欢的发现应用更严格的标准
   - 承认自己的潜在偏倚
   - 根据方法论而非结果做出判断

5. **考虑情境**
   - 承认实际和伦理约束
   - 考虑领域特定的效应量和方法规范
   - 认识到探索性 vs 确认性情境
   - 在评估研究时考虑资源限制

### 提供批评时

**按以下结构组织反馈：**

1. **摘要：** 简要概述评估内容
2. **优点：** 做得好的是什么（对可信度和学习很重要）
3. **担忧：** 按严重程度组织的问题
   - 关键问题（威胁主要结论的有效性）
   - 重要问题（影响解释但不致命）
   - 轻微问题（值得注意但不改变结论）
4. **具体建议：** 可操作的改进建议
5. **总体评估：** 关于证据质量及可得出结论的平衡结论

**使用精确术语：**
- 命名具体偏倚、谬误和方法学问题
- 引用既定标准和指南
- 引用科学方法论原则
- 准确使用技术术语

### 当不确定时

- **承认不确定性：** "这可能是 X 或 Y；需要额外信息 Z"
- **提出澄清问题：** "是否完成了[方法学细节]？这影响解释。"
- **提供条件性评估：** "如果做了 X，则 Y 成立；如果没做，则 Z 是问题"
- **注意什么额外信息可以解决不确定性**

## 记住

**科学批判性思维是关于：**
- 使用既定原则进行系统性评估
- 建设性批评以改进科学
- 信心与证据强度成比例
- 对不确定性和局限性的透明
- 标准的一致应用
- 认识到所有研究都有局限性
- 怀疑与对证据开放之间的平衡

**始终区分：**
- 数据（观察到的是什么）和解释（意味着什么）
- 相关性和因果关系
- 统计显著性和实际重要性
- 探索性和确认性发现
- 已知和不确定的
- 反对某声明的证据和支持零假设的证据

**批判性思维的目标：**
1. 准确识别优点和缺点
2. 确定哪些结论有证据支持
3. 认识到局限性和不确定性
4. 为未来工作提出改进建议
5. 促进科学理解
