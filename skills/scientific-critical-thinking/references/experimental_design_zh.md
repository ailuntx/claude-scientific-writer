# 实验设计核查清单

## 研究问题的提出

### 问题是否已明确形成？
- [ ] **具体性：** 变量和关系定义清晰
- [ ] **可回答性：** 可以用现有方法来解决
- [ ] **相关性：** 解决知识或实际需求中的空白
- [ ] **可行性：** 资源、时间和伦理考量允许
- [ ] **可证伪性：** 如果不正确可以被证明是错的

### 您是否已查阅文献？
- [ ] 确定已知的内容
- [ ] 发现需要解决的空白或矛盾
- [ ] 从方法论的成功和失败中学习
- [ ] 确定适当的结果指标
- [ ] 确定该领域典型的效应量

## 假设开发

### 您的假设是否可检验？
- [ ] 做出具体的、可量化的预测
- [ ] 变量已被操作定义
- [ ] 指定预期关系的方向/性质
- [ ] 可以被潜在的观察所证伪

### 假设类型
- [ ] **零假设（H₀）：** 不存在效应/关系
- [ ] **备择假设（H₁）：** 存在效应/关系
- [ ] **定向与非定向：** 单尾检验与双尾检验

## 研究设计选择

### 什么类型的研究是适当的？

**实验（干预）研究：**
- [ ] **随机对照试验（RCT）：** 因果关系的金标准
- [ ] **准实验：** 非随机分配但有操纵
- [ ] **被试内：** 相同参与者在所有条件下
- [ ] **被试间：** 每个条件下的参与者不同
- [ ] **多因素：** 多个自变量
- [ ] **交叉设计：** 参与者按顺序接受多个干预

**观察性研究：**
- [ ] **队列：** 随时间追踪组
- [ ] **病例对照：** 比较有结果和无结果的人
- [ ] **横断面：** 一个时间点的快照
- [ ] **生态学：** 人群水平数据

**考虑：**
- [ ] 您能随机分配参与者吗？
- [ ] 您能操纵自变量吗？
- [ ] 结果是罕见（支持病例对照）还是常见？
- [ ] 您需要建立时间顺序吗？
- [ ] 在伦理和实践约束下什么是可行的？

## 变量

### 自变量（操纵/预测变量）
- [ ] 定义清晰并操作化
- [ ] 选择了适当的水平/类别
- [ ] 操纵足以检验假设
- [ ] 计划了操纵检查（如适用）

### 因变量（结果/反应变量）
- [ ] 直接测量感兴趣的构念
- [ ] 已验证且测量可靠
- [ ] 足够敏感以检测预期效应
- [ ] 适合 planned 统计分析
- [ ] 主要结果已明确指定

### 控制变量
- [ ] **已识别混淆变量：**
  - 影响自变量和因变量的变量
  - 发现的替代解释
- [ ] **控制策略：**
  - 随机化
  - 匹配
  - 分层
  - 统计调整
  - 限制（纳入/排除标准）
  - 盲法

### 无关变量
- [ ] 已识别潜在的噪声来源
- [ ] 标准化程序以最小化
- [ ] 控制环境因素
- [ ] 标准化时间、地点、设备

## 抽样

### 人群定义
- [ ] **目标人群：** 您想推广至的人群
- [ ] **可及人群：** 您实际可以从中抽取的人群
- [ ] **样本：** 实际参与的群体
- [ ] 记录这些之间的差异

### 抽样方法
- [ ] **概率抽样（有利于推广的首选）：**
  - 简单随机抽样
  - 分层抽样
  - 整群抽样
  - 系统抽样
- [ ] **非概率抽样（常见但限制推广性）：**
  - 便利抽样
  - 目的性抽样
  - 雪球抽样
  - 配额抽样

### 样本量
- [ ] **进行了先验功效分析**
  - 预期效应量（来自文献或预试验）
  - 期望功效（通常 .80 或 .90）
  - 显著性水平（通常 .05）
  - 要使用的统计检验
- [ ] 考虑预期脱落/流失
- [ ] 足够进行计划的中层分析
- [ ] 承认实际约束

### 纳入/排除标准
- [ ] 定义清晰且有理由
- [ ] 不过度限制（限制推广性）
- [ ] 基于理论或实际考量
- [ ] 已解决伦理考量
- [ ] 一致地记录和应用

## 盲法和随机化

### 随机化
- [ ] **随机化的内容：**
  - 参与者分配到条件
  - 条件顺序（被试内）
  - 呈现的刺激/项目
- [ ] **随机化方法：**
  - 计算机生成的随机数
  - 随机数表
  - 掷硬币（对于非常小的研究）
- [ ] **分配隐藏：**
  - 在招募前生成顺序
  - 分配在入组后隐藏
  - 顺序编号、密封信封（如需要）
- [ ] **分层随机化：**
  - 跨组平衡重要变量
  - 区块随机化以确保组大小相等
- [ ] **检查随机化：**
  - 在基线时比较组
  - 报告任何显著差异

### 盲法
- [ ] **单盲：** 参与者不知道组分配
- [ ] **双盲：** 参与者和研究者都不知道
- [ ] **三盲：** 参与者、研究者和数据分析师都不知道
- [ ] **盲法可行性：**
  - 真正的盲法是否可能？
  - 需要安慰剂/假对照？
  - 干预外观相同？
- [ ] **盲法检查：**
  - 评估盲法是否保持
  - 让参与者/研究者猜测分配

## 对照组和条件

### 什么类型的对照？
- [ ] **无治疗对照：** 条件的自然过程
- [ ] **安慰剂对照：** 惰性治疗用于比较
- [ ] **主动对照：** 标准治疗比较
- [ ] **等待列表对照：** 延迟治疗
- [ ] **注意对照：** 匹配接触时间但无活性成分

### 多条件
- [ ] 多因素的析因设计
- [ ] 剂量-反应关系评估
- [ ] 成分分析进行机制检验

## 程序

### 方案开发
- [ ] **详细的书面积方案：**
  - 分步程序
  - 标准化说明文本
  - 处理问题的决策规则
  - 数据收集表
- [ ] 在主要研究前进行预试验
- [ ] 工作人员培训至标准
- [ ] 计划依从性监测

### 标准化
- [ ] 对所有参与者相同的说明
- [ ] 相同的设备和材料
- [ ] 尽可能相同的环境/设置
- [ ] 相同的评估时间
- [ ] 记录与方案的偏差

### 数据收集
- [ ] **收集时间：**
  - 基线测量
  - 干预后
  - 随访时间点
- [ ] **谁收集：**
  - 训练有素的研究者
  - 尽可能盲法
  - 建立评分者间信度
- [ ] **如何收集：**
  - 有效、可靠的工具
  - 标准化管理
  - 尽可能使用多种方法（三角验证）

## 测量

### 效度
- [ ] **表面效度：** 看起来测量该构念
- [ ] **内容效度：** 涵盖构念的所有方面
- [ ] **效标效度：** 与金标准相关
  - 同时效度
  - 预测效度
- [ ] **构念效度：** 测量理论构念
  - 聚敛效度（与相关测量相关）
  - 区分效度（与不相关测量不相关）

### 信度
- [ ] **重测信度：** 随时间一致
- [ ] **内部一致性：** 项目测量相同构念（Cronbach's α）
- [ ] **评分者间信度：** 评分者之间的一致（Cohen's κ，ICC）
- [ ] **平行形式：** 替代版本一致

### 测量考量
- [ ] 尽可能使用客观测量
- [ ] 使用已验证的工具
- [ ] 对关键构念使用多种测量
- [ ] 考虑对变化的敏感性
- [ ] 避免地板/天花板效应
- [ ] 反应格式适当
- [ ] 回忆期限适当
- [ ] 考虑文化适切性

## 偏差最小化

### 选择偏差
- [ ] 尽可能随机抽样
- [ ] 明确界定资格标准
- [ ] 记录谁拒绝以及原因
- [ ] 最小化自我选择

### 实施偏差
- [ ] 标准化方案
- [ ] 提供者盲法
- [ ] 监测方案依从性
- [ ] 记录偏差

### 检测偏差
- [ ] 结果评估者盲法
- [ ] 尽可能使用客观测量
- [ ] 标准化评估程序
- [ ] 多个评分者进行信度检查

### 脱落偏差
- [ ] 最小化退出的策略
- [ ] 追踪退出原因
- [ ] 比较退出者和完成者
- [ ] 计划意向治疗分析

### 报告偏差
- [ ] 预注册研究和分析计划
- [ ] 指定主要与次要结果
- [ ] 承诺报告所有结果
- [ ] 区分计划与探索性分析

## 数据管理

### 数据收集
- [ ] 数据收集表格设计和测试
- [ ] REDCap、Qualtrics 或类似平台
- [ ] 范围检查和验证规则
- [ ] 定期备份
- [ ] 安全存储（如果需要符合HIPAA/GDPR）

### 数据质量
- [ ] 实时数据验证
- [ ] 定期质量检查
- [ ] 监测缺失数据模式
- [ ] 识别和处理异常值
- [ ] 记录方案偏差

### 数据安全
- [ ] 去识别化程序
- [ ] 访问控制
- [ ] 审计追踪
- [ ] 符合法规（IRB、HIPAA、GDPR）

## 统计分析计划

### 分析计划（在数据收集前预先指定）
- [ ] **主要分析：**
  - 指定统计检验
  - 明确陈述假设
  - 设置显著性水平（通常 α = .05）
  - 单尾或双尾
- [ ] **次要分析：**
  - 明确指定为次要
  - 标记为探索性分析
- [ ] **多重比较：**
  - 指定调整方法（如需要）
  - 主要结果防止膨胀

### 假设
- [ ] 已识别统计检验的假设
  - 检查假设的计划
  - 备用非参数替代方案
  - 考虑转换选项

### 缺失数据
- [ ] 预期的缺失量
- [ ] 缺失数据机制（MCAR、MAR、MNAR）
- [ ] 处理策略：
  - 完全案例分析
  - 多重插补
  - 最大似然
- [ ] 计划敏感性分析

### 效应量
- [ ] 已确定适当的效应量指标
  - 将与p值一起报告
  - 计划置信区间

### 统计软件
- [ ] 选择软件（R、SPSS、Stata、Python等）
- [ ] 记录版本
- [ ] 提前准备分析脚本
  - 将可用（开放科学）

## 伦理考量

### 伦理批准
- [ ] 获得IRB/伦理委员会批准
- [ ] 如适用，注册研究（ClinicalTrials.gov等）
- [ ] 方案遵循赫尔辛基宣言或同等文件

### 知情同意
- [ ] 自愿参与
- [ ] 可理解的解释
- [ ] 披露风险和收益
- [ ] 无惩罚退出权利
- [ ] 解释隐私保护
- [ ] 披露补偿

### 风险-收益分析
- [ ] 潜在收益超过风险
- [ ] 最小化风险
- [ ] 保护弱势群体
- [ ] 数据安全监测（如高风险）

### 保密性
- [ ] 数据去识别化
- [ ] 安全存储
- [ ] 限制访问
- [ ] 报告不允许重新识别

## 效度威胁

### 内部效度（因果关系）
- [ ] **历史：** 测量之间的外部事件
- [ ] **成熟：** 参与者随时间的变化
- [ ] **测试：** 重复测量的效应
- [ ] **工具：** 随时间测量的变化
- [ ] **回归至均值：** 极端分数变得不那么极端
- [ ] **选择：** 组在基线时不同
- [ ] **脱落：** 差异性流失
- [ ] **扩散：** 对照组接受治疗元素

### 外部效度（推广性）
- [ ] 样本代表人群
- [ ] 设置现实/自然
- [ ] 治疗典型真实世界实施
- [ ] 结果测量生态有效
- [ ] 时间框架适当

### 构念效度（测量）
- [ ] 测量实际触及预期构念
- [ ] 操作与理论定义匹配
- [ ] 没有构念混淆
- [ ] 构念覆盖充分

### 统计结论效度
- [ ] 统计功效足够
- [ ] 满足假设
- [ ] 使用适当的检验
- [ ] Alpha水平适当
- [ ] 解决多重比较

## 报告和透明度

### 预注册
- [ ] 研究预注册（OSF、ClinicalTrials.gov、AsPredicted）
- [ ] 先验陈述假设
- [ ] 记录分析计划
- [ ] 区分验证性与探索性

### 报告指南
- [ ] **RCTs：** CONSORT核查清单
- [ ] **观察性研究：** STROBE核查清单
- [ ] **系统综述：** PRISMA核查清单
- [ ] **诊断研究：** STARD核查清单
- [ ] **定性研究：** COREQ核查清单
- [ ] **病例报告：** CARE指南

### 透明度
- [ ] 报告所有测量
- [ ] 披露所有操纵
- [ ] 解释样本量确定
- [ ] 报告排除标准和数量
- [ ] 记录流失
- [ ] 记录与方案的偏差
- [ ] 披露利益冲突

### 开放科学
- [ ] 计划数据共享（如伦理允许）
- [ ] 共享分析代码
- [ ] 可用材料
- [ ] 发布预印本
- [ ] 可能时开放获取出版

## 研究后考量

### 数据分析
- [ ] 遵循预注册计划
- [ ] 明确标记偏差和探索性分析
- [ ] 检查假设
- [ ] 报告所有结果
- [ ] 报告效应量和置信区间，而不仅仅是p值

### 解读
- [ ] 结论有数据支持
- [ ] 承认局限性
- [ ] 考虑替代解释
- [ ] 讨论推广性
- [ ] 解决临床/实际意义

### 传播
- [ ] 无论结果如何都发表（减少发表偏差）
- [ ] 在会议上展示
- [ ] 与参与者分享发现（如适当）
- [ ] 与相关利益相关者沟通
- [ ] 简明语言摘要

### 下一步
- [ ] 需要复制？
- [ ] 已确定后续研究
- [ ] 计划机制研究
- [ ] 考虑临床应用

## 要避免的常见陷阱

- [ ] 无功效分析 → 功效不足的研究
- [ ] 在看到数据后形成假设（HARKing）
- [ ] 可行时无盲法 → 偏差
- [ ] P黑客（数据钓鱼，可选停止）
- [ ] 无校正的多重检验 → 假阳性
- [ ] 对照组不足
- [ ] 未解决混淆
- [ ] 工具未验证
- [ ] 未解决高流失
- [ ]  cherry-picking 结果报告
- [ ] 相关数据使用因果语言
- [ ] 忽略统计检验的假设
- [ ] 不预注册改变文献偏差
- [ ] 未披露利益冲突

## 开始前的最终核查清单

- [ ] 研究问题清晰且重要
- [ ] 假设可检验且具体
- [ ] 研究设计适当
- [ ] 样本量足够（功效分析）
- [ ] 测量有效且可靠
- [ ] 控制混淆
- [ ] 实施随机化和盲法
- [ ] 数据收集标准化
- [ ] 分析计划预先指定
- [ ] 获得伦理批准
- [ ] 研究已预注册
- [ ] 资源充足
- [ ] 团队已培训
- [ ] 方案已记录
- [ ] 问题有备份计划

## 记住

**好的实验设计是关于：**
- 提出清晰的问题
- 最小化偏差
- 最大化效度
- 适当的推断
- 透明度
- 可重复性

**思考这些问题的最佳时间是在收集数据之前，而不是之后。**
