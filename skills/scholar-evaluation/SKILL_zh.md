# 学者评估

## 概述

应用 ScholarEval 框架对学术和研究工作进行系统评估。这项技能提供基于同行评审研究评估标准的结构化评估方法，能够对学术论文、研究提案、文献综述和学术写作进行多维度的全面分析。

## 何时使用此技能

在以下情况下使用此技能：
- 评估研究论文的质量和严谨性
- 评估文献综述的全面性和质量
- 审查研究方法设计
- 评分数据分析方法
- 评估学术写作和展示
- 对学术工作提供结构化反馈
- 根据既定标准对研究质量进行基准测试
- 评估目标出版物的发表就绪状态
- 提供定量评估以补充定性同行评审

## 评估工作流程

### 步骤 1：初始评估和范围定义

首先确定要评估的学术工作类型和评估范围：

**工作类型：**
- 完整研究论文（实证、理论或综述）
- 研究提案或方案
- 文献综述（系统性、叙事性或范围性）
- 论文或学位论文章节
- 会议摘要或短篇论文

**评估范围：**
- 全面（所有维度）
- 针对性（特定方面，如方法论或写作）
- 对比性（与其他工作进行基准测试）

如果范围不明确，请请用户澄清。

### 步骤 2：基于维度的评估

在 ScholarEval 维度上系统评估工作。对于每个适用的维度，评估质量、识别优势和劣势，并在适用时提供评分。

有关每个维度的详细标准和评分规则，请参阅 `references/evaluation_framework.md`。

**核心评估维度：**

1. **问题表述与研究问题**
   - 研究问题的清晰度和具体性
   - 理论或实践意义
   - 可行性和范围适当性
   - 创新性和贡献潜力

2. **文献综述**
   - 覆盖的全面性
   - 批判性综合与单纯总结
   - 研究差距的识别
   - 来源的时效性和相关性
   - 适当的上下文定位

3. **方法论与研究设计**
   - 对研究问题的适当性
   - 严谨性和有效性
   - 可重复性和透明度
   - 伦理考量
   - 局限性承认

4. **数据收集与来源**
   - 数据的质量和适当性
   - 样本量和代表性
   - 数据收集程序
   - 来源的可信度和可靠性

5. **分析与解释**
   - 分析方法的适当性
   - 分析的严谨性
   - 逻辑一致性
   - 考虑替代解释
   - 结果与主张的一致性

6. **结果与发现**
   - 呈现的清晰度
   - 统计或定性严谨性
   - 可视化质量
   - 解释的准确性
   - 含义讨论

7. **学术写作与展示**
   - 清晰度和组织
   - 学术语气和风格
   - 语法和表达
   - 逻辑流程
   - 对目标受众的可访问性

8. **引用与参考文献**
   - 引用的完整性
   - 来源的质量和适当性
   - 引用的准确性
   - 观点的平衡
   - 对引用标准的遵循

### 步骤 3：评分和评级

对于每个评估的维度，提供：

**定性评估：**
- 主要优势（2-3 个具体要点）
- 改进领域（2-3 个具体要点）
- 关键问题（如有）

**定量评分（可选）：**
在适用时使用 5 分制：
- 5：优秀 - 模范质量，可在顶级出版物发表
- 4：良好 - 强劲质量，需要少量改进
- 3：合格 - 可接受质量，有明显的改进领域
- 2：需要改进 - 需要重大修订
- 1：差 - 存在需要重大修订的基本问题

要程序化计算总体评分，请使用 `scripts/calculate_scores.py`。

### 步骤 4：综合整体评估

提供综合评估总结：

1. **整体质量评估** - 对作品学术价值的整体判断
2. **主要优势** - 跨维度的 3-5 个主要优势
3. **关键劣势** - 需要关注的 3-5 个主要领域
4. **优先建议** - 按影响力排序的改进列表
5. **发表就绪状态**（如适用） - 对目标出版物的适用性评估

### 步骤 5：提供可操作的反馈

将评估发现转化为建设性、可操作的反馈：

**反馈结构：**
- **具体** - 引用具体章节、段落或页码
- **可操作** - 提供具体的改进建议
- **优先排序** - 按重要性和可行性对建议进行排序
- **平衡** - 在解决劣势的同时承认优势
- **基于证据** - 将反馈基于评估标准

**反馈格式选项：**
- 按维度进行分析的结构化报告
- 映射到具体文档部分的注释
- 包含主要发现和建议的执行摘要
- 针对基准标准的对比分析

### 步骤 6：情境考量

根据以下情况调整评估方法：

**发展阶段：**
- 早期草稿：关注概念和结构问题
- 高级草稿：关注完善和润色
- 最终提交：全面质量检查

**目的和场所：**
- 期刊文章：对严谨性和贡献的高标准
- 会议论文：平衡创新性与展示清晰度
- 学生作业：具有发展重点的教育反馈
- 资助提案：强调可行性和影响力

**学科特定规范：**
- STEM 领域：强调可重复性和统计严谨性
- 社会科学：平衡定量和定性标准
- 人文学科：关注论点论证和学术解释

## 资源

### references/evaluation_framework.md

每个 ScholarEval 维度的详细评估标准、评分规则和质量指标。在进行评估时加载此参考，以访问特定的评估指南和评分规则。

快速访问的搜索模式：
- "问题表述标准"
- "文献综述评分规则"
- "方法论评估"
- "数据质量指标"
- "分析严谨性标准"
- "写作质量清单"

### scripts/calculate_scores.py

用于从维度级别评分计算总体评估评分的 Python 脚本。支持加权平均、阈值分析和分数可视化。

用法：
```bash
python scripts/calculate_scores.py --scores <dimension_scores.json> --output <report.txt>
```

## 最佳实践

1. **保持客观性** - 基于既定标准进行评估，而非个人偏好
2. **全面评估** - 系统评估所有适用维度
3. **提供证据** - 用作品中的具体例子支持评估
4. **保持建设性** - 将劣势转化为改进机会
5. **考虑情境** - 根据工作阶段和目的调整期望
6. **记录理由** - 解释评估和评分背后的推理
7. **鼓励优势** - 明确承认作品的优点
8. **优先反馈** - 首先关注高影响力的改进

## 评估工作流程示例

**用户请求：** "评估这篇关于用于药物发现的机器学习研究论文"

**响应流程：**
1. 识别工作类型（实证研究论文）和范围（全面评估）
2. 加载 `references/evaluation_framework.md` 以获取详细标准
3. 系统评估每个维度：
   - 问题表述：关于 ML 模型性能的清晰研究问题
   - 文献综述：对近期 ML 和药物发现工作的全面覆盖
   - 方法论：适当的深度学习架构及验证程序
   - [继续评估所有维度...]
4. 计算维度评分和整体评估
5. 将发现综合成结构化报告，突出：
   - 方法论强项和可重复代码
   - 需要更多样化的数据集评估
   - 写作可在结果部分提高清晰度
6. 提供带有具体建议的优先排序建议

## 与科学作家的集成

此技能可与科学作家工作流程无缝集成：

**论文生成后：**
- 使用学者评估作为同行评审的替代或补充
- 生成 `SCHOLAR_EVALUATION.md` 以及 `PEER_REVIEW.md`
- 提供定量评分以跟踪修订过程中的改进

**修订期间：**
- 解决反馈后重新评估特定维度
- 跟踪多个版本的分数改进
- 识别需要关注的持续性劣势

**发表准备：**
- 评估目标期刊/会议的就绪状态
- 提交前识别差距
- 对照发表标准进行基准测试

## 注释

- 评估的严谨性应与作品的目的和阶段相匹配
- 某些维度可能不适用于所有工作类型（例如，纯理论论文的数据收集）
- 应考虑文化和学科在学术规范方面的差异
- 此框架补充而非替代领域专业知识
- 与同行评审技能结合使用，以进行综合评估

## 引用

此技能基于以下文献中介绍的 ScholarEval 框架：

**Moussa, H. N., Da Silva, P. Q., Adu-Ampratwum, D., East, A., Lu, Z., Puccetti, N., Xue, M., Sun, H., Majumder, B. P., & Kumar, S. (2025).** _ScholarEval: Research Idea Evaluation Grounded in Literature_. arXiv 预印本 arXiv:2510.16234. [https://arxiv.org/abs/2510.16234](https://arxiv.org/abs/2510.16234)

**摘要：** ScholarEval 是一个检索增强评估框架，基于两个基本标准评估研究想法：可靠性（基于现有文献提出的方法的实证有效性）和贡献（相对于先前研究在不同维度上推进的程度）。该框架实现了对专家标注评估点的高得多的覆盖率，并且在评估可操作性、深度和证据支持方面始终优于基线系统。
