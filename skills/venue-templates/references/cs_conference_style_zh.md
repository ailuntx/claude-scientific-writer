# CS会议论文写作指南

针对ACL、EMNLP、NAACL（NLP）、CHI、CSCW（HCI）、SIGKDD、WWW、SIGIR（数据挖掘/信息检索）以及其他主要计算机科学会议的全面写作指南。

**最后更新**：2024

---

## 概述

计算机科学会议涵盖不同的子领域，各自有着独特的写作文化。本指南涵盖NLP、HCI和数据挖掘/信息检索这三个领域，每个领域都有独特的期望和评估标准。

---

# 第一部分：NLP会议（ACL、EMNLP、NAACL）

## NLP写作理念

> "在标准基准数据集上取得强有力的实证结果，并提供有见地的分析。"

NLP论文需要在实证严谨性与语言学洞察之间取得平衡。随着自动指标日益重要，人类评估也越来越受到重视。

## 读者与语气

### 目标读者
- NLP研究人员和计算语言学家
- 熟悉Transformer架构和标准基准数据集
- 期望可复现的结果和错误分析

### 语气特点

| 特点 | 描述 |
|---------------|-------------|
| **任务导向** | 清晰的问题定义 |
| **基准导向** | 强调标准数据集 |
| **分析丰富** | 错误分析、定性示例 |
| **可复现** | 完整的实现细节 |

## 摘要（NLP风格）

### 结构
- **任务/问题**（1句话）
- **先前工作的局限性**（1句话）
- **你的方法**（1-2句话）
- **基准数据集上的结果**（2句话）
- **分析发现**（可选，1句话）

### 示例摘要

```
共指消解对于具有遥远或歧义先行词的代词仍然具有挑战性。先前的神经
方法由于有限的上下文建模能力，在处理这些困难案例时表现不佳。我们
提出LongContext-Coref，一种检索增强的共指模型，可从文档历史中动态
检索相关上下文。在OntoNotes 5.0基准数据集上，LongContext-Coref达到
83.4 F1分数，比之前的最新方法提高了1.2分。在具有挑战性的WinoBias数据
集上，我们将性别偏见减少34%，同时保持准确率。定性分析表明，我们的模
型成功解决了需要世界知识的代词消解，这是先前方法的已知弱点。
```

## NLP论文结构

```
├── 引言
│   ├── 任务动机
│   ├── 先前工作的局限性
│   ├── 你的贡献
│   └── 贡献要点
├── 相关工作
├── 方法
│   ├── 问题形式化
│   ├── 模型架构
│   └── 训练过程
├── 实验
│   ├── 数据集（包含统计信息）
│   ├── 基线方法
│   ├── 主要结果
│   ├── 分析
│   │   ├── 错误分析
│   │   ├── 消融实验
│   │   └── 定性示例
│   └── 人类评估（如适用）
├── 讨论/局限性
└── 结论
```

## NLP特定要求

### 数据集
- 使用**标准基准数据集**：GLUE、SQuAD、CoNLL、OntoNotes
- 报告**数据集统计信息**：训练/开发/测试集规模
- **数据预处理**：记录所有步骤

### 评估指标
- **任务适当的指标**：F1、BLEU、ROUGE、准确率
- **统计显著性**：成对bootstrap检验、p值
- **多次运行**：报告不同随机种子下的均值±标准差

### 人类评估

对于生成任务越来越受到重视：
- **标注者详细信息**：人数、资质、一致性
- **评估协议**：指南、界面、报酬
- **标注者间一致性**：Cohen's κ或Krippendorff's α

### 人类评估表示例

```
表3：人类评估结果（100个样本，3个标注者）
─────────────────────────────────────────────────────────────
方法        | 流利性 | 连贯性 | 事实性 | 总体评分
─────────────────────────────────────────────────────────────
基线方法    |   3.8   |    3.2    |    3.5     |   3.5
GPT-3.5     |   4.2   |    4.0    |    3.7     |   4.0
我们的方法  |   4.4   |    4.3    |    4.1     |   4.3
─────────────────────────────────────────────────────────────
标注者间κ = 0.72。评分标准：1-5（分数越高越好）。
```

## ACL特定说明

- **ARR（ACL滚动审查）**：跨ACL场所的共享审查系统
- **负责任NLP清单**：伦理、局限性、风险
- **长文（8页） vs 短文（4页）**：不同的期望
- **Findings论文**：较低层次的接受轨道

# 第二部分：HCI会议（CHI、CSCW、UIST）

## HCI写作理念

> "技术为人服务——首先理解用户，然后设计和评估。"

HCI论文本质上是**以用户为中心**的。单纯的技术创新是不够的；理解人类需求并展示用户收益至关重要。

## 读者与语气

### 目标读者
- HCI研究人员和实践者
- 用户体验设计师和产品开发人员
- 跨学科（计算机科学、心理学、设计、社会科学）

### 语气特点

| 特点 | 描述 |
|---------------|-------------|
| **以用户为中心** | 关注人，而非技术 |
| **设计导向** | 根植于设计思维 |
| **实证性** | 用户研究提供证据 |
| **反思性** | 考虑更广泛的影响 |

## HCI摘要

### 关注用户和影响

```
视频通话已成为远程协作的重要组成部分，但现有界面无法很好地支持
使现场工作有效的周边感知。通过与24名远程工作者进行形成性访谈，
我们确定了三个关键挑战：难以判断同事 availability、缺乏环境存在
线索，以及中断焦虑。我们设计了AmbientOffice，一种周边显示系统，
可通过微妙的环境可视化传达队友的存在。在与三个分布式团队的18名
参与者进行的为期两周的部署研究中，AmbientOffice将自发协作增加了
40%，并减少了感知到的孤独感（p<0.01）。参与者重视该系统的非侵入性，
并表示感觉与远程同事的联系更紧密。我们讨论了设计环境感知系统的
含义以及远程工作中可见性与隐私之间的紧张关系。
```

## HCI论文结构

### 通过设计进行研究 / 系统论文

```
├── 引言
│   ├── 以人类术语表述的问题
│   ├── 技术如何提供帮助
│   └── 贡献总结
├── 相关工作
│   ├── 领域背景
│   ├── 先前系统
│   └── 理论框架
├── 形成性工作（通常）
│   ├── 访谈/观察
│   └── 设计需求
├── 系统设计
│   ├── 设计依据
│   ├── 实现
│   └── 界面演示
├── 评估
│   ├── 研究设计
│   ├── 参与者
│   ├── 过程
│   ├── 发现（定量+定性）
│   └── 局限性
├── 讨论
│   ├── 设计含义
│   ├── 可推广性
│   └── 未来工作
└── 结论
```

### 定性/访谈研究

```
├── 引言
├── 相关工作
├── 方法
│   ├── 参与者
│   ├── 过程
│   ├── 数据收集
│   └── 分析方法（主题分析、扎根理论等）
├── 发现
│   ├── 主题1（附带引用）
│   ├── 主题2（附带引用）
│   └── 主题3（附带引用）
├── 讨论
│   ├── 设计含义
│   ├── 研究含义
│   └── 局限性
└── 结论
```

## HCI特定要求

### 参与者报告
- **人口统计信息**：年龄、性别、相关经验
- **招募方式**：如何及在哪里招募
- **补偿**：报酬金额和类型
- **IRB批准**：伦理委员会声明

### 发现中的引用

使用直接引用来支撑发现：
```
参与者重视显示的环境特性。正如P7所描述：
"就像有一扇可以看到我队友办公室的窗户。我不需要
主动查看，但我知道他们在那里。"这种被动感知
降低了联系他人的障碍。
```

### 设计含义部分

将发现转化为可操作的指导：
```
**含义1：支持周边感知而无需占用注意力。**
环境显示应该在周边视觉中可见，但不需要主动监控。设计师应
该考虑平静技术原则。

**含义2：平衡可见性与隐私。**
用户想要分享存在状态，但担心被监视。系统应该提供
细粒度控制，使可见性成为双向的。
```

## CHI特定说明

- **贡献类型**：实证、artifact、方法论、理论
- **ACM格式**：`acmart`文档类配合`sigchi`选项
- **可访问性**：期望有替代文本、包容性语言
- **贡献声明**：必需按作者列出贡献

# 第三部分：数据挖掘与信息检索（SIGKDD、WWW、SIGIR）

## 数据挖掘写作理念

> "用于现实世界数据的可扩展方法，并展示实际影响。"

数据挖掘论文强调**可扩展性**、**实际应用性**和**坚实的实验方法论**。

## 读者与语气

### 目标读者
- 数据科学家和机器学习工程师
- 行业研究人员
- 应用机器学习实践者

### 语气特点

| 特点 | 描述 |
|---------------|-------------|
| **可扩展** | 处理大型数据集 |
| **实用性** | 现实应用 |
| **可复现** | 分享数据集和代码 |
| **工业导向** | 重视行业数据集 |

## KDD摘要

### 强调规模和应用

```
电子商务欺诈检测需要在实时处理数百万笔交易的同时适应不断变化的
攻击模式。我们提出FraudShield，一个用于实时欺诈检测的图神经网络
框架，可扩展到十亿边的交易图。与需要完整图访问的先前方法不同，
FraudShield使用增量更新，每笔交易的推理成本为O(1)。在来自大型
电子商务平台的23亿笔交易 proprietary数据集上，FraudShield在
80%召回率下达到94.2%的精确率，比生产基线提高12%。该系统已在
[公司]部署，每秒处理50K笔交易，防止估计每年4亿美元的欺诈损失。
我们发布了匿名化基准数据集和代码。
```

## KDD论文结构

```
├── 引言
│   ├── 问题与影响
│   ├── 技术挑战
│   ├── 你的方法
│   └── 贡献
├── 相关工作
├── 预备知识
│   ├── 问题定义
│   └── 符号表示
├── 方法
│   ├── 概述
│   ├── 技术组件
│   └── 复杂度分析
├── 实验
│   ├── 数据集（包含规模统计）
│   ├── 基线方法
│   ├── 主要结果
│   ├── 可扩展性实验
│   ├── 消融实验
│   └── 案例研究/部署
└── 结论
```

## KDD特定要求

### 可扩展性
- **数据集规模**：报告节点、边、样本数量
- **运行时间分析**：wall-clock时间比较
- **复杂度**：时间和空间复杂度说明
- **扩展实验**：展示性能与数据规模的关系

### 工业部署
- **案例研究**：真实部署故事
- **A/B测试**：在线评估结果（如适用）
- **生产指标**：商业影响（如可分享）

```

──────────────────────────────────────────────────────

Citeseer    |  3.3K  |  4.7K  |  0.4  |    0.3    |  0.1

ogbn-arxiv  | 169K   |  1.17M |  8.4  |    4.2    |  1.6
```

# 第四部分：各CS会议的共同要素

## 写作质量

### 清晰性
- **每句话一个想法**
- **使用前先定义术语**
- **使用一致的符号表示**

### 精确性
- **精确数字**："23.4%"而非"约20%"
- **清晰的声明**：除非必要，否则避免模糊表达
- **具体比较**：说出基线方法名称

## 贡献要点

各CS会议通用：
```
我们的贡献是：
• 我们识别了[问题/洞察]
• 我们提出了[方法名称]，[关键创新点]
• 我们在[基准数据集]上展示了[结果]
• 我们在[URL]发布了[代码/数据]
```

## 可复现性标准

各CS会议越来越期望：
- **代码可用性**：GitHub链接（审查期间匿名）
- **数据可用性**：公共数据集或发布计划
- **完整超参数**：训练细节完整
- **随机种子**：用于复现的确切值

## 伦理与更广泛影响

### NLP（ACL/EMNLP）
- **局限性部分**：必需
- **负责任NLP清单**：伦理考虑
- **偏见分析**：用于影响人的模型

### HCI（CHI）
- **IRB/伦理批准**：人类受试者必需
- **知情同意**：描述程序
- **隐私考虑**：数据处理

### KDD/WWW
- **社会影响**：考虑滥用可能性
- **隐私保护**：敏感数据
- **公平性分析**：适用时

| 方面 | ACL/EMNLP | CHI | KDD/WWW | SIGIR |
|--------|-----------|-----|---------|-------|
| **重点** | NLP任务 | 用户研究 | 可扩展ML | IR/搜索 |
| **评估** | 基准+人类 | 用户研究 | 大规模实验 | 数据集 |

| **审查风格** | ARR | 直接 | 直接 | 直接 |
