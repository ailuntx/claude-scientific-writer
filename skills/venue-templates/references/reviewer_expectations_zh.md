# 各期刊/会议的审稿人期望

了解不同期刊/会议的审稿人关注什么，对于编写成功的投稿材料至关重要。本指南涵盖评估标准、常见拒稿原因以及如何回应审稿人的意见。

**最后更新**：2024年

---

## 概述

不同期刊/会议的审稿人优先考虑的事项各不相同。了解这些优先项有助于您：

1. 恰当地呈现您的贡献
2. 预见可能的批评
3. 准备有效的反驳
4. 决定投稿目标

---

## 高影响力期刊（Nature、Science、Cell）

### 审稿人关注的内容

| 优先级 | 权重 | 描述 |
|----------|--------|-------------|
| **广泛意义** | 关键 | 对特定子领域之外的影响 |
| **创新性** | 关键 | 首次展示或重大进展 |
| **技术严谨性** | 高 | 合理的方法论、适当的对照 |
| **清晰度** | 高 | 非专业人员也能理解 |
| **完整性** | 中等 | 全面但不冗长 |

### 审稿流程

1. **编辑初筛**：大多数稿件在未经审稿时被拒稿（Nature：约92%）
2. **专家审稿**：如送出审稿，则为2-4位审稿人
3. **跨学科审稿人**：通常包含非专业人士
4. **快速反馈**：首次决定通常在2-4周内

### 什么会导致稿件被拒稿

**编辑阶段**：

- 研究发现对广泛读者群体意义不够
- 相对于先前工作的进展较为有限
- 对该期刊而言过于专业化
- 主题不符合当前编辑兴趣

**审稿阶段**：

- 声称的内容未被数据支持
- 缺少关键对照
- 未讨论替代解释
- 统计方面存在问题
- 未充分承认先前的工作
- 写作风格让非专业人员难以理解

### 如何回应Nature/Science审稿人

**在稿件中**：

- 在第一段即指出意义
- 解释研究发现为何具有广泛影响
- 为所有主要论点包含对照
- 使用清晰、易懂的语言
- 包含概念性图表

**在反驳中**：

- 回应每一个问题（即使是小问题）
- 按要求提供新数据
- 优雅地承认合理的批评
- 如被质疑则解释意义

### 审稿人意见示例及回应

**审稿人**：「这项工作的意义对普通读者来说不清楚。」

**回应**：「我们已修改引言部分以阐明更广泛的意义。如第一段所述，我们的研究发现对[X]有影响，因为[Y]。我们还增加了讨论，说明这些结果如何帮助理解[Z]（第8页，第15-28行）。」

## 医学期刊（NEJM、Lancet、JAMA）

### 审稿人关注的内容

| 优先级 | 权重 | 描述 |
|----------|--------|-------------|
| **临床相关性** | 关键 | 这会改变实践吗？ |
| **方法论严谨性** | 关键 | 符合CONSORT/STROBE |
| **患者结局** | 关键 | 关注对患者重要的事项 |
| **统计有效性** | 高 | 适当的分析、统计效力 |
| **可推广性** | 高 | 适用于更广泛的群体 |

### 审稿流程

1. **统计审查**：专门的统计审稿人很常见
2. **临床专业知识**：亚专科专家
3. **方法论审查**：重点关注研究设计
4. **多轮修订**：经常要求修改

### 什么会导致稿件被拒稿

**主要问题**：

- 统计效力不足的研究
- 不适当的对照/比较
- 未解决混杂因素
- 选择性报告结局
- 缺少安全性数据
- 声称超出证据

**中等问题**：

- 可推广性不明确
- 缺少亚组分析
- CONSORT/STROBE报告不完整
- 统计方法描述不充分

### 审稿人意见示例及回应

**审稿人**：「该研究似乎对主要结局统计效力不足。考虑到200名参与者和5%的事件率，没有足够的统计效力来检测具有临床意义的差异。」

**回应**：「我们感谢这一意见。我们的效力计算（方法部分，第5页）基于对照组5%的事件率和50%的相对减少（至2.5%）。虽然观察到的事件率（4.8%）接近预期，但我们承认置信区间很宽（HR 0.65，95% CI 0.38-1.12）。我们已将此作为局限性添加（讨论部分，第12行）。重要的是，效应的大小和方向与更大的XYZ试验（n=5000）一致，表明我们的发现值得在更大的研究中验证。」

## Cell Press系列期刊

### 审稿人关注的内容

| 优先级 | 权重 | 描述 |
|----------|--------|-------------|
| **机制见解** | 关键 | 这是如何工作的？ |
| **研究深度** | 关键 | 多种方法、全面覆盖 |
| **生物学意义** | 高 | 对领域的重要性 |
| **技术严谨性** | 高 | 量化、统计、重复验证 |
| **创新性** | 中等至高 | 新发现，而不仅仅是验证 |

### 审稿流程

1. **扩展审稿**：通常3位以上审稿人
2. **修订轮次**：常见多轮修订
3. **全面修订**：经常要求重要的新实验
4. **详细评估**：逐图评估

### 审稿人的期望

- **多种互补方法**：同一发现用不同方式展示
- **体内验证**：对于细胞生物学论点
- **拯救实验**：对于敲低/敲除研究
- **量化数据**：不仅仅是代表性图片
- **完整的图表**：所有条件、所有对照

### 审稿人意见示例及回应

**审稿人**：「作者展示蛋白质X使用siRNA敲低是过程Y所必需的。然而，使用了单一的RNAi试剂，无法排除脱靶效应。需要更多证据。」

**回应**：「我们同意额外验证很重要。在修订后的手稿中，我们现在展示：（1）两个针对蛋白质X的独立siRNA产生相同的表型（新图S3A-B）；（2）CRISPR-Cas9敲除细胞重现了表型（新图2D-E）；（3）表达耐siRNA的蛋白质X拯救了表型（新图2F-G）。这些互补方法强力支持蛋白质X是过程Y所必需的结论。」

## ML会议（NeurIPS、ICML、ICLR）

### 审稿人关注的内容

| 优先级 | 权重 | 描述 |
|----------|--------|-------------|
| **创新性** | 关键 | 新方法、新见解或新视角 |
| **技术正确性** | 关键 | 正确的实现、公平的比较 |
| **意义** | 高 | 推动领域发展 |
| **实验严谨性** | 强有力的基线、适当的消融实验 |
| **可复现性** | 中等至高 | 其他人能否复现？ |
| **清晰度** | 中等 | 写作良好且组织有序 |

### 审稿流程

1. **领域主席分配**：按主题分组
2. **3-4位审稿人**：具有该领域的专业知识
3. **作者反驳**：回应审稿意见的机会
4. **审稿人讨论**：反驳后
5. **AC建议**：元审稿

### 评分维度

典型的NeurIPS/ICML评分：

| 维度 | 评分范围 | 评估内容 |
|-----------|-------------|------------------|
| **正确性** | 1-4 | 技术正确性 |
| **贡献** | 1-4 | 结果的意义 |
| **呈现** | 1-4 | 清晰度和组织 |
| **总体** | 1-10 | 整体评估 |
| **信心** | 1-5 | 审稿人专业性 |

### 什么会导致稿件被拒稿

**关键问题**：

- 弱基线或不公平的比较
- 缺少消融实验
- 结果未显著优于SOTA
- 方法或分析中的技术错误
- 无证据的夸大声称

**中等问题**：

- 相对于先前工作的创新性有限
- 评估范围窄（少数据集/任务）
- 缺少可复现性细节
- 呈现不佳
- 分析或见解有限

### ML审稿人的警示信号

❌「我们与2018年的方法进行比较」（基线过时）
❌「我们的方法实现了0.5%的提升」（边际收益）
❌「我们在一个数据集上评估」（可推广性有限）
❌「实现细节在补充材料中」（核心信息缺失）
❌「我们将消融实验留待未来工作」（评估不完整）

### 审稿人意见示例及回应

**审稿人**：「提出的方法仅与Transformer和Performer进行了比较。应该包含最近的FlashAttention和Longformer。」

**回应**：「感谢这一建议。我们已添加与FlashAttention（Dao等人，2022）、Longformer（Beltagy等人，2020）和BigBird（Zaheer等人，2020）的比较。如新表2所示，我们的方法优于所有基线：FlashAttention（差3.2%）、Longformer（差5.1%）和BigBird（差4.8%）。我们还包括了新分析（第4.3节），解释我们的方法为什么对超过16K标记的序列特别有效。」

## HCI会议（CHI、CSCW）

### 审稿人关注的内容

| 优先级 | 权重 | 描述 |
|----------|--------|-------------|
| **对HCI的贡献** | 关键 | 新设计、新见解或新方法 |
| **以用户为中心的方法** | 高 | 关注人类需求 |
| **适当的评估** | 高 | 与声称和贡献相匹配 |
| **设计理由** | 中等至高 | 有理有据的设计决策 |
| **影响** | 中等 | 指导未来工作 |

### 贡献类型

CHI明确分类贡献：

| 类型 | 审稿人期望 |
|------|----------------------|
| **实证** | 严格的用户研究、清晰的发现 |
| **成果** | 新系统/工具、使用的评估 |
| **方法论** | 新研究方法、验证 |
| **理论** | 概念框架、学术贡献 |
| **综述** | 全面、组织良好的覆盖 |

### 什么会导致稿件被拒稿

**关键问题**：

- 声称与评估不匹配
- 参与人数不足以得出结论
- 缺少伦理考虑（无IRB）
- 专注于技术而没有用户洞察
- 对HCI社区的贡献有限

**中等问题**：

- 设计理由薄弱
- 可推广性有限
- 缺少HCI相关工作
- 对实践者的影响不明确

### 审稿人意见示例及回应

**审稿人**：「评估包括一个由12名参与者进行的短期实验室研究。不清楚该系统在实际使用中长期表现如何。」

**回应**：「我们承认这一局限性，现在明确讨论（第7.2节）。我们添加了来自原始队列的8名参与者的2周部署研究（新第6.3节）。这些纵向数据显示持续参与（平均使用：每天4.2次），并揭示了使用模式如何随时间演变的额外见解。然而，我们同意更大和更长的部署将加强生态效度。」

## NLP会议（ACL、EMNLP）

### 审稿人关注的内容

| 优先级 | 权重 | 描述 |
|----------|--------|-------------|
| **任务性能** | 高 | SOTA或竞争性结果 |
| **分析质量** | 高 | 错误分析、见解 |
| **方法论** | 高 | 合理的方法、公平的比较 |
| **可复现性** | 高 | 提供完整细节 |
| **创新性** | 中等至高 | 新方法或见解 |

### ACL滚动审稿（ARR）

自2022年起，ACL期刊使用共享审稿系统：

- 审稿在期刊间转移
- 行动编辑管理稿件
- 审稿后承诺投递特定期刊

### 负责任的NLP清单

审稿人检查：

- 局限性部分（必填）
- 风险和伦理考虑
- 计算/碳足迹
- 偏倚分析（如适用）
- 数据文档

### 审稿人意见示例及回应

**审稿人**：「论文缺少对失败案例的分析。提出的方法何时以及为何失败？」

**回应**：「我们已添加第5.4节关于错误分析。我们手动检查了100个错误，并将它们分为三类：（1）复杂的共指链（42%），（2）隐式引用（31%），和（3）领域特定知识要求（27%）。图4展示了每类的代表性例子。这一分析表明，我们的方法特别难以处理隐式引用，我们将其作为未来工作的方向讨论。」

## 数据挖掘（KDD、WWW）

### 审稿人关注的内容

| 优先级 | 权重 | 描述 |
|----------|--------|-------------|
| **可扩展性** | 高 | 处理大型数据集 |
| **实际影响** | 高 | 实际适用性 |
| **实验严谨性** | 高 | 全面评估 |
| **技术创新性** | 中等至高 | 新方法或应用 |
| **可复现性** | 中等 | 代码/数据可用性 |

### 令KDD审稿人印象深刻的内容

- 大规模实验（数百万样本）
- 行业部署或A/B测试
- 效率比较（运行时间、内存）
- 真实数据集与基准一起
- 复杂性分析（时间和空间）

### 审稿人意见示例及回应

**审稿人**：「实验仅限于小数据集（<100K样本）。该方法如何扩展到行业规模的数据？」

**回应**：「我们添加了两个大规模数据集上的实验：（1）ogbn-papers100M（1.11亿节点，16亿边）和（2）由[公司]提供的专有电子商务图（5亿节点，40亿边）。新表4显示我们的方法随数据大小近乎线性扩展，在ogbn-papers上42分钟内完成，而基线则内存不足。新第5.5节提供了详细的可扩展性分析。」

## 通用反驳策略

### 应该做

✅ **回应每一个问题**：即使是小问题
✅ **提供证据**：新实验、数据或引用
✅ **具体明确**：参考具体部分、行数、图表
✅ **承认合理的批评**：表明您理解该问题
✅ **简洁明了**：审稿人阅读很多反驳
✅ **保持专业**：即使对于不公正的审稿
✅ **优先处理关键问题**：先回应主要关切

### 不应该做

❌ **防御性**：接受合理的批评
❌ **无证据地争论**：支持声称
❌ **忽略问题**：即使您不同意的
❌ **模糊不清**：具体说明更改
❌ **攻击审稿人**：保持专业
❌ **承诺未来工作**：如果可能，现在就做

### 反驳模板

```
感谢审稿人建设性的反馈。我们回应以下主要关切：

**R1/R2关切：[多位审稿人的共同关切]**

[您的回应，包括具体采取的行动和修订手稿中更改的参考]

**R1-1：[具体问题]**

[带证据的回应]

**R2-3：[具体问题]**

[带证据的回应]

我们还做了以下额外改进：
• [改进1]
• [改进2]
```

## 投稿前自我审查

在投稿前，像审稿人一样审查您的论文：

### 所有期刊/会议

- [ ] 声称是否有证据支持？
- [ ] 基线是否适当且是最近的？
- [ ] 贡献是否明确说明？
- [ ] 局限性是否承认？
- [ ] 可复现性信息是否完整？

### 高影响力期刊

- [ ] 意义对非专业人士是否清晰？
- [ ] 图表是否易于理解？
- [ ] 对声称的对照是否充分？

### 医学期刊

- [ ] CONSORT/STROBE合规是否完整？
- [ ] 是否报告了绝对数字？
- [ ] 临床相关性是否清晰？

### ML会议

- [ ] 消融实验是否全面？
- [ ] 比较是否公平？
- [ ] 可复现性信息是否完整？

### HCI会议

- [ ] 以用户为中心的视角是否清晰？
- [ ] 评估是否与声称相称？
- [ ] 设计影响是否可操作？
