# 场地写作风格：大师指南

本指南概述了写作风格在不同发表场地间的差异。理解这些差异对于撰写符合各场地真实出版风格的文章至关重要。

**最后更新**：2024

---

## 风格光谱

科学写作风格存在于从**广泛可及**到**深度技术**的光谱中：

```
可及 ◄─────────────────────────────────────────────► 技术

Nature/Science    PNAS    Cell    IEEE Trans    NeurIPS    专业
   │                │       │         │            │         期刊
   │                │       │         │            │            │
   ▼                ▼       ▼         ▼            ▼            ▼
普通             混合    深度    领域      密集ML      仅限
受众             深度   生物     专家      研究者      专家
```

## 快速风格参考

| 场地类型 | 受众 | 语气 | 人称 | 摘要风格 |
|------------|----------|------|-------|----------------|
| **Nature/Science** | 受过教育的非专业人士 | 可及性强，引人入胜 | 主动，允许第一人称 | 流畅段落，无术语 |
| **Cell Press** | 生物学家 | 机制性，精确 | 混合 | 摘要 + eTOC简介 + 亮点 |
| **医学 (NEJM/Lancet)** | 临床医生 | 证据导向 | 正式 | 结构化（背景/方法/结果/结论） |
| **PLOS/BMC** | 研究人员 | 标准学术 | 中性 | IMRaD结构化或流畅 |
| **IEEE/ACM** | 工程师/计算机科学 | 技术性 | 被动常见 | 简洁，技术性 |
| **ML会议** | ML研究人员 | 密集技术 | 混合 | 数字优先，关键结果 |
| **NLP会议** | NLP研究人员 | 技术性 | 多样 | 任务导向，基准 |

## 高影响力期刊（Nature、Science、Cell）

### 核心理念

高影响力多学科期刊重视**广泛意义**而非技术深度。问题不是"这在技术上是否可靠？"而是"为什么该领域外的科学家应该关心？"

### 关键写作原则

1. **从宏观图景开始**：开篇阐述为什么这对科学/社会重要
2. **尽量减少术语**：定义专业术语；首选常用词
3. **讲述故事**：结果应作为叙事流动，而非数据倾倒
4. **强调启示**：这改变了我们对什么的理解？
5. **可及的图表**：原理图和模型优先于原始数据图

### 结构差异

**Nature/Science** vs. **专业期刊**：

| 元素 | Nature/Science | 专业期刊 |
|---------|---------------|---------------------|
| 引言 | 3-4段，广泛 → 具体 | 广泛文献综述 |
| 方法 | 常在补充材料或简要介绍中 | 正文详述 |
| 结果 | 按发现/故事组织 | 按实验组织 |
| 讨论 | 先讲启示，后讲局限 | 与文献详细比较 |
| 图表 | 重视概念原理图 | 强调原始数据 |

### 示例：同一发现，不同风格

**Nature风格**：
> "我们发现蛋白X在发育过程中作为控制细胞命运决定的分子开关，解决了关于干细胞如何选择其命运这一长期存在的问题。"

**专业期刊风格**：
> "利用小鼠胚胎干细胞（mESCs）中的CRISPR-Cas9敲除，我们证明蛋白X（由基因ABC1编码）通过直接启动子结合调节多能性因子Oct4、Sox2和Nanog的表达，经ChIP-seq分析确认（n=3个生物学重复，FDR < 0.05）。"

## 医学期刊（NEJM、Lancet、JAMA、BMJ）

### 核心理念

医学期刊重视**临床相关性**和**患者结局**。每个发现都必须与实践相关联。

### 关键写作原则

1. **以患者为中心的语言**：使用"接受X治疗的患者"而非"X治疗受试者"
2. **证据强度**：根据研究设计谨慎措辞
3. **临床可操作性**：对执业医师来说"那又怎样？"
4. **绝对数字**：报告绝对风险降低，而不仅是相对风险
5. **结构化摘要**：需要带标签部分

### 结构化摘要格式（医学）

```
背景：[1-2句话问题与原理]

方法：[研究设计、地点、参与者、干预、结局、分析]

结果：[主要结局及置信区间，次要结局，不良事件]

结论：[临床启示，承认局限]
```

### 证据语言惯例

| 研究设计 | 适当语言 |
|-------------|---------------------|
| 随机对照试验 | "X治疗降低了死亡率..." |
| 观察性研究 | "X治疗与死亡率降低相关..." |
| 病例系列 | "这些发现提示X治疗可能..." |
| 病例报告 | "本病例说明X治疗可以..." |

## ML/AI会议（NeurIPS、ICML、ICLR、CVPR）

### 核心理念

ML会议重视**创新性**、**严格实验**和**可复现性**。重点在于用实证证据推进技术 state-of-the-art。

### 关键写作原则

1. **贡献要点**：引言中列出编号清单，明确说明创新之处
2. **基线至关重要**：与强劲、近期基线比较
3. **消融实验预期**：展示方法的哪些部分重要
4. **可复现性**：随机种子、超参数、计算要求
5. **局限部分**：诚实承认（越来越必要）

### 引言结构（ML会议）

```
[第1段：问题动机 - 为什么重要]

[第2段：现有方法的局限]

[第3段：我们的方法概览]

我们的贡献如下：
• 我们提出[方法名]，一种解决[问题]的新方法，其[关键创新]。
• 我们提供理论分析，证明[保证/性质]。
• 我们在[基准]上展示最先进结果，比[基线]提高[X%]。
• 我们在[匿名审查URL]发布代码和模型。
```

### 摘要风格（ML会议）

ML摘要**密集且数字聚焦**：

> "我们提出TransformerX，一种用于长程序列建模的新架构，在保持表达力的同时实现O(n log n)复杂度。在Long Range Arena基准上，TransformerX达到86.2%的平均准确率，优于Transformer（65.4%）和Performer（78.1%）。在语言建模上，TransformerX使用少40%的参数达到GPT-2困惑度（18.4）。我们提供的理论分析表明TransformerX可以近似任何连续序列到序列函数。"

### 实验部分期望

1. **数据集**：标准基准，数据集统计
2. **基线**：近期强劲方法，公平比较
3. **主结果表**：清晰，全面
4. **消融研究**：系统地移除/修改组件
5. **分析**：错误分析，定性示例，失败案例
6. **计算成本**：训练时间，推理速度，内存

## CS会议（ACL、EMNLP、CHI、SIGKDD）

### ACL/EMNLP（NLP）

- **任务导向**：清晰的问题定义
- **基准密集**：标准数据集（GLUE、SQuAD等）
- **重视错误分析**：它在哪里失败？
- **人类评估**：通常期望与自动指标一起
- **伦理考虑**：偏见、公平、环境成本

### CHI（人机交互）

- **以用户为中心**：关注人，而不仅是技术
- **研究设计细节**：参与者招募，IRB批准
- **接受定性研究**：访谈研究、民族志有效
- **设计启示**：从业者的具体要点
- **可及性**：考虑多元用户群体

### SIGKDD（数据挖掘）

- **强调可扩展性**：处理大数据
- **真实世界应用**：工业数据集受重视
- **效率指标**：时间和空间复杂度
- **方法或应用创新**：两条路径都有效

## 在场地类型间适应

### 期刊 → ML会议

将期刊论文转换为会议格式时：

1. **精简引言**：移除广泛背景
2. **添加贡献列表**：明确列举贡献
3. **重组结果**：按实验组织，添加消融实验
4. **移除单独讨论**：简要整合解读
5. **添加可复现性部分**：随机种子、超参数、代码

### ML会议 → 期刊

将会议论文扩展为期刊时：

1. **扩展相关工作**：全面文献综述
2. **详细方法**：完整算法描述
3. **更多实验**：额外数据集、分析
4. **扩展讨论**：启示、局限、未来工作
5. **附录 → 正文**：将重要细节移上

### 专业 → 高影响力期刊

从专业场地 targeting Nature/Science/Cell时：

1. **以重要性引领**：为什么这广泛重要？
2. **减少80%术语**：替换技术术语
3. **添加概念图表**：原理图、模型，不仅是数据
4. **故事驱动结果**：叙事流动，而非逐个实验
5. **扩展讨论**：超越子领域的启示

## 语气和风格指南

### 主动 vs. 被动语态

| 场地 | 偏好 | 示例 |
|-------|-----------|---------|
| Nature/Science | 鼓励主动 | "我们发现..." |
| Cell | 混合 | "我们的结果证明..." |
| 医学 | 常见被动 | "患者被随机分配到..." |
| IEEE | 传统被动 | "该算法被实现..." |
| ML会议 | 首选主动 | "我们提出一种方法..." |

### 第一人称使用

| 场地 | 第一人称 | 示例 |
|-------|-------------|---------|
| Nature/Science | 是（我们） | "我们表明..." |
| Cell | 是（我们） | "我们发现..." |
| 医学 | 有时 | "我们进行了一项试验..." |
| IEEE | 较少见 | 倾向"本文提出..." |
| ML会议 | 是（我们） | "我们介绍..." |

### 措辞与确定性

| 声明强度 | 语言 |
|---------------|----------|
| 强 | "X导致Y"（仅在有因果证据时） |
| 中等 | "X与Y相关" / "X导致Y" |
| 试探性 | "X可能促成Y" / "X提示..." |
| 推测性 | "X可能是可能的..." / "一种解释是..." |

## 各场地常见风格错误

### Nature/Science 提交

❌ 过于技术："我们使用靶向外显子3的CRISPR-Cas9和sgRNAs..."
✅ 可及："利用基因编辑技术，我们禁用了该基因..."

❌ 枯燥开头："蛋白X参与细胞信号传导..."
✅ 引人入胜开头："细胞如何决定自己的命运？我们发现..."

### ML会议 提交

❌ 模糊贡献："我们提出一种X的新方法"
✅ 具体贡献："我们提出在基准W上实现Z%改进的方法Y"

❌ 缺少消融实验：仅展示完整方法结果
✅ 完整：展示各组件贡献的表格

### 医学期刊 提交

❌ 缺少绝对数字："风险降低50%"
✅ 完整："相对降低50%（绝对风险降低2.5%，需治数40）"

❌ 观察性数据使用因果语言："治疗导致改善"
✅ 适当："治疗与改善相关"

## 提交前快速检查清单

### 所有场地
- [ ] 摘要符合场地风格（流畅 vs. 结构化）
- [ ] 语气适合受众
- [ ] 术语水平适当
- [ ] 图表符合场地期望
- [ ] 引用格式正确

### 高影响力期刊（Nature/Science/Cell）
- [ ] 首段明确显示广泛意义
- [ ] 非专业人士能理解摘要
- [ ] 故事驱动的结果叙事
- [ ] 包含概念图表
- [ ] 强调启示

### ML会议
- [ ] 引言中有贡献列表
- [ ] 包含强劲基线
- [ ] 存在消融研究
- [ ] 可复现性信息完整
- [ ] 承认局限

### 医学期刊
- [ ] 结构化摘要（如需要）
- [ ] 以患者为中心的语言
- [ ] 证据强度适当
- [ ] 报告绝对数字
- [ ] 符合CONSORT/STROBE
