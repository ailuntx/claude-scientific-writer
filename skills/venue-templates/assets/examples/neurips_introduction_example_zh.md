# NeurIPS/ICML 引言示例

本示例展示了ML会议独特的引言结构，包含带编号的贡献和技术细节。

---

## 完整引言示例

**论文主题**：高效长上下文Transformer

---

### 第一段：问题动机

```
大型语言模型（LLM）在自然语言理解、代码生成和推理任务中展现了卓越的能力[1, 2, 3]。
这些能力随着模型规模和上下文长度而扩展——更长的上下文能够处理整个文档、多轮对话
和跨越多个步骤的复杂推理链[4, 5]。然而，标准Transformer注意力机制[6]相对于序列长度N
具有O(N²)的时间和内存复杂度，这为处理长序列造成了根本性瓶颈。对于100K token的上下文
窗口，仅注意力矩阵就需要100亿次标量运算和40 GB内存，使得在当前硬件上进行训练和推理
成本高昂。
```

**关键特征**：
- 阐明其重要性（LLM能力）
- 关联到扩展性（更长上下文=更好性能）
- 具体数值（O(N²)，100K tokens，100亿次运算，40 GB）
- 引用以建立可信度

---

### 第二段：现有方法的局限性

```
先前的工作通过三种主要方法来解决注意力效率问题。稀疏注意力模式[7, 8, 9]通过限制
注意力到局部窗口、固定步幅模式或学习的稀疏掩码，将复杂度降低到O(N√N)或O(N log N)。
线性注意力近似[10, 11, 12]使用核特征映射重新 formulation注意力，实现O(N)计算，
但牺牲了对任意成对交互建模的能力。低秩因式分解[13, 14]将注意力矩阵近似为较小
矩阵的乘积，以表达能力为代价实现效率。虽然这些方法降低了理论复杂度，但它们引入的
近似误差在深度网络中累积，在长程建模基准测试上通常导致2-5%的准确率下降[15]。
或许更重要的是，它们从根本上改变了注意力机制，使得难以将标准注意力的进步
（如旋转位置嵌入、分组查询注意力）应用于高效变体。
```

**关键特征**：
- 对先前工作的有序分类
- 每种方法的复杂度明确指出
- 局限性清晰识别
- 量化缺点（2-5%准确率下降）
- 识别更深层问题（与进步不兼容）

---

### 第三段：你的方法（高层视角）

```
我们采用不同的方法：与其近似注意力，我们通过优化内存访问模式来加速精确注意力。
我们的关键观察是，在现代GPU上，注意力受限于内存带宽，而非计算。从GPU高带宽内存
（HBM）读写N×N注意力矩阵占据了运行时间的主导地位，而GPU的张量核心却利用不足。
我们提出LongFlash，一种IO感知的精确注意力算法，在快速的片上SRAM中逐块计算注意力，
从不将完整的注意力矩阵物化到HBM中。通过精心编排平铺模式并将softmax计算与矩阵乘法
融合，LongFlash将HBM访问从O(N²)降低到O(N²d/M)，其中d是头维度，M是SRAM大小，
实现了渐近最优的IO复杂度。
```

**关键特征**：
- 明确与先前工作区分（"不同方法"）
- 关键洞察明确陈述
- 技术机制解释
- 复杂度改进量化
- 方法名称引入

---

### 第四段：贡献（关键）

```
我们的贡献如下：

• 我们提出LongFlash，一种IO感知的精确注意力算法，在1K到128K tokens的序列上
  相比FlashAttention[16]实现2-4倍加速，相比标准PyTorch注意力实现最高9倍加速
  （第三节）。

• 我们提供理论分析，证明LongFlash在所有计算精确注意力的算法中实现最优的
  O(N²d/M) IO复杂度，并分析我们的算法提供最大收益的 regime（第三节第三部分）。

• 我们引入序列并行技术，使LongFlash能够扩展到多个GPU上处理1M+ tokens的序列，
  具有接近线性的弱扩展效率（第四节）。

• 我们证明LongFlash能够在相同硬件上训练8倍更长的上下文：我们在128K token上下文中
  训练70亿参数模型，使用相同的内存此前只能支持16K tokens（第五节）。

• 我们发布优化的CUDA内核，在A100和H100 GPU上实现理论峰值FLOPS的80%，
  以及PyTorch和JAX绑定，发布于[匿名URL]（第六节）。
```

**关键特征**：
- 编号/项目符号格式
- 每个贡献都是具体和量化的
- 每个声明都有章节引用
- 方法论和实证贡献兼有
- 提及代码发布
- 自包含的项目符号（每个单独看都有意义）

## 替代开头段落

### 方法论文

```
可扩展优化算法是现代机器学习的基础。随机梯度下降（SGD）及其变体[1, 2, 3]使得
能够在大规模数据集上训练数十亿参数的模型。然而，这些一阶方法在病态问题上收敛缓慢，
二阶方法可能数十次迭代就能收敛的任务，SGD往往需要数千次迭代[4, 5]。
```

### 应用论文

```
药物发现是一个昂贵且耗时的过程，平均新药需要10-15年和26亿美元来开发[1]。
机器学习提供了加速这一过程的潜力，通过预测分子性质、识别有希望的候选物和
计算优化先导化合物[2, 3]。蛋白质结构预测[4]和分子生成[5]的最新成功表明，
深度学习可以捕获复杂的化学模式，这提高了对机器学习驱动药物发现的期待。
```

### 理论论文

```
理解为什么深度神经网络尽管参数多于训练样本却能良好泛化，这仍是现代机器学习的
核心难题之一[1, 2]。经典统计学习理论预测，这样过度参数化的模型应该严重过拟合，
而实际上，用SGD训练的大型网络实现了优异的测试准确率[3]。理论与实践之间的差距
激发了关于隐式正则化[5]、神经切核[5]和特征学习[6]的丰富文献，但完整的理论
图景仍未出现。
```

## 贡献项目符号模板

### 新方法

```
• 我们提出[方法名称]，一种通过[关键创新]实现相比[基线]在[基准测试上][性能提升]的
  [方法类型]。
```

### 理论分析

```
• 我们证明了[声明]，提供了[问题设置]的首个[结果类型]。这解决了[先前工作]的
  开放问题。
```

### 实证研究

```
• 我们对[M]个数据集上的[N]种方法进行了全面评估，揭示了[关键发现]并识别了
  [失败模式/最佳实践]。
```

### 代码/数据发布

```
• 我们发布[资源名称]，一个包含[规模/范围]的[描述]，发布于[URL]。这实现了
  [未来工作/可复现性]。
```

## 需要避免的常见错误

### 模糊的贡献

❌ **错误**：
```
• 我们提出了一种用于注意力的新方法
• 我们展示了我们的方法优于基线
• 我们提供了理论分析
```

✅ **正确**：
```
• 我们提出LongFlash，实现相比FlashAttention 2-4倍加速
• 我们证明LongFlash实现最优的O(N²d/M) IO复杂度
• 我们在固定硬件预算下实现8倍更长上下文训练
```

### 缺少量化

❌ **错误**："我们的方法显著优于先前工作"
✅ **正确**："我们的方法在GLUE上提升3.2%准确率，在SuperGLUE上提升4.1%"

### 重叠的项目符号

❌ **错误**：
```
• 我们提出了一种新的注意力机制
• 我们引入了LongFlash注意力
• 我们的新型注意力方法...
```
（这说了三遍同样的事）

### 埋没的贡献

❌ **错误**：贡献项目符号在第二页末尾
✅ **正确**：贡献项目符号在第一页末尾清晰可见
