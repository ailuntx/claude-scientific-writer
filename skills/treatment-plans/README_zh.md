# 治疗计划技能

## 概述

用于在所有临床专科生成**简洁、以临床医生为中心**的医学治疗计划的技能。提供带有SMART目标框架、循证干预、监管合规和患者为中心护理计划验证工具的LaTeX/PDF模板。

**默认采用1页格式**用于大多数情况——想想"快速参考卡"而非"综合教科书"。

## 包含内容

### 📋 七种治疗计划类型

1. **单页治疗计划**（首选）- 用于大多数临床场景的简洁、快速参考格式
2. **普通内科治疗计划** - 初级保健、慢性疾病（糖尿病、高血压、心力衰竭）
3. **康复治疗计划** - 物理治疗、职业治疗、心肺康复
4. **心理健康治疗计划** - 精神科护理、抑郁、焦虑、创伤后应激障碍、药物滥用
5. **慢性疾病管理计划** - 复杂多发病、长期护理协调
6. **围手术期护理计划** - 术前优化、ERAS协议、术后恢复
7. **疼痛管理计划** - 急性和慢性疼痛、多模式镇痛、阿片类药物节约策略

### 📚 参考文件（5本综合指南）

## 快速开始

### 生成治疗计划模板

```bash
cd .claude/skills/treatment-plans/scripts
python generate_template.py

# 或直接指定类型
python generate_template.py --type general_medical --output diabetes_plan.tex
```

可用模板类型：
- `one_page`（首选 - 用于大多数情况）
- `general_medical`
- `rehabilitation`
- `mental_health`
- `chronic_disease`
- `perioperative`
- `pain_management`

### 编译为PDF

```bash
cd /path/to/your/treatment/plan
pdflatex my_treatment_plan.tex
```

### 验证您的治疗计划

```bash
# 检查完整性
python check_completeness.py my_treatment_plan.tex

# 全面验证
python validate_treatment_plan.py my_treatment_plan.tex
```

### 生成治疗时间线

```bash
python timeline_generator.py --plan my_treatment_plan.tex --output timeline.pdf
```

## 标准治疗计划组件

所有模板都包含以下必要部分：

### 1. 患者信息（去标识化）
- 人口统计学和相关医疗背景
- 活跃疾病和合并症
- 当前用药和过敏史
- 功能状态基线
- HIPAA合规的去标识化

### 2. 诊断和评估摘要
- 主要诊断（ICD-10编码）
- 次要诊断
- 严重程度分类
- 功能限制
- 风险分层

### 3. 治疗目标（SMART格式）

**短期目标**（1-3个月）：
- 具体、可测量的结果
- 有明确时间框架的现实目标
- 以患者为中心的优先事项

**长期目标**（6-12个月）：
- 疾病控制目标
- 功能改善目标
- 生活质量提升
- 并发症预防

### 4. 干预措施

- **药物**：含剂量、频率、监测的药物
- **非药物**：生活方式调整、行为干预、教育
- **程序**：计划程序、专科转诊、诊断检查

### 5. 时间和日程
- 治疗阶段及时间框架
- 预约频率
- 里程碑评估
- 预期治疗持续时间

### 6. 监测参数
- 要追踪的临床结果
- 评估工具和量表
- 监测频率
- 干预阈值

### 7. 预期结果
- 主要结果指标
- 成功标准
- 改善时间线
- 长期预后

### 8. 随访计划
- 预约日程
- 沟通协议
- 紧急程序
- 过渡计划

### 9. 患者教育
- 疾病理解
- 自我管理技能
- 预警体征
- 资源和支持

### 10. 风险缓解
- 不良反应管理
- 安全监测
- 紧急行动计划
- 跌倒/感染预防

```
- 介入治疗：若反应不足考虑硬膜外注射
```

## SMART目标框架

所有治疗计划都使用SMART标准来设定目标：

- **具体**：清晰、明确的结果（而非模糊）
- **可测量**：可量化指标或可观察行为
- **可实现**：基于患者能力和资源的现实目标
- **相关**：与患者优先事项和价值观一致
- **有时限**：实现的具体时间框架

### 示例

**良好的SMART目标**：
- 3个月内将HbA1c从8.5%降至<7%
- 8周后借助辅具独立行走150英尺
- 8周内将PHQ-9抑郁评分从18降至<10
- 术后第14天达到膝关节屈曲>90度
- 6周内将疼痛从7/10降至≤4/10

**不佳的目标**（非SMART）：
- "感觉更好"（不具体或不可测量）
- "改善糖尿病"（不具体或无时间限制）
- "变得更强壮"（不可测量）
- "恢复正常"（模糊、不具体）

## 工作流程示例

### 标准治疗计划工作流程

1. **评估患者** - 完成病史、体格检查、诊断检查
2. **选择模板** - 根据临床情境选择合适的模板
3. **生成模板** - `python generate_template.py --type [type]`
4. **定制计划** - 填写患者特定信息（去标识化）
5. **设定SMART目标** - 定义可测量的短期和长期目标
6. **指定干预措施** - 循证的药物和非药物干预
7. **创建时间线** - 安排预约、里程碑、重新评估
8. **定义监测** - 结果指标、评估频率
9. **验证完整性** - `python check_completeness.py plan.tex`
10. **质量检查** - `python validate_treatment_plan.py plan.tex`
11. **审查质量清单** - 与`quality_checklist.md`比较
12. **生成PDF** - `pdflatex plan.tex`
13. **与患者审查** - 共同决策，确认理解
14. **实施和记录** - 执行计划，在临床笔记中追踪进展
15. **重新评估和修改** - 根据结果调整计划

### 多学科护理计划工作流程

1. **确定团队成员** - 主治医师、专科医生、治疗师、病例管理员
2. **创建基础计划** - 为主要疾病生成模板
3. **添加专科部分** - 整合会诊建议
4. **协调目标** - 确保各学科之间的一致性
5. **定义沟通** - 团队会议日程、文档共享
6. **分配责任** - 明确谁管理每个干预措施
7. **创建护理时间线** - 协调各提供者的预约
8. **分享计划** - 分发给所有团队成员和患者
9. **共同追踪** - 共享监测和结果追踪
10. **定期团队审查** - 协作调整计划

## 最佳实践

✓ 让患者参与目标设定和决策

✓ 与患者价值观和生活情况保持一致

✓ 签署并注明所有治疗计划日期  
✓ 保持专业文档标准

## 专业标准

### 文档要求
- 完整准确的患者信息
- 清晰诊断及适当的ICD-10编码
- 循证干预措施
- 可测量的目标和结果

治疗计划必须证明：
- 干预措施的医学适当性

- 知情同意文档

- 风险披露和缓解

## 常见问题

### 如何选择正确的模板？

将模板与您的主要临床重点匹配：
- **慢性内科疾病** → general_medical 或 chronic_disease
- **术后或损伤** → rehabilitation 或 perioperative
- **精神科疾病** → mental_health
- **疼痛作为主要问题** → pain_management

### 如果我的患者有多种疾病怎么办？

对于复杂多发病使用`chronic_disease_management_plan.tex`模板，或选择主要疾病的模板并为合并症添加部分。

### 治疗计划应该多久更新一次？

- **初始创建**：诊断或治疗开始时
- **定期更新**：慢性疾病每3-6个月
- **重大变化**：目标达成或治疗修改时
- **年度审查**：所有慢性疾病计划的最低要求

### 我可以修改LaTeX模板吗？

可以！模板设计为可定制的。修改部分、添加专科特定内容，或调整格式以满足您的需求。

### 如何确保HIPAA合规？

- 移除所有18个HIPAA标识符（参见安全港方法）
- 使用年龄范围而非确切年龄（例如"60-65"而非"63"）
- 移除具体日期，使用相对时间线
- 省略小于州的地理标识符
- 使用clinical-reports技能中的`check_deidentification.py`脚本

### 如果验证脚本发现问题怎么办？

查看识别的具体问题，参考文件中的指南，并相应修改计划。常见问题包括：
- 缺少必需部分
- 目标不符合SMART标准
- 监测参数不足
- 药物信息不完整
