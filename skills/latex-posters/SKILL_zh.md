# LaTeX 研究海报

## 概述

研究海报是会议、研讨会和学术活动中进行科学交流的关键媒介。本技能为使用 LaTeX 包创建专业、视觉上吸引人的研究海报提供全面指导。通过合理的布局、字体、配色方案和视觉层次，生成出版质量的海报。

## 何时使用本技能

在以下情况下应使用本技能：
- 为会议、研讨会或海报展示环节制作研究海报
- 为大学活动或论文答辩设计学术海报
- 为公众参与准备研究视觉摘要
- 将科学论文转换为海报格式
- 为研究小组或部门创建海报模板
- 设计符合特定会议尺寸要求（A0、A1、36×48 英寸等）的海报
- 构建具有复杂多栏布局的海报
- 在海报纸格式中集成图表、表格、公式和参考文献

## 基于 AI 的视觉元素生成

**标准工作流程：在创建 LaTeX 海报之前，使用 AI 生成所有主要的视觉元素。**

这是创建视觉上引人注目的海报的推荐方法：
1. 规划所需的所有视觉元素（标题、引言、方法、结果、结论）
2. 使用 scientific-schematics 或 Nano Banana Pro 生成每个元素
3. 将生成的图像组装到 LaTeX 模板中
4. 在视觉元素周围添加文本内容

**目标：海报区域 60-70% 应为 AI 生成的视觉效果，30-40% 为文本。**

---

### 关键：防止内容溢出

**⚠️ 海报边缘绝对不能出现文本或内容被截断的情况。**

**常见的溢出问题：**
1. **标题/页脚文本超出页面边界**
2. **在可用空间中塞入过多章节**
3. **图表放置得离边缘太近**
4. **文本块超出列宽**

**预防规则：**

**1. 限制内容章节数（A0 海报最多 5-6 个章节）：**
```
✅ 好的——5 个章节，有足够呼吸空间：
   - 标题/页眉
   - 引言/问题
   - 方法
   - 结果（1-2 个关键发现）
   - 结论

❌ 不好的——8 个以上章节拥挤在一起：
   - 概述、引言、背景、方法、
   - 结果 1、结果 2、讨论、结论、未来工作
```

**2. 在 LaTeX 中设置安全边距：**
```latex
% tikzposter - 添加宽裕的边距
\documentclass[25pt, a0paper, portrait, margin=25mm]{tikzposter}

% baposter - 确保内容不触及边缘
\begin{poster}{
  columns=3,
  colspacing=2em,           % 列间距
  headerheight=0.1\textheight,  % 较小的页眉高度
  % 底部留出空间
}
```

**3. 图形尺寸——绝不用 100% 宽度：**
```latex
% 在图形四周留出间距
\includegraphics[width=0.85\linewidth]{figure.png}  % 不要用 1.0\linewidth
```

**4. 打印前检查溢出：**
```bash
# 编译并以 100% 缩放检查 PDF
pdflatex poster.tex

# 查找：
# - 任何边缘有文本被截断
# - 内容触及页面边界
# - .log 文件中的 overfull hbox 警告
grep -i "overfull" poster.log
```

**5. 字数限制：**
- **A0 海报**：最多 300-800 字
- **每个章节**：最多 50-100 字
- **如果内容更多**：删减或做成讲义

### 关键：海报显示所需字体大小

**⚠️ AI 生成的图形中所有文本都必须适合海报阅读。**

在生成海报图形时，你必须在每个提示中包含字体大小规范。海报图形是从 4-6 英尺外观看的，所以文本必须大。

**⚠️ 常见问题：内容溢出和密度过大**

AI 生成的海报图形最常见的问题是**内容太多**。这会导致：
- 文本溢出边界
- 字体太小难以阅读
- 视觉效果杂乱不堪
- 空白空间利用不佳

**解决方案：生成简单、内容最少的图形。**

**每个海报图形都必须包含的提示要求：**

```
海报格式要求（严格执行）：
- 每个图形绝对最多 3-4 个元素（3 个为理想状态）
- 整个图形中文字总数最多 10 个
- 没有包含 5 个以上步骤的复杂工作流（拆分为 2-3 个简单图形）
- 没有多层嵌套示意图（扁平化为单层）
- 没有包含多个子部分的案例分析（每个案例只突出一个关键点）
- 所有文本为巨型粗体（标签 80pt+，关键数字 120pt+）
- 仅使用高对比度（深色在白色上或白色在深色上，文本不要使用渐变）
- 强制最少 50% 的空白空间（图形的一半应为空白）
- 仅使用粗线条（最小 5px+），大图标（最小 200px+）
- 每个图形只传达一个信息（不是 3 个相关信息）
```

**⚠️ 生成前：检查提示并数元素数量**
- 如果描述中有 5 个以上项目 → 停止。拆分为多个图形
- 如果工作流有 5 个以上阶段 → 停止。只展示 3-4 个高层次步骤
- 如果对比有 4 种以上方法 → 停止。只展示前 3 种或“我们的方法”与最佳基线

**每种图形类型的严格内容限制：**
| 图形类型 | 最多元素 | 最多字数 | 拒绝条件 | 好的示例 |
|---------|---------|-------|-------|--------|
| 流程图 | **3-4 个框** | **8 个字** | 5+ 阶段、嵌套步骤 | “发现→验证→批准”（3 个字） |
| 关键发现 | **3 个项目** | **9 个字** | 4+ 个指标、段落 | “95% 准确率”“快 2 倍”“FDA 就绪”（6 个字） |
| 对比图 | **3 个柱** | **6 个字** | 4+ 种方法、图例文字 | “我们的：95%” “最佳：85%”（4 个字） |
| 案例分析 | **1 个案例，3 个元素** | **6 个字** | 多个案例、子故事 | 徽标 +“18 个月”+“到发现”（2 个字） |
| 时间线 | **3-4 个点** | **8 个字** | 逐年详细信息 | “2020 开始”“2022 试验”“2024 获批”（6 个字） |

**示例——错误（7 阶段工作流，过于复杂）：**
```bash
# ❌ 不好——这会创建如同药物发现海报那样小到无法阅读的文字
python scripts/generate_schematic.py "药物发现工作流显示：阶段 1 靶点识别，阶段 2 分子合成，阶段 3 虚拟筛选，阶段 4 AI 先导化合物优化，阶段 5 临床试验设计，阶段 6 FDA 批准。包含每个阶段的成功指标、时间线和验证步骤。" -o figures/workflow.png
# 结果：7+ 个阶段，文字太小，从 6 英尺外无法阅读——海报失败
```

**示例——正确（简化为 3 个关键阶段）：**
```bash

python scripts/generate_schematic.py "A0 海报格式。超简单 3 框工作流：‘发现’ → ‘验证’ → ‘批准’。每个词用巨型粗体（120pt+）。粗箭头（10px）。60% 空白空间。没有子步骤，没有细节。总共 3 个字。可从 10 英尺外阅读。" -o figures/workflow_overview.png
# 结果：干净、有冲击力、可读——如果需要，可单独添加细节图形
```

**示例——错误（包含多个部分的复杂案例分析）：**
```bash
# ❌ 不好——创建出拥挤且无法阅读的章节
python scripts/generate_schematic.py "案例研究：Insilico Medicine（候选药物、发现时间、临床试验），Recursion Pharma（平台、方法论、结果），Exscientia（候选药物、FDA 状态、时间线）。包含公司标志、指标和成果。" -o figures/cases.png
```

**海报提示中使用的字体大小参考：**
| 元素 | 最小大小 | 提示关键词 |
|------|-------|----------|
| 主要数字/指标 | 72pt+ | “巨大”、“非常大”、“巨型”、“海报尺寸” |
| 章节标题 | 60pt+ | “大粗体”、“突出” |
| 标签/说明 | 36pt+ | “从 6 英尺外可读”、“清晰标签” |
| 正文 | 24pt+ | “海报可读”、“大文本” |

**始终包含在提示中：**
- “海报格式”或“用于 A0 海报”或“从 6 英尺外可读”
- “非常大的文本”或“巨型粗体字体”
- 应出现的具体文本（以便嵌入图像中）
- “文本最少，冲击力最大”
- “高对比度”以确保可读性
- “宽裕的边距”和“文本不靠近边缘”

### 关键：AI 生成图形的尺寸

**⚠️ 每个 AI 生成的图形应聚焦一个概念，且内容最少。**

**问题**：生成包含许多元素的复杂图表会导致文字变小。

**解决方案**：生成简单的图形，元素少，文字大。

**示例——错误（太复杂，文字会很小）：**
```bash
# 不好——一个图形中元素太多
python scripts/generate_schematic.py "完整的 ML 流水线，显示数据收集、
包含 5 个步骤的预处理、包含 8 种技术的特征工程、包含超参数调优的模型训练、
包含交叉验证的验证，以及包含监控的部署。包含所有标签和描述。" -o figures/pipeline.png
```

**示例——正确（简单、聚焦、大文字）：**
```bash
# 好的——拆分为多个简单的图形，文字大

# 图形 1：高层次概览（最多 3-4 个元素）
python scripts/generate_schematic.py "A0 海报格式：简单 4 步流水线。
四个大框：数据 → 处理 → 模型 → 结果。
巨型标签（80pt+），粗箭头，大量空白空间。
总共仅 4 个字。可从 8 英尺外阅读。" -o figures/overview.png

# 图形 2：关键结果（突出显示 1 个指标）
python scripts/generate_schematic.py "A0 海报格式：单个关键指标显示。
巨型‘95%’文字（150pt+），下方‘准确率’（60pt+）。
对勾图标。极简设计，高对比度。
可从 10 英尺外阅读。" -o figures/accuracy.png
```

**AI 生成的海报图形规则：**
| 规则 | 限制 | 原因 |
|------|------|------|
| **每图形元素数** | 最多 3-5 个 | 元素越多，文字越小 |
| **每图形字数** | 最多 10-15 个 | 文字最少，字体更大 |
| **流程图步骤** | 最多 4-5 步 | 保持标签可读 |
| **图表类别数** | 最多 3-4 个 | 防止拥挤 |
| **嵌套层数** | 最多 1-2 层 | 避免复杂 |

**将复杂内容拆分为多个简单图形：**
```
不要用 1 个包含 12 个元素的复杂图表：
→ 创建 3 个各含 4 个元素的简单图表
→ 每个图形可以有更大的文字
→ 在布局中排列得清晰，视觉上要有明确的流程
```

### 步骤 0：强制性的预生成审查（首先执行）

**⚠️ 在生成任何图形之前，审查你的内容规划：**

**对于每个计划生成的图形，提出以下问题：**
1. **元素计数**：我能否用 3-4 个或更少的项目描述它？
   - ❌ 不能 → 简化或拆分为多个图形
   - ✅ 能 → 继续

2. **复杂度检查**：这是否是一个多阶段工作流（5+ 步骤）或嵌套示意图？
   - ❌ 是 → 扁平化为仅 3-4 个高层次步骤
   - ✅ 否 → 继续

3. **字数统计**：我能否用 10 个字或更少描述所有文字？
   - ❌ 不能 → 删减文字，使用单词标签
   - ✅ 能 → 继续

4. **信息清晰度**：这个图形是否传达一个明确的信息？
   - ❌ 否 → 拆分为多个聚焦的图形
   - ✅ 是 → 继续生成

**始终失败的常见模式（拒绝这些）：**
- “显示阶段 1 到 7……” → 拆分为高层次概览（3 个阶段）+ 细节图形
- “多个案例研究……” → 每个图形一个案例
- “时间线从 2015 年到 2024 年，每年里程碑……” → 仅显示 3-4 个关键年份
- “6 种方法的比较……” → 仅显示前 3 个或“我们的方法”与最佳基线
- “包含所有层和连接的架构……” → 仅高层次（3-4 个组件）

### 步骤 1：规划你的海报元素

通过预生成审查后，确定所需的视觉元素：

1. **标题块** - 带有机构品牌元素的风格化标题（可选——可以是 LaTeX 文本）
2. **引言图形** - 概念概览（最多 3 个元素）
3. **方法图表** - 高层次工作流（最多 3-4 步）
4. **结果图** - 关键发现（每个图最多 3 个指标，可能需要 2-3 个独立图）
5. **结论图形** - 视觉摘要（最多 3 个要点）
6. **补充图标** - 简单的图标、二维码、徽标（极简）

### 步骤 2b：强制性的生成后审查（在组装之前）

**⚠️ 关键：在添加到海报之前，审查每一个生成的图形。**

**对于每个生成的图，以 25% 缩放打开并检查：**

1. **✅ 通过标准（必须全部为真）：**
   - 在 25% 缩放下所有文字清晰可读
   - 数元素个数：3-4 个或更少
   - 空白空间：图像 50% 以上为空白
   - 足够简单，2 秒内就能理解
   - 不是包含 5 个以上阶段的复杂工作流
   - 不是多个嵌套章节

2. **❌ 失败标准（如果有任何一项为真，则重新生成）：**
   - 在 25% 缩放下文字小或难以阅读 → 用“150pt+”的字体重新生成
   - 元素超过 4 个 → 用“仅 3 个元素”重新生成
   - 空白空间少于 50% → 用“60% 空白空间”重新生成
   - 多阶段复杂工作流 → 拆分成 2-3 个简单图形
   - 多个案例分析拥挤在一起 → 拆分成多个独立图形
   - 花费超过 3 秒才能理解 → 简化并重新生成

**常见失败及修复：**
- “7 阶段工作流，文字极小” → 重新生成“仅 3 个高层次阶段”
- “3 个案例分析在一个图形中” → 生成 3 个独立的简单图形
- “时间线有 8 个年份” → 重新生成“仅 3 个关键里程碑”
- “5 种方法的对比” → 重新生成“仅我们的方法与最佳基线（2 个柱）”

**如果任何图形未通过上述检查，不要继续组装。**

### 视觉元素指南

**⚠️ 关键：每个图形必须有一个信息，并且最多 3-4 个元素。**

**绝对限制——这些不是指南，而是硬性限制：**
- **每个图形最多 3-4 个元素**（3 个为理想状态）
- **每个图形最多 10 个字**
- **最少 50% 空白空间**（60% 更好）
- **关键数字/指标最少 120pt**
- **标签最少 80pt**

**对于每个海报章节——严格要求：**

| 章节 | 最多元素 | 最多字数 | 示例提示（必需的句式） |
|------|------|------|------|
| **引言** | 3 个图标 | 6 个字 | “A0 海报格式：超简单，仅 3 个图标：[图标1] [图标2] [图标3]。一个词标签（100pt 粗体）。60% 空白空间。总共 3 个字。” |
| **方法** | 3 个框 | 6 个字 | “A0 海报格式：超简单，3 框工作流：‘步骤1’ → ‘步骤2’ → ‘步骤3’。巨型标签（120pt+）。60% 空白空间。仅 3 个词。” |
| **结果** | 2-3 个柱 | 6 个字 | “A0 海报格式：两个柱：‘基线 70%’ ‘我们的 95%’。柱上巨型百分比（150pt+）。没有坐标轴。60% 空白空间。” |
| **结论** | 3 张卡片 | 9 个字 | “A0 海报格式：三张卡片：‘95%’（150pt）‘准确’、‘2X’（150pt）‘更快’、对勾‘就绪’。60% 空白空间。” |
| **案例分析** | 3 个元素 | 5 个字 | “A0 海报格式：一个案例：徽标 + ‘18 个月’（150pt） + ‘到发现’（60pt）。60% 空白空间。” |
| **时间线** | 3 个点 | 3 个字 | “A0 海报格式：仅三个年份：‘2018’ ‘2021’ ‘2024’（每个 150pt）。大图标。60% 空白空间。没有细节。” |

**强制性的提示元素（所有都必须，无例外）：**
1. **“A0 海报格式”** - 必须是第一个
2. **“超简单”** 或 **“仅 X 个元素”** - 内容限制
3. **“巨型（120pt+）”** 或指定字号 - 可读性
4. **“60% 空白空间”** - 强制留白
5. **“可从 10-12 英尺外阅读”** - 观看距离
6. **确切数量** 字/元素 - “总共 3 个字”或“仅 3 个图标”

**始终失败的模式（立即拒绝）：**
- ❌ “7 阶段药物发现工作流” → 拆分为“3 个超级阶段”
- ❌ “2015-2024 年时间线，每年更新” → “仅 3 个关键年份”
- ❌ “3 个案例研究及详情” → 制作 3 个独立简单图形
- ❌ “5 种方法对比及指标” → “仅 2 个：我们的与最佳”
- ❌ “显示所有层的完整架构” → “仅 3 个组件”
- ❌ “显示阶段 1,2,3,4,5,6” → “3 个高层次阶段”

**行之有效的模式：**
- ✅ “从 7 个合并为 3 个超级阶段” → 适当的简化
- ✅ “一个案例，一个指标” → 如果需要的多，就制作多个
- ✅ “仅 3 个里程碑” → 有选择，聚焦
- ✅ “2 个柱：我们的与基线” → 直接对比
- ✅ “3 组件高层次视图” → 适当简化

### 1. LaTeX 海报包

**beamerposter**：
- Beamer 演示文稿类的扩展

- 内置颜色主题和布局模板

### 2. 海报布局和结构

遵循视觉传达原则创建有效的海报布局。综合布局指导，请参考 `references/poster_layout_design.md`。

**常见的海报章节**：
- **页眉/标题**：标题、作者、单位、徽标
- **引言/背景**：研究背景和动机
- **方法/途径**：方法论和实验设计
- **结果**：关键发现，包含图表和数据可视化
- **结论**：主要要点和影响
- **参考文献**：关键引用（通常缩写）
- **致谢**：资助、合作者、机构

**布局策略**：
- **基于列的布局**：2 列、3 列或 4 列网格
- **基于块的布局**：内容块的灵活排列
- **Z 字形流程**：引导读者通过内容逻辑
- **视觉层次**：使用大小、颜色和间距强调关键点

### 3. 研究海报的设计原则

应用基于证据的设计原则以获得最大影响力。详细设计指导，请参考 `references/poster_design_principles.md`。

**字体排版**：
- 标题：72-120pt，以便在一定距离内可见
- 章节标题：48-72pt
- 正文：最小 24-36pt，以便从 4-6 英尺外可读
- 使用无衬线字体（Arial、Helvetica、Calibri）以提高清晰度
- 最多使用 2-3 个字体族

**颜色和对比度**：
- 使用高对比度配色方案以确保可读性
- 机构调色板用于品牌
- 色盲友好调色板（避免红-绿组合）
- 空白是活动空间——不要过度拥挤

**视觉元素**：
- 高分辨率图形（打印最小 300 DPI）
- 所有图形上的标签大而清晰
- 贯穿全文的图形样式一致
- 策略性地使用图标和图形
- 平衡文本与视觉内容（推荐 40-50% 视觉）

**内容指南**：
- **少即是多**：建议总共 300-800 字
- 使用项目符号而不是段落，便于浏览
- 信息清晰、简洁
- 图形能自我说明，文字解释最少
- 为补充材料或在线资源提供二维码

### 4. 标准海报尺寸

支持国际和会议特定的海报尺寸：

**国际标准**：
- A0（841 × 1189 mm / 33.1 × 46.8 英寸）——最常见的欧洲标准
- A1（594 × 841 mm / 23.4 × 33.1 英寸）——较小的格式
- A2（420 × 594 mm / 16.5 × 23.4 英寸）——紧凑的海报

**北美标准**：
- 36 × 48 英寸（914 × 1219 mm）——常见的美国会议尺寸
- 42 × 56 英寸（1067 × 1422 mm）——大幅面
- 48 × 72 英寸（1219 × 1829 mm）——超大幅面

**方向**：
- 肖像（垂直）——最常见，传统
- 风景（水平）——更适合宽内容，时间线

   ```bash
   python scripts/generate_schematic.py "A0 海报格式。简单图表，仅 3 个柱。柱上巨型百分比（120pt+）。没有坐标轴、没有图例。50% 空白空间。" -o figures/results.png
   
   # 结论——恰好 3 个项目，带巨型数字
   python scripts/generate_schematic.py "A0 海报格式。恰好 3 个关键发现：‘[数字]’（150pt）‘[标签]’（60pt）各自。50% 空白空间。没有其他文字。" -o figures/conclusions.png
   ```

3. **审查生成的图形——检查溢出：**
   - **以 25% 缩放查看**：所有文字仍然可读吗？
   - **数元素个数**：多于 5 个？ → 重新生成更简单的
   - **检查空白空间**：少于 40%？ → 在提示中添加“60% 空白空间”
   - **字体太小？**：添加“更大”或增加 pt 值
   - **仍然溢出？**：减少为 3 个元素而不是 4-5 个

### 阶段 3：设计和布局

1. **选择或创建模板**：
   - 从 `assets/` 中提供的模板开始
   - 自定义配色方案以匹配品牌
   - 配置页面大小和方向

2. **设计布局结构**：
   - 规划列结构（2、3 或 4 列）
   - 映射内容流（通常从左到右，从上到下）
   - 为标题（10-15%）、内容（70-80%）、页脚（5-10%）分配空间

3. **设置字体排版**：
   - 为不同层次级别配置字体大小
   - 确保正文至少 24pt
   - 测试从 4-6 英尺距离的可读性

### 阶段 4：内容集成

1. **创建海报页眉**：
   - 标题（简洁、描述性，10-15 个字）
   - 作者和单位
   - 机构徽标（高分辨率）
   - 会议徽标（如有需要）

2. **集成 AI 生成的图形**：
   - 将阶段 2 中的所有图形添加到适当章节
   - 使用 `\includegraphics` 并设置适当大小
   - 确保图形在每个章节占主导（视觉第一，文本第二）
   - 将图形居中于块内以获得视觉冲击力

3. **添加最少的支持文本**：
   - 保持文本最少且易于浏览（总共 300-800 字）
   - 使用项目符号，不要使用段落
   - 用主动语态写作
   - 文本应补充图形，而非重复图形

4. **添加补充元素**：
   - 补充材料的二维码
   - 参考文献（仅引用关键论文，通常 5-10 篇）
   - 联系信息和致谢

### 阶段 5：细化和测试

1. **审查和迭代**：
   - 检查拼写错误和错误
   - 核实所有图形都是高分辨率
   - 确保格式一致
   - 确认配色方案协调良好

2. **测试可读性**：
   - 以 25% 比例打印并从 2-3 英尺外阅读（模拟从 8-12 英尺外看海报）
   - 在不同的显示器上检查颜色
   - 验证二维码功能正常
   - 请同事审查

3. **优化打印**：
   - 在 PDF 中嵌入所有字体
   - 验证图像分辨率
   - 检查 PDF 大小要求
   - 如有需要，包含出血区域

### 阶段 6：编译和交付

1. **编译最终 PDF**：
   ```bash
   pdflatex poster.tex
   # 或为更好的字体支持：
   lualatex poster.tex
   ```

2. **验证输出质量**：
   - 检查所有元素都可见且位置正确
   - 放大到 100% 并检查图形质量
   - 验证颜色符合预期
   - 确认 PDF 在不同的查看器中正确打开

3. **准备打印**：
   - 如有需要，导出为 PDF/X-1a
   - 保存备份副本
   - 先在普通纸上进行测试打印
   - 在截止日期前 2-3 天订购专业打印

4. **创建补充材料**：
   - 保存 PNG/JPG 版本用于社交媒体
   - 创建讲义版本（8.5×11 英寸摘要）
   - 准备数字版本以便电子邮件分享

## 避免的常见陷阱

**AI 生成图形错误（最常见）：**
- ❌ 一个图形中元素太多（10+ 个项目） → 保持 3-5 个
- ❌ AI 图形中文字太小 → 指定“巨型（100pt+）”或“超大（150pt+）”
- ❌ 提示中细节太多 → 使用“简单”和“仅 X 个元素”
- ❌ 没有指定空白空间 → 在每个提示中添加“50% 空白空间”
- ❌ 包含 8+ 步骤的复杂流程图 → 限制在 4-5 步
- ❌ 包含 6+ 个项目的对比图 → 限制在 3 个项目
- ❌ 包含 5+ 个指标的关键发现 → 仅显示前 3 个

**修复 AI 图形中的溢出：**
如果 AI 生成的图形溢出或文字很小：
1. 在提示中添加“更简单”或“仅 3 个元素”
2. 增加字体大小：“150pt+”而不是“80pt+”
3. 添加“60% 空白空间”而不是“50%”
4. 删除子细节：“没有子步骤”、“没有坐标轴标签”、“没有图例”
5. 用更少的元素重新生成

**设计错误**：
- ❌ 文字太多（超过 1000 字）
- ❌ 字体大小太小（正文小于 24pt）
- ❌ 低对比度颜色组合
- ❌ 布局杂乱，没有空白空间
- ❌ 各章节样式不一致
- ❌ 质量差或像素化的图像

**内容错误**：
- ❌ 没有清晰的叙述或信息
- ❌ 太多研究问题或目标
- ❌ 过度使用术语且没有定义
- ❌ 结果没有上下文或解释
- ❌ 缺少作者联系信息

**技术错误**：
- ❌ 海报尺寸不符合会议要求
- ❌ RGB 颜色发送给 CMYK 打印机（颜色偏移）
- ❌ PDF 中未嵌入字体
- ❌ 文件大小太大，无法提交
- ❌ 二维码太小或未测试

**最佳实践**：
- ✅ 生成简单的 AI 图形，每个最多 3-5 个元素
- ✅ 对图形中的关键数字使用超大字体（100pt+）
- ✅ 在每个 AI 提示中指定“50% 空白空间”
- ✅ 严格遵循会议尺寸规格
- ✅ 在最终打印前以缩小比例测试打印
- ✅ 使用高对比度、可访问的配色方案
- ✅ 保持文本最少且高度可浏览
- ✅ 包含明确的联系信息和二维码
- ✅ 仔细校对（海报上的错误会被放大！）
