# 科学假设生成

## 概述

假设生成是开发可检验解释的系统性过程。基于观察形成循证假设，设计实验，探索竞争性解释，并发展预测。将此技能应用于跨领域的科学探究。

## 何时使用此技能

在以下情况下应使用此技能：
- 基于观察或初步数据发展假设
- 设计实验来检验科学问题
- 探索现象的竞争性解释
- 为研究制定可检验的预测
- 进行基于文献的假设生成
- 跨科学领域规划机制研究

## 工作流程

遵循此系统性流程来生成 robust 的科学假设：

### 1. 理解现象

首先澄清需要解释的观察、问题或现象：

- 确定需要解释的核心观察或模式
- 定义现象的范围和边界
- 注意任何约束或特定上下文
- 明确已知与不确定的内容
- 确定相关的科学领域

### 2. 进行综合文献检索

搜索现有科学文献以使假设基于当前证据。使用 PubMed（生物医学主题）和通用网络搜索（更广泛的科学领域）：

**对于生物医学主题：**
- 使用 WebFetch 访问 PubMed URL 以获取相关文献
- 搜索最近的综述、荟萃分析和原创研究
- 寻找类似现象、相关机制或类似系统

**对于所有科学领域：**
- 使用 WebSearch 查找最近的论文、预印本和综述
- 搜索已确立的理论、机制或框架
- 确定当前理解的空白

**检索策略：**
- 从广泛检索开始以了解整体情况
- 缩小到特定机制、途径或理论
- 寻找矛盾发现或未解决的争论
- 参阅 `references/literature_search_strategies.md` 获取详细检索技术

### 3. 综合现有证据

分析和整合文献检索的发现：

- 总结对该现象的当前理解
- 确定可能适用的已确立机制或理论
- 注意矛盾的证据或替代观点
- 认识空白、局限性或未回答的问题
- 从相关系统或领域识别类比

### 4. 生成竞争性假设

开发3-5个可以解释该现象的不同假设。每个假设应该：

- 提供机制解释（不仅仅是描述）
- 可以与其他假设区分开来
- 借鉴文献综合中的证据
- 考虑不同层次的解释（分子、细胞、系统、群体等）

**生成假设的策略：**
- 从类似系统应用已知机制
- 考虑多种致病途径
- 探索不同层次的解释
- 质疑现有解释中的假设
- 以新颖方式组合机制

### 5. 评估假设质量

根据 `references/hypothesis_quality_criteria.md` 中确立的质量标准评估每个假设：

**可检验性：** 假设能否被实证检验？
**可证伪性：** 什么观察可以否定它？
**简洁性：** 它是否是最简单且符合证据的解释？
**解释力：** 它能解释多少现象？
**范围：** 它涵盖多大范围的观察？
**一致性：** 它是否与确立的原则一致？
**新颖性：** 它是否提供超出现有解释的新见解？

明确注明每个假设的优势和劣势。

### 6. 设计实验检验

对于每个可行的假设，提出具体的实验或研究来检验它。参阅 `references/experimental_design_patterns.md` 获取常见方法：

**实验设计要素：**
- 将测量或观察什么？
- 需要什么比较或对照？
- 将使用什么方法或技术？
- 什么样本量或统计方法合适？
- 有什么潜在的混淆因素以及如何解决？

**考虑多种方法：**
- 实验室实验（体外、体内、计算）
- 观察性研究（横断面、纵向、病例对照）
- 临床试验（如适用）
- 自然实验或准实验设计

### 7. 制定可检验预测

为每个假设生成具体、定量的预测：

- 说明如果假设正确应该观察到什么
- 如可能，指定预期的影响方向和程度
- 确定预测应成立的条件
- 区分竞争假设之间的预测
- 注明可以证伪假设的预测

### 8. 展示结构化输出

使用 `assets/hypothesis_report_template.tex` 中的模板生成专业的 LaTeX 文档。报告应格式良好，使用彩色框进行视觉组织，分为简洁的正文和全面的附录。

**文档结构：**

**正文（最多4页）：**
1. **执行摘要** - 摘要框中的简要概述（0.5-1页）
2. **竞争性假设** - 每个假设放在自己的彩色框中，包含简要机制解释和关键证据（3-5个假设共2-2.5页）
   - **重要：** 在每个假设框之前使用 `\newpage` 以防止内容溢出
   - 每个框最多0.6页
3. **可检验预测** - 琥珀色框中的关键预测（0.5-1页）
4. **关键比较** - 优先比较框（0.5-1页）

正文应极为简洁 - 仅包含最必要的信息。所有详细信息移至附录。

**分页策略：**
- 在假设框之前始终使用 `\newpage` 以确保它们在新页面开始
- 这防止内容溢出页面边界
- LaTeX 框（tcolorbox）不会自动跨页分割

**附录（全面、详细）：**
- **附录A：** 全面的文献综述，包含广泛引用
- **附录B：** 详细实验设计，包含完整方案
- **附录C：** 质量评估表格和详细评价
- **附录D：** 补充证据和类似系统

**彩色框使用：**

使用 `hypothesis_generation.sty` 中的自定义框环境：

- `hypothesisbox1` 到 `hypothesisbox5` - 用于每个竞争性假设（蓝色、绿色、紫色、青色、橙色）
- `predictionbox` - 用于可检验预测（琥珀色）
- `comparisonbox` - 用于关键比较（钢灰色）
- `evidencebox` - 用于支持证据高亮（浅蓝色）
- `summarybox` - 用于执行摘要（蓝色）

**每个假设框应包含（为4页限制保持简洁）：**
- **机制解释：** 简要说明1-2段（最多6-10句）解释如何以及为何
- **关键支持证据：** 2-3个带引用的要点（仅最重要的证据）
- **核心假设：** 1-2个关键假设

所有详细解释、补充证据和综合讨论属于附录。

**关键溢出预防：**
- 在每个假设框之前插入 `\newpage` 以使其在新页面开始
- 将每个完整假设框保持在 ≤0.6 页（约15-20行内容）
- 如果内容超过此值，将额外细节移至附录A
- 永远不要让框溢出页面边界 - 这会产生无法读取的PDF

**引用要求：**

争取广泛引用以支持所有声明：
- **正文：** 10-15个关键引用，仅用于最重要的证据（为4页限制保持简洁）
- **附录A：** 40-70+全面引用，涵盖所有相关文献
- **目标总计：** 参考文献目标50+篇

正文引用应有选择性 - 仅引用最关键的论文。所有综合引用和详细文献讨论属于附录。使用 `\citep{author2023}` 进行圆括号引用。

**LaTeX编译：**

模板需要 XeLaTeX 或 LuaLaTeX 才能正确渲染：

```bash
xelatex hypothesis_report.tex
bibtex hypothesis_report
xelatex hypothesis_report.tex
xelatex hypothesis_report.tex
```

**所需包：** `hypothesis_generation.sty` 样式包必须位于同一目录或 LaTeX 路径中。它需要：tcolorbox, xcolor, fontspec, fancyhdr, titlesec, enumitem, booktabs, natbib。

**页面溢出预防：**

为防止内容溢出页面，请遵循以下关键准则：

1. **监控框内容长度：** 每个假设框应能舒适地放在单页上。如果内容超过约0.7页，可能会溢出。

2. **使用战略性分页：** 在包含大量内容的框之前插入 `\newpage`：
   ```latex
   \newpage
   \begin{hypothesisbox1}[Hypothesis 1: Title]
   % Long content here
   \end{hypothesisbox1}
   ```

3. **保持正文框简洁：** 对于4页正文限制：
   - 每个假设框：最多0.5-0.6页
   - 机制解释：仅1-2个简要段落（最多6-10句）
   - 关键证据：仅2-3个要点
   - 核心假设：仅1-2项
   - 如果内容更长，将细节移至附录

4. **分割长内容：** 如果假设需要广泛解释，请在正文和附录之间分配：
   - 正文框：简要机制概述 + 2-3个关键证据点
   - 附录A：详细机制解释、全面的证据、扩展讨论

5. **测试页面边界：** 在每个新框之前，考虑剩余页面空间是否足够。如果剩余空间少于0.6页，使用 `\newpage` 在新页面开始框。

6. **附录页面管理：** 在附录中，在主要部分之间使用 `\newpage` 以避免详细内容区域溢出。

**快速参考：** 参阅 `assets/FORMATTING_GUIDE.md` 获取所有框类型、配色方案和常见格式模式的详细示例。

## 质量标准

确保所有生成的假设符合这些标准：

- **循证：** 基于现有文献并有引用
- **可检验：** 包含具体、可测量的预测
- **机制性：** 解释如何/为何，而不仅仅是是什么
- **全面：** 考虑替代解释
- **严格：** 包含检验预测的实验设计
